# Proposal: Capture Instructions [20-24]（提案：捕获指令 [20-24]）

> **Prerequisite:** 本规范基于 [Base Constraints](0.Base-Constraints.md) 文档，该文档定义了统一使用的指令格式、运行时模型与记号约定。


## 1. Overview（概览）

捕获指令占用 opcode 范围 **[20-24]**（共 5 条）。它们是 **prefix modifiers**，用于拦截、重定向或切换其他指令周围的数据流来源。捕获指令并不直接对 data stack 本身运算，而是改变“紧随其后的那条指令”与运行时环境交互的方式。

这类指令是脚本表达能力的关键。它把指令从纯栈机固定的 push/pop 模式中解耦出来，使数据可以在 argument space、local scope 与循环变量之间灵活路由。

### Design Principle（设计原则）

捕获指令的 mnemonic 使用符号（`@`、`~`、`$`）而非单词，可直接贴附到目标指令上，不强制要求空格分隔。这种紧凑写法体现了它们“轻量修饰符”而非独立操作的角色。

### Key Distinction（关键区分）

5 条捕获指令中：
- **Three are value-retrieving**（`@`、`$[n]`、`$X[n]`）：将值放入 argument space。
- **One is value-storing**（`$`）：将输出重定向到 local scope。
- **One is source-switching**（`~`）：强制参数直接从 stack 消费。


## 2. Instruction Definitions（指令定义）

### 2.1 Argument Capture（参数捕获）

```
20      @           Argument capture
```

- **Auxiliary params:** None
- **Associated data:** None
- **Behavior:** 拦截 **immediately following** 指令的返回值，并将其放入 argument space（FIFO 队列），而不是压入 data stack。
- **Examples:**
  - `@PEEKS[2]`：查看栈底 2 项，并展开写入 argument space。
  - `@{1234}`：将整数 1234 放入 argument space（原本会自动压入栈）。
- **Semantics:**
  - 若后续指令返回多个值（如 `PEEKS` 返回 N 项），会按顺序全部追加到 argument space。
  - 捕获效果只作用于紧接的一条指令，不会向后传播。

### 2.2 Direct Stack Consumption（直接栈消费）

```
21      ~           Direct stack consumption
```

- **Auxiliary params:** None
- **Associated data:** None
- **Behavior:** 强制紧随其后的指令 **directly from the data stack** 取参数，完全绕过 argument space，即使 argument space 中已有值也不使用。
- 这是显式的数据通道切换指令。
- **Ordering with other prefixes:** 若目标指令同时带有捕获前缀（`@` 或 `$`），`~` 必须位于捕获前缀之后、目标指令之前：
  ```
  @~EQUAL    — take operands from stack, capture result to argument space
  $~EQUAL    — take operands from stack, store result to local scope
  ```
- **Examples:**
  - `~EQUAL`：从栈顶取两个值做相等比较（忽略 argument space）。
  - `@~EQUAL`：同样比较，但结果写入 argument space。
  - `$~EQUAL`：同样比较，但结果写入 local scope。
- **Design note:** 显式支持“直接走栈”能显著提升数据编排灵活性，尤其当 argument space 中缓存的是给后续指令使用的值时。

### 2.3 Local Scope Store（本地作用域存储）

```
22      $           Local scope store
```

- **Auxiliary params:** None
- **Associated data:** None
- **Behavior:** 拦截紧随其后指令的返回值，并将其 **appends** 到当前 block 的 local scope，而不是压入 data stack。
- local scope 是每个语法块独有的 **append-only** 静态存储区，容量上限为 128 条目。
- **Examples:**
  - `$TOP`：引用栈顶项并追加到 local scope（引用拷贝）。
  - `$POPS[3]`：弹出栈顶 3 项，全部展开写入 local scope（新增 3 条）。
  - `${100}`：将值 100 直接写入 local scope。
  - `$SHIFT[3]`：取栈顶 3 项并打包为单个 slice，再追加到 local scope（新增 1 条）。
- **Scope isolation:** local scope 严格块私有。不同块（含同级块和嵌套块）不能互访彼此 local scope。仅同一语法层级内的指令共享同一个 local scope。

### 2.4 Local Scope Retrieve（本地作用域读取）

```
23      $(1)        Local scope retrieve (by reference)
```

- **Auxiliary params:** 1 byte，成员索引（signed int8，范围 `[-128, 127]`）。
- **Associated data:** None
- **Behavior:** 按索引从当前 block 的 local scope 读取值，并放入 **argument space**。
- 负索引从末尾反向计数：`$[-1]` 表示最后追加的一项。
- 必须与先前 `$`（store）操作配合使用。
- **Examples:**
  - `$[0]`：读取 local scope 中第一项。
  - `$[-1]`：读取最近一次写入的值。
- **Possible extension:** 后续可扩展按名称读取（如 `${POP}` 引用 `$POP` 存储值），但同名多次存储会带来命名冲突问题。

### 2.5 Loop Variable Reference（循环变量引用）

```
24      $X(1)       Loop variable reference
```

- **Auxiliary params:** 1 byte，变量标识 `[0-3]`。
- **Associated data:** None
- **Behavior:** 在循环体（`EACH{}`）内读取循环上下文变量，并写入 **argument space**。与 local scope（`$[n]`）不同，循环变量引用可以在所属循环范围内 **cross syntactic block boundaries**。
- **Variable identifiers:**

  | Index | Named Reference | Description |
  |-------|----------------|-------------|
  | 0 | `$Value` | 当前迭代元素值 |
  | 1 | `$Index` / `$Key` | 当前迭代键：slice 为整数索引，dict 为字符串键 |
  | 2 | `$Slice` / `$Dict` | 当前被迭代的集合 |
  | 3 | `$Size` | 集合大小（成员数量） |

- 命名引用 **case-sensitive**，且都以 `$` 前缀书写。
- 具体可用别名由迭代目标决定：迭代 slice 时使用 `$Index`、`$Slice`；迭代 dict 时使用 `$Key`、`$Dict`。
- **Cross-block access:** 与块私有的 `$[n]` 不同，`$Value`、`$Index` 等可在所属 `EACH{}` 的任意嵌套深度访问（包括 `IF{}`、`BLOCK{}` 等子块）。


## 3. Prefix Ordering Rules（前缀顺序规则）

当多个捕获前缀组合使用时，必须遵循以下严格顺序：

```
[capture: @ or $] [source-switch: ~] <target instruction>
```

| Combination | Meaning |
|-------------|---------|
| `@INSTR` | 捕获 INSTR 结果 -> argument space |
| `$INSTR` | 捕获 INSTR 结果 -> local scope |
| `~INSTR` | 强制 INSTR 从 stack 读取参数（绕过 argument space） |
| `@~INSTR` | 强制 stack 取参 + 结果捕获到 argument space |
| `$~INSTR` | 强制 stack 取参 + 结果捕获到 local scope |

非法组合：
- `~@INSTR`：顺序错误（source-switch 不能在 capture 之前）。
- `@$INSTR`：目标冲突（不能同时捕获到 argument space 和 local scope）。


## 4. Data Flow Summary（数据流摘要）

```graph
                   ┌─────────────┐
                   │  Data Stack │ <-- default push target
                   └──────┬──────┘
                          │ fallback source
                          v
┌──────────────┐   ┌─────────────┐   ┌──────────────┐
│ Local Scope  │   │  Argument   │   │   Target     │
│   $[n]       │<--│   Space     │-->│  Instruction │
│   $X[n]      │   │  (FIFO)     │   │              │
└──────────────┘   └─────────────┘   └──────────────┘
       ^                  ^
       │                  │
    $ prefix           @ prefix
    (store)           (capture)

    ~ prefix: forces target to bypass argument space -> read from stack directly
```

三个读取类捕获指令（`@`、`$[n]`、`$X[n]`）都会把结果写入 argument space，供下一条消费参数的指令使用。存储指令（`$`）把结果改写到 local scope。source-switch（`~`）会覆盖参数来源优先级。
