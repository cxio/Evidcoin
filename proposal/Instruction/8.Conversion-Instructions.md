# Proposal: Conversion Instructions [67‚Äì79]

> **Prerequisite:** This specification builds upon the [Base Constraints](0.Base-Constraints.md) document, which defines the instruction format, runtime model, and notation conventions used throughout.


## 1. Overview

Conversion instructions occupy opcode range **[67‚Äì79]** (13 instructions). They transform data between different **types and values** ‚Äî not just type coercion, but potentially changing the underlying representation. For example, converting a string `"123"` to integer `123` changes both type and internal representation.

### Design Principles

- **No conversion to Script type:** For security reasons, arbitrary data cannot be converted into executable scripts. The `Script` type can only be created via `CODE{}`, `SCRIPT`, or loaded through `GOTO`/`EMBED`.
- **Basic types are freely interconvertible** (with range/format constraints).
- **Compound types** (slices, dicts) have limited conversion paths.
- Conversions that would lose data or produce ambiguous results cause **execution failure** rather than silent truncation.


## 2. Instruction Definitions

### 2.1 To Boolean

```
67      BOOL        Convert to Bool
```

- **Arguments:** A single value of any type.
- **Return value:** Bool (`true` or `false`).
- **Conversion rules:**

  | Source Type | false | true |
  |------------|-------|------|
  | Nil | always | ‚Äî |
  | Byte | `0` | non-zero |
  | Rune | `0` | non-zero |
  | Int | `0` | non-zero |
  | BigInt | `0` | non-zero |
  | Float | `<= math.SmallestNonzeroFloat64` | above threshold |
  | String | `""` (empty) | non-empty |

- **Tip:** To test whether a collection is non-empty: `SIZE BOOL`.

### 2.2 To Byte

```
68      BYTE        Convert to Byte
```

- **Arguments:** A single value.
- **Return value:** Byte (`uint8`, range `[0, 255]`).
- **Conversion rules:**
  - Nil ‚Üí `0`
  - Bool ‚Üí `false` = 0, `true` = 1
  - Rune ‚Üí valid if code point < 256; otherwise **error**
  - Int ‚Üí valid if value in `[0, 255]`; otherwise **error**
  - BigInt ‚Üí same range constraint
  - Float ‚Üí truncated to integer first, then range check
- **Not supported:** String ‚Üî Byte conversion.

### 2.3 To Rune

```
69      RUNE        Convert to Rune
```

- **Arguments:** A single value.
- **Return value:** Rune (`uint32`, range `[0, 0x10FFFF]`).
- **Conversion rules:**
  - Nil ‚Üí `0`
  - Bool ‚Üí `false` = 0, `true` = 1
  - Byte ‚Üí direct type widening, value unchanged
  - Int ‚Üí valid if `<= 0x10FFFF`; otherwise **error**
  - BigInt ‚Üí same range constraint
  - Float ‚Üí truncated to integer first, then range check
- **Not supported:** String ‚Üî Rune conversion.

### 2.4 To Integer

```
70      INT         Convert to Int (int64)
```

- **Arguments:** A single value.
- **Return value:** Int (`int64`).
- **Conversion rules:**
  - Nil ‚Üí `0`
  - Bool ‚Üí `false` = 0, `true` = 1
  - Byte ‚Üí direct type widening
  - Rune ‚Üí direct type widening
  - BigInt ‚Üí valid if within int64 range; otherwise **error**
  - Float ‚Üí fractional part truncated, then range check
- **Extended support:**
  - Time ‚Üí extracts UNIX timestamp in milliseconds.
  - String ‚Üí parsed as integer literal using Go `strconv` rules:
    - Decimal: `"12345"`
    - Binary: `"0b1010"` / `"0B1010"`
    - Octal: `"0o777"` / `"0O777"`
    - Hexadecimal: `"0xFF"` / `"0XFF"`
    - Underscore separators permitted: `"1_000_000"`

### 2.5 To Big Integer

```
71      BIGINT      Convert to BigInt
```

- **Arguments:** A single value.
- **Return value:** BigInt (`*big.Int`).
- **Conversion rules:**
  - Nil ‚Üí `BigInt(0)`
  - Bool ‚Üí `false` = 0, `true` = 1
  - Byte ‚Üí direct widening
  - Rune ‚Üí direct widening
  - Int ‚Üí direct widening
- **Extended support:**
  - Bytes ‚Üí interpreted as big-endian encoding (`math/big` convention).

### 2.6 To Float

```
72      FLOAT       Convert to Float (float64)
```

- **Arguments:** A single value.
- **Return value:** Float (`float64`).
- **Conversion rules:**
  - Nil ‚Üí `0.0`
  - Bool ‚Üí `false` = 0.0, `true` = 1.0
  - Byte ‚Üí integer-to-float
  - Rune ‚Üí integer-to-float
  - Int ‚Üí standard conversion
- **Extended support:**
  - String ‚Üí parsed as float literal or scientific notation.

### 2.7 To String

```
73      STRING(1){} Convert to String
```

- **Auxiliary params:** 1 byte ‚Äî base or format specifier.
- **Associated data:** Format character (when using `{}` notation for float formats).
- **Arguments:** A single value.
- **Return value:** String.
- **Format specifiers:**
  - **Integer bases** (using `[]` notation): `2`‚Äì`36`. E.g., `STRING[16]` ‚Üí hexadecimal string.
  - **Float formats** (using `{}` notation): `b`, `e`, `E`, `f`, `g`, `G`, `x`, `X`. E.g., `STRING{E}` ‚Üí exponential notation.
- **Conversion rules:**
  - Nil ‚Üí `""` (empty string)
  - Bool ‚Üí `"true"` / `"false"`
  - Byte ‚Üí treated as UTF-8 byte; displays valid character or hex representation
  - Rune ‚Üí treated as Unicode code point; displays character or `\uhhhh` / `\Uhhhhhhhh` form
  - Int ‚Üí converted using base specifier (via `strconv`)
  - BigInt ‚Üí converted using base specifier
  - Float ‚Üí converted using format specifier
- **Extended support:**
  - Bytes ‚Üí treated as UTF-8 encoded byte sequence, displayed as string
  - `[]Rune` ‚Üí UTF-8 encoded (`[]rune` ‚Üí `string`)

### 2.8 To Bytes

```
74      BYTES       Convert to Bytes ([]byte)
```

- **Arguments:** A single value.
- **Return value:** Bytes (`[]byte`).
- **Conversion rules:**
  - Nil ‚Üí empty byte slice
  - Rune ‚Üí UTF-8 encoded bytes
  - Int ‚Üí big-endian encoding, fixed 8 bytes
  - BigInt ‚Üí big-endian encoding, variable length
- **Extended support:**
  - String ‚Üí UTF-8 encoded bytes
  - `[]Rune` ‚Üí UTF-8 encoded bytes
  - Script ‚Üí clone of the source bytecode (type conversion only)
- **Security note:** Reverse conversion (Bytes ‚Üí Script) is **not supported**, preventing injection of arbitrary bytecode.

### 2.9 To Rune Slice

```
75      RUNES       Convert to []Rune
```

- **Arguments:** A single value.
- **Return value:** `[]Rune` (rune slice).
- A special-case conversion for convenient Unicode character-level manipulation.
- **Conversion rules:**
  - Nil ‚Üí empty rune slice
  - Rune ‚Üí single-element rune slice
  - String ‚Üí UTF-8 decoded to `[]rune`
  - Bytes ‚Üí UTF-8 decoded; invalid sequences become `\uFFFD` (replacement character)

### 2.10 To Time

```
76      TIME        Convert to Time
```

- **Arguments:** A single value.
- **Return value:** Time object.
- **Conversion rules:**
  - Int ‚Üí interpreted as UNIX timestamp in milliseconds
  - String ‚Üí parsed as RFC 3339 format
- **Examples:**
  - `{1608883593536} TIME` ‚Üí equivalent to `DATE{2020-12-25T08:06:33.536Z}`
  - `"2020-12-25T08:06:33.536+08:00" TIME` ‚Üí equivalent to `DATE{2020-12-25T08:06:33.536+08:00}`

### 2.11 To Regular Expression

```
77      REGEXP      Convert to RegExp
```

- **Arguments:** A string value.
- **Return value:** RegExp object (RE2).
- Only supports conversion from String. The string must be a valid RE2 expression.

### 2.12 Construct Dict

```
78      DICT        Construct a Dict (map[string]any)
```

- **Arguments:**
  1. Key sequence (`[]string`).
  2. Value sequence (`[]any`).
- **Return value:** A new Dict with insertion-order iteration.
- **Pairing rules:**
  - Keys and values are paired by positional correspondence.
  - If keys outnumber values, excess keys receive `nil` values.
  - If values outnumber keys, excess values are **ignored**.
- **Example:**
  ```
  {"one", "two", "123"}
  {100} {200} NIL {1.0}
  DICT
  // Result: {"one": 100, "two": 200, "123": nil}
  // The value 1.0 is ignored (only 3 keys).
  ```
- **Design note:** Dict is not a fundamental script type and cannot be created via value instructions. This conversion instruction provides dict construction as a utility.
- **Iteration order:** Members iterate in insertion order (deterministic).

### 2.13 Slice Type Conversion (ANYS)

```
79      ANYS(1){}   Slice type conversion ([]any ‚Üî []T)
```

- **Auxiliary params:** 1 byte ‚Äî conversion target identifier.
- **Associated data:** Optional type name (when using `{}` notation).
- **Arguments:** A slice or `nil`.
- **Return value:** A converted slice.
- Converts between concrete typed slices (`[]Int`, `[]String`, etc.) and generic `[]any` slices, or vice versa.
- **Target identifiers:**

  | ID | Direction | Result Type |
  |----|-----------|-------------|
  | 0 | `[]T` ‚Üí `[]any` | `[]any` |
  | 1, 2 | `[]any` ‚Üí `[]Bool` | `[]Bool` |
  | 3 | `[]any` ‚Üí `[]Byte` | `[]Byte` (same as Bytes) |
  | 4 | `[]any` ‚Üí `[]Rune` | `[]Rune` |
  | 5 | `[]any` ‚Üí `[]Int` | `[]Int` |
  | 6 | `[]any` ‚Üí `[]*big.Int` | `[]*big.Int` |
  | 7 | `[]any` ‚Üí `[]Float` | `[]Float` |
  | 8, 9 | `[]any` ‚Üí `[]String` | `[]String` |

- **Design note:** Except for `0`, the identifier values match the corresponding value instruction opcodes (1/2 = TRUE/FALSE ‚Üí Bool, 3 = Byte literal, etc.).
- Named syntax is also supported: `ANYS{String}` is equivalent to `ANYS[8]`.
- `nil` converts to an empty slice of the target type.
- An empty `[]any` converts to an empty slice of any target type.
- Member values are automatically converted to the target type during the transformation.
- **Examples:**
  ```
  NIL ANYS          // ‚Üí empty []any
  {1,3,5} ANYS      // []Int ‚Üí []any
  "AB" "XY" SHIFT[2] ANYS[8]       // []any ‚Üí []String
  "AB" "XY" SHIFT[2] ANYS{String}  // same as above
  {0} NIL TRUE SHIFT[3] ANYS{Bool} // ‚Üí [false, false, true]
  ```
- **Use case:** Dict values are `any`-typed, so converting concrete typed slices to `[]any` is sometimes necessary for dict operations.
