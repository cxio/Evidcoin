# Proposal: Conversion Instructions [67–79]（提案：转换指令 [67–79]）

> **Prerequisite:** 本规范建立在 [Base Constraints](0.Base-Constraints.md) 文档之上；该文档定义了全局采用的指令格式、运行时模型与记号约定。


## 1. Overview（概览）

转换指令占用 opcode 范围 **[67–79]**（共 13 条）。它们用于在不同**类型与值表示**之间做转换，不仅是类型强转，也可能改变底层表示。例如把字符串 `"123"` 转成整数 `123`，类型与内部表示都会变化。

### Design Principles（设计原则）

- **No conversion to Script type:** 出于安全考虑，任意数据都不能转换为可执行 Script。`Script` 只能通过 `CODE{}`、`SCRIPT` 创建，或由 `GOTO`/`EMBED` 加载。
- 基础类型可在约束范围内自由互转（受范围/格式限制）。
- 复合类型（slice、dict）仅支持有限的转换路径。
- 凡是会导致数据丢失或结果歧义的转换，均触发**执行失败**，而不是静默截断。


## 2. Instruction Definitions（指令定义）

### 2.1 To Boolean（转 Bool）

```
67      BOOL        Convert to Bool
```

- **Arguments:** 任意类型的单个值。
- **Return value:** Bool（`true` 或 `false`）。
- **Conversion rules:**

  | Source Type | false | true |
  |------------|-------|------|
  | Nil | 总是 | — |
  | Byte | `0` | 非 0 |
  | Rune | `0` | 非 0 |
  | Int | `0` | 非 0 |
  | BigInt | `0` | 非 0 |
  | Float | `<= math.SmallestNonzeroFloat64` | 高于阈值 |
  | String | `""`（空） | 非空 |

- **Tip:** 需要判断集合是否非空时，可用：`SIZE BOOL`。

### 2.2 To Byte（转 Byte）

```
68      BYTE        Convert to Byte
```

- **Arguments:** 单个值。
- **Return value:** Byte（`uint8`，范围 `[0, 255]`）。
- **Conversion rules:**
  - Nil → `0`
  - Bool → `false` = 0，`true` = 1
  - Rune → code point < 256 时有效，否则 **error**
  - Int → 值在 `[0, 255]` 内有效，否则 **error**
  - BigInt → 同样受该范围约束
  - Float → 先截断为整数，再做范围校验
- **Not supported:** String ↔ Byte 转换。

### 2.3 To Rune（转 Rune）

```
69      RUNE        Convert to Rune
```

- **Arguments:** 单个值。
- **Return value:** Rune（`uint32`，范围 `[0, 0x10FFFF]`）。
- **Conversion rules:**
  - Nil → `0`
  - Bool → `false` = 0，`true` = 1
  - Byte → 直接扩宽，数值不变
  - Int → `<= 0x10FFFF` 时有效，否则 **error**
  - BigInt → 同样受该范围约束
  - Float → 先截断为整数，再做范围校验
- **Not supported:** String ↔ Rune 转换。

### 2.4 To Integer（转 Int）

```
70      INT         Convert to Int (int64)
```

- **Arguments:** 单个值。
- **Return value:** Int（`int64`）。
- **Conversion rules:**
  - Nil → `0`
  - Bool → `false` = 0，`true` = 1
  - Byte → 直接扩宽
  - Rune → 直接扩宽
  - BigInt → 位于 int64 范围内有效，否则 **error**
  - Float → 截断小数部分后再做范围校验
- **Extended support:**
  - Time → 提取 UNIX 毫秒时间戳。
  - String → 按 Go `strconv` 规则解析整数文本：
    - 十进制：`"12345"`
    - 二进制：`"0b1010"` / `"0B1010"`
    - 八进制：`"0o777"` / `"0O777"`
    - 十六进制：`"0xFF"` / `"0XFF"`
    - 允许下划线分隔：`"1_000_000"`

### 2.5 To Big Integer（转 BigInt）

```
71      BIGINT      Convert to BigInt
```

- **Arguments:** 单个值。
- **Return value:** BigInt（`*big.Int`）。
- **Conversion rules:**
  - Nil → `BigInt(0)`
  - Bool → `false` = 0，`true` = 1
  - Byte → 直接扩宽
  - Rune → 直接扩宽
  - Int → 直接扩宽
- **Extended support:**
  - Bytes → 按 big-endian 编码解释（`math/big` 约定）。

### 2.6 To Float（转 Float）

```
72      FLOAT       Convert to Float (float64)
```

- **Arguments:** 单个值。
- **Return value:** Float（`float64`）。
- **Conversion rules:**
  - Nil → `0.0`
  - Bool → `false` = 0.0，`true` = 1.0
  - Byte → 整数转浮点
  - Rune → 整数转浮点
  - Int → 标准转换
- **Extended support:**
  - String → 解析浮点文本或科学计数法。

### 2.7 To String（转 String）

```
73      STRING(1){} Convert to String
```

- **Auxiliary params:** 1 byte —— 进制或格式标识。
- **Associated data:** 格式字符（当使用 `{}` 表示浮点格式时）。
- **Arguments:** 单个值。
- **Return value:** String。
- **Format specifiers:**
  - **Integer bases**（`[]` 语法）：`2`–`36`。例如 `STRING[16]` 生成十六进制字符串。
  - **Float formats**（`{}` 语法）：`b`, `e`, `E`, `f`, `g`, `G`, `x`, `X`。例如 `STRING{E}` 使用指数形式。
- **Conversion rules:**
  - Nil → `""`（空字符串）
  - Bool → `"true"` / `"false"`
  - Byte → 视为 UTF-8 byte，显示有效字符或十六进制表示
  - Rune → 视为 Unicode code point，显示字符或 `\uhhhh` / `\Uhhhhhhhh` 形式
  - Int → 按进制标识转换（基于 `strconv`）
  - BigInt → 按进制标识转换
  - Float → 按格式标识转换
- **Extended support:**
  - Bytes → 视为 UTF-8 byte 序列并显示为字符串
  - `[]Rune` → UTF-8 编码（`[]rune` → `string`）

### 2.8 To Bytes（转 Bytes）

```
74      BYTES       Convert to Bytes ([]byte)
```

- **Arguments:** 单个值。
- **Return value:** Bytes（`[]byte`）。
- **Conversion rules:**
  - Nil → 空 byte slice
  - Rune → UTF-8 编码 bytes
  - Int → big-endian 固定 8 bytes 编码
  - BigInt → big-endian 可变长度编码
- **Extended support:**
  - String → UTF-8 编码 bytes
  - `[]Rune` → UTF-8 编码 bytes
  - Script → 源 bytecode 的克隆（仅类型转换）
- **Security note:** 反向转换（Bytes → Script）**不支持**，以防注入任意 bytecode。

### 2.9 To Rune Slice（转 Rune slice）

```
75      RUNES       Convert to []Rune
```

- **Arguments:** 单个值。
- **Return value:** `[]Rune`（rune slice）。
- 这是一个为 Unicode 字符级处理提供便利的特例转换。
- **Conversion rules:**
  - Nil → 空 rune slice
  - Rune → 单元素 rune slice
  - String → UTF-8 解码为 `[]rune`
  - Bytes → UTF-8 解码；非法序列替换为 `\uFFFD`（replacement character）

### 2.10 To Time（转 Time）

```
76      TIME        Convert to Time
```

- **Arguments:** 单个值。
- **Return value:** Time 对象。
- **Conversion rules:**
  - Int → 解释为 UNIX 毫秒时间戳
  - String → 按 RFC 3339 格式解析
- **Examples:**
  - `{1608883593536} TIME` → 等价于 `DATE{2020-12-25T08:06:33.536Z}`
  - `"2020-12-25T08:06:33.536+08:00" TIME` → 等价于 `DATE{2020-12-25T08:06:33.536+08:00}`

### 2.11 To Regular Expression（转 RegExp）

```
77      REGEXP      Convert to RegExp
```

- **Arguments:** 一个字符串值。
- **Return value:** RegExp 对象（RE2）。
- 仅支持从 String 转换，且字符串必须是合法 RE2 表达式。

### 2.12 Construct Dict（构造 Dict）

```
78      DICT        Construct a Dict (map[string]any)
```

- **Arguments:**
  1. Key 序列（`[]string`）。
  2. Value 序列（`[]any`）。
- **Return value:** 新建 Dict，迭代顺序为插入顺序。
- **Pairing rules:**
  - Key 与 Value 按位置配对。
  - Key 多于 Value 时，多出的 Key 对应 `nil`。
  - Value 多于 Key 时，多出的 Value 会被**忽略**。
- **Example:**
  ```
  {"one", "two", "123"}
  {100} {200} NIL {1.0}
  DICT
  // 结果：{"one": 100, "two": 200, "123": nil}
  // 1.0 被忽略（只有 3 个 key）
  ```
- **Design note:** Dict 不是脚本的基础值类型，无法通过 value instructions 直接创建；此转换指令提供了构造工具。
- **Iteration order:** 成员按插入顺序迭代（确定性）。

### 2.13 Slice Type Conversion (ANYS)（Slice 类型转换）

```
79      ANYS(1){}   Slice type conversion ([]any ↔ []T)
```

- **Auxiliary params:** 1 byte —— 目标类型标识。
- **Associated data:** 可选类型名（使用 `{}` 语法时）。
- **Arguments:** 一个 slice 或 `nil`。
- **Return value:** 转换后的 slice。
- 在具体类型 slice（`[]Int`、`[]String` 等）与通用 `[]any` 之间双向转换。
- **Target identifiers:**

  | ID | Direction | Result Type |
  |----|-----------|-------------|
  | 0 | `[]T` → `[]any` | `[]any` |
  | 1, 2 | `[]any` → `[]Bool` | `[]Bool` |
  | 3 | `[]any` → `[]Byte` | `[]Byte` (same as Bytes) |
  | 4 | `[]any` → `[]Rune` | `[]Rune` |
  | 5 | `[]any` → `[]Int` | `[]Int` |
  | 6 | `[]any` → `[]*big.Int` | `[]*big.Int` |
  | 7 | `[]any` → `[]Float` | `[]Float` |
  | 8, 9 | `[]any` → `[]String` | `[]String` |

- **Design note:** 除 `0` 外，标识值与对应 value instruction opcode 对齐（1/2 = TRUE/FALSE → Bool，3 = Byte literal，等等）。
- 同时支持命名写法：`ANYS{String}` 等价于 `ANYS[8]`。
- `nil` 会转换为目标类型的空 slice。
- 空 `[]any` 可转换为任意目标类型的空 slice。
- 转换过程中会自动将成员值转为目标类型。
- **Examples:**
  ```
  NIL ANYS          // → empty []any
  {1,3,5} ANYS      // []Int → []any
  "AB" "XY" SHIFT[2] ANYS[8]       // []any → []String
  "AB" "XY" SHIFT[2] ANYS{String}  // 同上
  {0} NIL TRUE SHIFT[3] ANYS{Bool} // → [false, false, true]
  ```
- **Use case:** Dict 的 value 是 `any` 类型，因此在 dict 操作中，常需要把具体类型 slice 先转为 `[]any`。
