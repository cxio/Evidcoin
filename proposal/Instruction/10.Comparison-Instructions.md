# Proposal: Comparison Instructions [104–111]

> **Prerequisite:** This specification builds upon the [Base Constraints](0.Base-Constraints.md) document, which defines the instruction format, runtime model, and notation conventions used throughout.


## 1. Overview

Comparison instructions occupy opcode range **[104–111]** (8 instructions). They compare two or more values and return a boolean result. These instructions are the primary mechanism for producing boolean conditions that feed into `PASS`, `CHECK`, `IF{}`, and other control flow instructions.

### Float Comparison Caveat

Equality comparison (`EQUAL`) is technically supported for Float values, but should be **used with caution** due to floating-point precision issues. For reliable float comparison, use the `COMFLO` tool instruction instead, which supports epsilon-based approximate equality.


## 2. Instruction Definitions

### 2.1 Relational Comparisons

```
104     EQUAL       Equal (a == b)
105     NEQUAL      Not equal (a != b)
106     LT          Less than (a < b)
107     LTE         Less than or equal (a <= b)
108     GT          Greater than (a > b)
109     GTE         Greater than or equal (a >= b)
```

- **Arguments:** Two comparable values.
- **Return value:** Bool (`true` or `false`).
- **Supported types:**
  - All numeric types: Byte, Rune, Int, BigInt, Float
  - String (lexicographic comparison)
  - Bytes (left-to-right byte-by-byte comparison)
  - Bool: only supported for `EQUAL` and `NEQUAL` (ordering is undefined for booleans)
- **Cross-type comparison:** When comparing values of different numeric types, standard promotion rules apply (Byte → Rune → Int → BigInt/Float).

### 2.2 NaN Check

```
110     ISNAN       Is Not-a-Number
```

- **Arguments:** A single value.
- **Return value:** Bool.
- Applicable to **Float type only**. Equivalent to `math.IsNaN(...)`.
- Returns `true` if the float value is NaN (Not-a-Number), which can arise from undefined floating-point operations (e.g., `0.0 / 0.0`).
- **Design note:** Provided because NaN has the unique property that `NaN != NaN` — standard equality checks cannot detect it.

### 2.3 Range Check

```
111     WITHIN      Range check (min <= x < max)
```

- **Arguments:**
  1. The value to check.
  2. A 2-member slice defining the range: `[min, max]`.
- **Return value:** Bool — `true` if `min <= x < max` (half-open interval: inclusive lower bound, exclusive upper bound).
- **Supported types:** Byte, Rune, Int, BigInt, Float.
- **Example:**
  ```
  ENV{Height} {0, 240} WITHIN
  // Check if the transaction's block height is within the first day
  // (approximately 240 blocks per day)
  ```
- **Float caveat:** The lower-bound equality check (`min <= x`) may be imprecise for floating-point values.
