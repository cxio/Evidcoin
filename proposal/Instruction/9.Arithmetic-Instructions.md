# Proposal: Arithmetic Instructions [80–103]

> **Prerequisite:** This specification builds upon the [Base Constraints](0.Base-Constraints.md) document, which defines the instruction format, runtime model, and notation conventions used throughout.


## 1. Overview

Arithmetic instructions occupy opcode range **[80–103]** (24 instructions). They provide two distinct forms of computation:

1. **Symbol instructions** (opcodes 81–85): Operator symbols (`*`, `/`, `%`, `+`, `-`) used exclusively **inside expressions**.
2. **Named instructions** (opcodes 86–103): Word-based instructions usable both inside and outside expressions.

This dual-form design allows scripts to use natural infix notation for mathematical expressions while retaining the stack-based instruction model for general computation.

### Expression Model

The expression instruction `()` (opcode 80) creates a self-contained arithmetic evaluation context:

- All numeric values inside an expression are **automatically converted to Float (float64)**.
- The expression returns a **single Float value**.
- Expressions can be nested via sub-expressions `((...))`.
- Within an expression, value instructions do not require curly braces — numbers are written in natural notation.
- Values produced by instructions inside an expression become operands, not stack entries.

> **Design decision (confirmed):** Expressions always return Float (float64). This is intentional — expressions are for floating-point computation only. Integer-precise arithmetic must use the named instructions (`MUL`, `ADD`, `SUB`, etc.) outside of expressions.


## 2. Expression Instruction

### 2.1 Expression Wrapper

```
80      ()(1)       Expression encapsulation / grouping
```

- **Auxiliary params:** 1 byte — expression byte length (maximum 255 bytes).
- **Arguments:** None.
- **Return value:** Float (float64).
- Creates an independent computation unit. Inside an expression:
  - Regular instructions can fetch values and compute, but their return values are **operands** — they do not auto-push to stack and cannot be captured by `@` or `$`.
  - Sub-expressions `((...))` serve as priority groupings; their results are also operands.
  - The top-level expression result auto-pushes to the stack (or can be captured by a prefix on the outer `(`).
- **Complexity limit:** The 1-byte auxiliary parameter (not varint) intentionally limits expression length to 255 bytes, constraining complexity.
- **Examples:**
  ```
  ( POP + 100 )             // pop top value, add 100
  ((POP + $[0]) * 10)       // sub-expression with local scope access
  @( POP + 100 )            // capture result to argument space
  ```


## 3. Symbol Instructions (Expression-Only)

These operators are valid **only inside** `()` expressions. They follow standard mathematical precedence: `*`, `/`, `%` bind tighter than `+`, `-`. Sub-expressions `((...))` override precedence.

### 3.1 Multiply

```
81      *           Multiply
```

### 3.2 Divide

```
82      /           Divide
```

### 3.3 Modulo

```
83      %           Modulo
```

### 3.4 Add

```
84      +           Add (unary or binary)
```

### 3.5 Subtract / Negate

```
85      -           Subtract (binary) or Negate (unary)
```

- All symbol operators work on Float operands (auto-converted within the expression context).
- `+` and `-` support both **unary** (single operand) and **binary** (two operands) usage.
- **Examples:**
  ```
  ( -100 )                  // unary negate: -100.0
  ( 100 + POP * 1.5 )       // binary: 100 + (top_value * 1.5)
  ```


## 4. Named Instructions — Standard Arithmetic

### 4.1 Binary Numeric Operations

These instructions take two arguments and return the same type as the input (Int or Float).

```
86      MUL         Multiply: MUL(x, y) → x * y
87      DIV         Divide:   DIV(x, y) → x / y
88      ADD         Add:      ADD(x, y) → x + y
89      SUB         Subtract: SUB(x, y) → x - y
90      MOD         Modulo:   MOD(x, y) → x % y (Int) or math.Mod(x, y) (Float)
91      POW         Power:    POW(x, y) → math.Pow(x, y)
```

- **Supported types:** Int, Float. Return type matches input type.
- **ADD extended behavior:**
  - String + String → concatenation (`"Hello" + "World"` → `"HelloWorld"`)
  - Bytes + Bytes → concatenation
  - Dict + Dict → merge (second dict's entries override first's for shared keys)
- **Examples:**
  ```
  {2} {8} POW       // → 256 (Int)
  {2.0} {8} POW     // → 256.0 (Float)
  "Hello" "World" ADD   // → "HelloWorld"
  ```

### 4.2 Bitwise Operations (Integer-Only)

These instructions operate exclusively on Int values and return Int.

```
92      LMOV        Left shift:   LMOV(x, y) → x << y
93      RMOV        Right shift:  RMOV(x, y) → x >> y
94      AND         Bitwise AND:  AND(x, y)  → x & y
95      ANDX        Bit clear:    ANDX(x, y) → x &^ y
96      OR          Bitwise OR:   OR(x, y)   → x | y
97      XOR         Bitwise XOR:  XOR(x, y)  → x ^ y
```

### 4.3 Unary Operations

```
98      NEG         Negate: -x
```

- **Arguments:** A single numeric value (Int or Float).
- **Return value:** Same type, negated.
- **Example:** `PEEK NEG` — reference stack bottom, return its negation.

```
99      NOT         Logical NOT: !v
```

- **Arguments:** A single Bool value.
- **Return value:** Bool (inverted).


## 5. Named Instructions — Quasi-Arithmetic

These instructions are not strictly mathematical but are categorized under arithmetic for organizational simplicity. They are **not usable inside expressions**.

### 5.1 Divide and Modulo

```
100     DIVMOD      Divide and modulo: DIVMOD(x, y) → x/y, x%y
```

- **Arguments:** Two values (Int or Float).
- **Return value:** Two values — quotient and remainder. Auto-expanded to the target space.

### 5.2 Repeat (Duplicate)

```
101     REP(1)      Repeat (duplicate)
```

- **Auxiliary params:** 1 byte — number of copies.
- **Arguments:** A single value of any type.
- **Return value:** The specified number of copies, auto-expanded.
- Creates multiple shallow copies of the argument value.
- **Edge cases:**
  - `REP[0]` — zero copies: effectively **removes** the top item (counter-intuitive, use with caution).
  - `REP[1]` — one copy: returns the same item (no practical effect).
  - `REP[2]` — two copies: equivalent to `TOP` (duplicates top item).
  - `REP[3]` — three copies: first useful duplication count.
- **Distinction from TOPS:** `TOPS[n]` references N distinct stack items. `REP[n]` duplicates a single item N times.

### 5.3 Delete Dict Entry

```
102     DEL         Delete dict entry
```

- **Arguments:**
  1. Target dict.
  2. Key (string) or key sequence (`[]string`).
- **Return value:** The original dict reference (modified in-place).
- **Examples:**
  ```
  "abc" DEL           // remove key "abc" from dict
  {"A", "B"} DEL      // remove keys "A" and "B"
  ```

### 5.4 Clear Dict

```
103     CLEAR       Clear dict contents
```

- **Arguments:** Target dict.
- **Return value:** The original dict reference (now empty).


## 6. Type Rules Summary

| Category | Instructions | Input Types | Return Type |
|----------|-------------|-------------|-------------|
| Expression symbols | `*`, `/`, `%`, `+`, `-` | Auto-converted to Float | Float |
| Expression wrapper | `()` | — | Float |
| Standard binary | MUL, DIV, ADD, SUB, MOD, POW | Int or Float | Same as input |
| ADD (extended) | ADD | String, Bytes, Dict | Same as input |
| Bitwise | LMOV, RMOV, AND, ANDX, OR, XOR | Int only | Int |
| Unary numeric | NEG | Int or Float | Same as input |
| Unary logic | NOT | Bool | Bool |
| Divide+mod | DIVMOD | Int or Float | Two values, same type |
| Repeat | REP | Any | Same as input |
| Dict operations | DEL, CLEAR | Dict | Dict |
