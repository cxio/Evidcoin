# Proposal: Flow Control Instructions [57–66]

> **Prerequisite:** This specification builds upon the [Base Constraints](0.Base-Constraints.md) document, which defines the instruction format, runtime model, and notation conventions used throughout.


## 1. Overview

Flow control instructions occupy opcode range **[57–66]** (10 instructions, with opcode 65 reserved). They direct the script's execution path through conditional branching, multi-way selection, iteration, and block scoping. All flow control instructions contain **sub-statement blocks** as their associated data — embedded instruction sequences that execute conditionally.

### Design Principle

Flow control instructions produce **no return values**. They affect execution flow only. Any data produced within their sub-blocks is placed onto the shared data stack (or other target spaces via capture prefixes), but the flow control instruction itself returns nothing.

Each sub-statement block creates its own **local scope** (block domain), isolating locally stored values from sibling and parent blocks.

### Important Constraint

The script system has **no infinite loops**. All iteration is bounded by the collection size passed to `EACH{}`. Loop bodies support `BREAK` (to exit early) and `CONTINUE` (to skip to next iteration), but not `RETURN` (which is reserved for functional blocks `MAP{}`/`FILTER{}`).


## 2. Instruction Definitions

### 2.1 Conditional — IF

```
57      IF{}(~)     Conditional execution
```

- **Auxiliary params:** Varint — sub-block byte length.
- **Arguments:** A boolean value.
- **Return value:** None.
- If the argument is `true`, the sub-block executes. If `false`, it is skipped.
- The condition is expressed via the argument, which must be prepared before the IF instruction.
- **IF state:** At the same syntactic level, a later `IF` **overwrites** the state of a prior `IF`. This state is used by `ELSE` to determine its execution.
- **Example:**
  ```
  IF {
       @POP PRINT    // pop and print top item
  }
  ```

### 2.2 Conditional — ELSE

```
58      ELSE{}(~)   Alternative execution
```

- **Auxiliary params:** Varint — sub-block byte length.
- **Arguments:** None.
- **Return value:** None.
- Checks the IF-state at the current syntactic level. If the most recent `IF` evaluated to `false`, the ELSE block executes.
- `ELSE` matches the **nearest preceding** `IF` at the same level.
- `IF` and `ELSE` do not need to be adjacent — other instructions may appear between them.
- **State machine rules:**
  - IF and ELSE are **peer-level** — they cannot nest with each other at the same level, but parent-child nesting is normal.
  - After either an IF block or an ELSE block executes, the IF-state is marked as **completed**.
  - Consecutive ELSE blocks: only the first one after an IF has effect; subsequent ELSE blocks at the same level are **inactive** (illegal usage).
  - All IF/ELSE pairs at the same level share a single state variable.
- **Valid patterns:**
  ```
  IF{} ELSE{} IF{} ELSE{}           // Legal: two independent IF/ELSE pairs
  IF{ IF{} ELSE{} } ELSE{}          // Legal: nested IF/ELSE inside outer IF
  ```
- **Invalid pattern:**
  ```
  IF{} IF{} ELSE{} ELSE{}           // Illegal: second ELSE has no matching IF
  ```
- **Example with interleaved instructions:**
  ```
  IF {
       ...
  }
  @POP PRINT        // does not affect IF/ELSE relationship
  ELSE {
       ...
  }
  ```

### 2.3 Switch Block

```
59      SWITCH{}(~) Multi-way branch block
```

- **Auxiliary params:** Varint — block byte length.
- **Arguments:**
  1. The **target value** — to be compared against CASE branch values. Any comparable type.
  2. A **branch value list** — a sequence of values, one per CASE branch, in positional order.
- **Return value:** None.
- Creates a switch selection context. The target value and all branch comparison values are provided upfront, enabling CASE blocks to be concise (they carry no comparison logic themselves).
- **Design note:** Providing all comparison values at once (rather than per-CASE) keeps individual CASE blocks simple and clean.
- **Tip:** If the target value is `true`, CASE branch values can be boolean expressions, effectively simulating `if/elseif/else` chains.

### 2.4 Case Branch

```
60      CASE{}(~)   Conditional branch within SWITCH
```

- **Auxiliary params:** Varint — sub-block byte length.
- **Arguments:** None (comparison is pre-computed by SWITCH).
- **Return value:** None.
- Within a SWITCH block, each CASE is compared positionally: CASE[0] matches branch value[0], CASE[1] matches branch value[1], etc.
- **First-match semantics:** Only the first matching CASE block executes. After execution, the SWITCH block terminates (no fall-through).
- CASE blocks may also use `BREAK` to exit early.
- **Example:**
  ```
  {10}                // target value for SWITCH
  {3, 10, 20}         // branch values: [0]=3, [1]=10, [2]=20
  SWITCH {
       CASE{          // [0]: compare 3 == 10 → false, skip
            ...
       }
       CASE{...}      // [1]: compare 10 == 10 → true, execute then exit SWITCH
       CASE{...}      // never reached
       DEFAULT{        // never reached (a CASE matched)
            ...
       }
  }
  ```
- **Note:** Instructions may appear between CASE blocks, though this is uncommon.

### 2.5 Default Branch

```
61      DEFAULT{}(~)    Default branch within SWITCH
```

- **Auxiliary params:** Varint — sub-block byte length.
- **Arguments:** None.
- **Return value:** None.
- Executes when all CASE comparisons evaluate to false. Optional — if absent and no CASE matches, the SWITCH block simply exits.
- Should be the last branch within a SWITCH block.

### 2.6 Iteration — EACH

```
62      EACH{}(~)   Bounded iteration (loop)
```

- **Auxiliary params:** Varint — sub-block byte length.
- **Arguments:** A collection (slice or dict) — the iteration target.
- **Return value:** None.
- Iterates over each member of the collection, executing the sub-block once per member.
- **Loop variables** are accessible via `$X` capture instructions (see [Capture Instructions](2.Capture-Instructions.md)):
  - `$Value` (index 0) — current element value.
  - `$Index` / `$Key` (index 1) — current index (slice) or key (dict).
  - `$Slice` / `$Dict` (index 2) — the collection itself.
  - `$Size` (index 3) — collection size.
- Unlike `MAP{}`/`FILTER{}` (tool instructions), EACH does **not** create a private data stack — it operates on the shared data stack.
- To exit the loop early, use `BREAK` (not `RETURN`).
- **Examples:**
  ```
  {1, 3, 5}
  EACH{
       ($Value * 10)     // expression: multiply current value by 10, push result
  }
  @SHIFT[3] PRINT        // → [10, 30, 50]
  ```
  ```
  {10, 20, 30}
  EACH {
       $Value PUSH        // push current value onto data stack
  }
  // equivalent to: {10, 20, 30} SPREAD
  ```

### 2.7 Continue

```
63      CONTINUE    Skip to next iteration
```

- **Auxiliary params:** None
- **Arguments:** Optional — a boolean value.
- **Return value:** None.
- Terminates the current `EACH{}` iteration and advances to the next one.
- If an argument is provided, `CONTINUE` only executes when the value is `true`. If no argument, it executes unconditionally.

### 2.8 Break

```
64      BREAK       Exit loop or switch
```

- **Auxiliary params:** None
- **Arguments:** Optional — a boolean value.
- **Return value:** None.
- Exits the enclosing `EACH{}` loop or `SWITCH{}` block entirely.
- If an argument is provided, `BREAK` only executes when the value is `true`. If no argument, it executes unconditionally.
- **Example:**
  ```
  {1, 3, 30, 5}
  EACH{
       $Value {10} GT     // compare current value > 10
       @POP BREAK          // if true, exit loop
       @$Index PRINT       // print current index
  }
  @"Done!" PRINT

  // Output:
  // 0        (value=1, not > 10)
  // 1        (value=3, not > 10)
  // Done!    (value=30, > 10 → break)
  ```

### 2.9 Reserved

```
65      (Reserved, unused)
```

### 2.10 Block Scope

```
66      BLOCK{}(~)  Block scope
```

- **Auxiliary params:** Varint — sub-block byte length.
- **Arguments:** None.
- **Return value:** None.
- Creates an explicit sub-block with its own local scope. The primary purposes are:
  1. **Local variable isolation** — organizing local scope usage within a larger script.
  2. **Code region demarcation** — defining a contiguous instruction range for extraction via `SOURCE`.
- The block's contents execute normally within the shared data stack context.
- **Example:**
  ```
  BLOCK{
       ......           // normal execution
       SOURCE           // extract current block's instruction sequence onto stack
  }
  @POP PRINT            // print the extracted instruction sequence
  ```
- **Note:** If local scope isolation or `SOURCE` extraction is not needed, using `BLOCK` is equivalent to not using it — it adds no behavioral difference.


## 3. Scoping and State Rules

### 3.1 Block-Level Local Scope

Every flow control sub-block (`IF{}`, `ELSE{}`, `CASE{}`, `DEFAULT{}`, `EACH{}`, `BLOCK{}`) creates its own local scope:

- Capacity: 128 entries per block.
- Append-only (via `$` prefix).
- Not accessible from sibling or parent blocks.
- Destroyed when the block exits.

### 3.2 IF/ELSE State Machine

At each syntactic level, a single IF-state variable tracks the most recent IF result:

```
State transitions:
  IF(true)  → state = EXECUTED     → ELSE skips
  IF(false) → state = NOT_EXECUTED → ELSE executes → state = EXECUTED
  IF(any)   → overwrites previous IF state at same level
  ELSE      → consumes state, sets to EXECUTED regardless of outcome
```

### 3.3 SWITCH First-Match Semantics

SWITCH pre-computes all comparisons. CASE blocks are evaluated in positional order. The first match executes and terminates the SWITCH. No fall-through occurs.

### 3.4 EACH Bounded Iteration

- Iteration count is bounded by collection size — no infinite loops are possible.
- The data stack is shared (not private to the loop body).
- `BREAK` exits the loop; `CONTINUE` advances to the next iteration.
- `RETURN` is invalid within `EACH{}` (reserved for `MAP{}`/`FILTER{}`).
