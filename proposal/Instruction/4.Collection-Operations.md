# Proposal: Collection Operations [36-45]（提案：集合操作 [36-45]）

> **Prerequisite:** 本规范基于 [Base Constraints](0.Base-Constraints.md) 文档，该文档定义了统一使用的指令格式、运行时模型与记号约定。


## 1. Overview（概览）

集合操作占用 opcode 范围 **[36-45]**（10 条指令）。它们为脚本系统中的两类可迭代集合提供基础操作：**slices**（有序、可索引序列）与 **dicts**（字符串键 map，按插入顺序迭代）。

这些指令作用于位于 data stack 或 argument space 的集合值。由于脚本执行期是 read-only，凡是逻辑上会“修改”集合的指令（如 `SET`、`EXTEND`、`REVERSE`）都必须返回 **new** 集合或在副本上操作，不能改写原始脚本数据。

### Collection Types（集合类型）

| Type | Description | Key Type | Iteration Order |
|------|------------|----------|----------------|
| Slice | 有序的同类值序列 | Integer index | 顺序迭代（0, 1, 2, ...） |
| Dict | 字符串键关联映射 | String key | **Insertion order**（确定性） |

> **Critical design decision:** Dict 迭代必须使用 **insertion-order semantics**（不是 Go 原生 map 的随机顺序）。这是区块链确定性的核心要求，所有节点必须以完全一致的顺序遍历 dict 条目。


## 2. Instruction Definitions（指令定义）

### 2.1 Sub-Slice（子切片）

```
36      SLICE(1)    Create sub-slice
```

- **Auxiliary params:** 1 byte，子切片长度（`[0-255]`），`0` 表示从起始位置取到末尾。
- **Arguments:**
  1. 目标 slice。
  2. 起始索引（0-based），支持负索引（从末尾反向计数）。
- **Return value:** 子切片（对原 slice 的引用视图，不是深拷贝）。
- **Strict mode:** 若位置或长度越界，执行失败。
- **Examples（给定 `[1, 2, 3, 4, 5, 6, 7, 8]`）:**
  - `{2} SLICE[3]` -> `[3, 4, 5]`
  - `{-4} SLICE[2]` -> `[5, 6]`
  - `{-3} SLICE` -> `[6, 7, 8]`（从 -3 位置取到末尾）
  - `{8} SLICE` -> `[]`（起点在尾部，返回空 slice）
- **Note:** 返回的子切片共享原始数据。若需独立深拷贝，请使用 `COPY`（tool 指令）。

### 2.2 Reverse（反转）

```
37      REVERSE     Reverse a slice
```

- **Auxiliary params:** None
- **Arguments:** 目标 slice。
- **Return value:** 成员顺序反转后的新 slice。
- **Example:**
  - 给定 `[1, 2, 3, 4, 5]`：`REVERSE` -> `[5, 4, 3, 2, 1]`

### 2.3 Merge Slices（合并多个 Slice）

```
38      MERGE       Merge multiple slices
```

- **Auxiliary params:** None
- **Arguments:** Variable count，多个 slice。
- **Return value:** 新 slice，按参数顺序拼接各输入 slice 的成员。
- **Example:**
  ```
  {1, 2, 3}
  {10, 20}
  @POP @POP MERGE  // -> [10, 20, 1, 2, 3]
  ```
  注意：`@POP` 按 FIFO 顺序写入 argument space，因此先捕获栈顶 `{10, 20}`，再捕获 `{1, 2, 3}`。

### 2.4 Extend Slice（扩展 Slice）

```
39      EXTEND      Extend a slice with additional members
```

- **Auxiliary params:** None
- **Arguments:** Variable count，第一个参数是目标 slice，后续参数是依次追加的值。
- **Return value:** 追加后得到的新 slice。
- **Example:**
  ```
  {1, 2, 3}
  {10} {20}
  @POPS[3] EXTEND  // -> [1, 2, 3, 10, 20]
  ```
  三个弹出项中位于最底部的第一个参数作为目标 slice，`10` 与 `20` 依次追加。

### 2.5 Pack (Compact Binary Encoding)（紧凑二进制打包）

```
40      PACK        Pack slice into bytes
```

- **Auxiliary params:** None
- **Arguments:** 一个 slice。
- **Return value:** 字节序列（`Bytes`）。
- 将 slice 中每个成员解码为底层二进制表示后紧密拼接（无 padding、无分隔符）。
- 成员可混合类型；每项都会转换为其基础二进制形态并追加。
- **Example:**
  - `POPS[3] PACK`：先弹出栈顶 3 项并打包为 slice，再压紧编码为连续 bytes。

### 2.6 Spread (Expand Slice)（展开 Slice）

```
41      SPREAD      Spread slice members
```

- **Auxiliary params:** None
- **Arguments:** 一个 slice。
- **Return value:** Variable count，slice 中各独立成员。
- 是“slice 打包”的逆操作：将成员逐项取出，并按原顺序写入目标空间。
- **Examples:**
  - `$SPREAD`：将 slice 成员展开到 local scope（每项成为独立条目）。
  - `@SPREAD`：展开到 argument space。
  - `SPREAD`：展开到 data stack（逐项压栈，保持顺序）。
- **Tip:** 连续使用 `@~SPREAD` 可将多个堆叠的 slice 依次展开进 argument space。

### 2.7 Set Dict Entry（设置 Dict 键值）

```
42      SET         Set dict key-value pair(s)
```

- **Auxiliary params:** None
- **Arguments:**
  1. 目标 dict（`map[string]any`）。
  2. Key：单个 string，或 string slice（`[]string`）。
  3. Value：单个值，或 value slice。
- **Return value:** 修改后的 dict（同一引用，不是拷贝；直接作用于 dict 的内部可变状态）。
- **Behavior:**
  - **Single key + single value:** 设置一组键值对。
  - **Key slice + value slice:** 按位置一一对应设置（key[i] -> value[i]）。
  - **Key slice + single value:** 为所有 key 赋同一个值。
- **Design note:** dict 不是脚本系统的主集合类型，主要作为便利结构，可通过转换指令由两个 slice 构造得到。

### 2.8 Get Dict Entry（读取 Dict 键值）

```
43      ITEM        Get dict value(s) by key
```

- **Auxiliary params:** None
- **Arguments:**
  1. 目标 dict。
  2. Key：单个 string，或 string slice。
- **Return value:** 对应 value。若 key 不存在，返回 `nil`（默认零值）。若 key 为 slice，则返回 value slice。
- **Examples:**
  - `"AAA" ITEM`：读取键 `"AAA"` 的值。
  - `{"AA", "BB"} ITEM`：按键 `"AA"`、`"BB"` 读取并返回 value slice。

### 2.9 Get Slice Member by Index（按索引读取 Slice 成员）

```
44      INDEX       Get slice member(s) by index
```

- **Auxiliary params:** None
- **Arguments:**
  1. 目标 slice。
  2. Index：单个整数，或整数 slice。支持负索引。
- **Return value:** 指定位置值。若 index 参数是 slice，则返回 value slice。
- **Examples:**
  - `{1} INDEX`：读取第 2 个成员（0-based）。
  - `{1, 3} INDEX`：读取第 2 与第 4 个成员并返回为 slice。
- **Design note:** 主要用于读取离散位置或单个元素。连续子序列应使用 `SLICE`。

### 2.10 Collection Size（集合大小）

```
45      SIZE        Get collection size
```

- **Auxiliary params:** None
- **Arguments:** 一个集合（slice 或 dict）。
- **Return value:** 整数，表示成员/条目数量。


## 3. Operational Semantics（操作语义）

### 3.1 Mutability Model（可变性模型）

由于脚本是 read-only，看似会“修改集合”的指令遵循如下规则：

| Operation | Creates New Collection? | Notes |
|-----------|----------------------|-------|
| SLICE | No（引用/子切片） | 与原值共享底层数据 |
| REVERSE | Yes | 返回反序新 slice |
| MERGE | Yes | 由多个来源构建新 slice |
| EXTEND | Yes | 返回追加后新 slice |
| PACK | Yes | 返回新字节序列 |
| SPREAD | N/A | 仅拆解，不产生新集合 |
| SET | No（dict 原地） | 直接修改 dict 值 |

`SET` 是例外：它操作 dict 的内部状态。由于 dict 属于运行时创建值（不在 read-only 的脚本 bytecode 中），该原地修改是安全的。

### 3.2 Dict Iteration Order（Dict 迭代顺序）

dict 实现 **must** 使用保持 insertion-order 的数据结构。这是区块链共识要求：同一 dict 在所有验证节点上必须产生一致的迭代序列。Go 原生 `map` **not** 满足该要求，不能直接用于该场景，必须使用有序 map 实现。

### 3.3 Strict Bounds Checking（严格边界检查）

`SLICE`、`INDEX` 以及其他位置访问都执行严格边界检查。越界会导致执行失败，而非静默返回默认值。这与栈操作采用的严格策略保持一致。
