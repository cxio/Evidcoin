# Proposal: Logic Instructions [112–115]

> **Prerequisite:** This specification builds upon the [Base Constraints](0.Base-Constraints.md) document, which defines the instruction format, runtime model, and notation conventions used throughout.


## 1. Overview

Logic instructions occupy opcode range **[112–115]** (4 instructions). They perform boolean logic on boolean values or boolean collections, returning a single boolean result. These instructions bridge the gap between individual comparisons and composite validation conditions.

### Relationship to Other Boolean Operations

- `NOT` (opcode 99, arithmetic) inverts a single boolean — it is categorized under arithmetic unary operations.
- `BOTH` / `EITHER` operate on exactly **two** boolean values (binary logic).
- `EVERY` / `SOME` operate on **boolean collections** (N-ary logic).


## 2. Instruction Definitions

### 2.1 Both (Logical AND)

```
112     BOTH        Both true (&&)
```

- **Arguments:** Two Bool values.
- **Return value:** Bool — `true` if both arguments are `true`.
- **Example:**
  ```
  BOTH PASS    // take top two stack values, pass if both are true
  ```

### 2.2 Either (Logical OR)

```
113     EITHER      Either true (||)
```

- **Arguments:** Two Bool values.
- **Return value:** Bool — `true` if at least one argument is `true`.
- **Example:**
  ```
  EITHER PASS  // take top two stack values, pass if either is true
  ```

### 2.3 Every (Universal Quantifier)

```
114     EVERY       All true (&&)
```

- **Arguments:** A boolean collection (`[]Bool` or `[]any` containing boolean-compatible values).
- **Return value:** Bool — `true` if **all** members are `true`.
- **Empty collection:** Returns `true` (vacuous truth — standard mathematical convention for universal quantification).
- Members can be actual Bool values or `any`-wrapped booleans. If the collection contains non-boolean `any` values, convert first with `ANYS{Bool}`.
- **Examples:**
  ```
  SHIFT[3] ANYS[1] EVERY       // pack top 3 items, convert to []Bool, check all true
  NIL ANYS[1] EVERY             // empty collection → true
  TRUE FALSE SHIFT[2] EVERY     // directly usable without ANYS conversion
  ```

### 2.4 Some (Existential Quantifier with Threshold)

```
115     SOME(1)     At least N true
```

- **Auxiliary params:** 1 byte — minimum count `n` of `true` members required.
- **Arguments:** A boolean collection.
- **Return value:** Bool — `true` if at least `n` members are `true`.
- **Empty collection:** Returns `false` (except when `n = 0`).
- **Special cases:**
  - `SOME[0]` — unconditionally `true` (zero threshold always met).
  - `SOME[1]` — equivalent to existential quantification ("at least one true") — the most common usage.
- **Examples:**
  ```
  SHIFT[3] SOME[2]         // at least 2 of 3 must be true
  ... SOME[1]               // at least 1 must be true (common)
  ... SOME[0]               // always true
  NIL ANYS[1] SOME[1]      // empty collection → false
  ```


## 3. Semantics Summary

| Instruction | Arity | Logic | Empty Collection |
|-------------|-------|-------|-----------------|
| BOTH | Binary (2 values) | AND | N/A |
| EITHER | Binary (2 values) | OR | N/A |
| EVERY | N-ary (collection) | Universal AND | `true` |
| SOME(n) | N-ary (collection) | Threshold OR | `false` (unless n=0) |

### Relationship to PASS/CHECK

Logic instructions are often used immediately before `PASS` or `CHECK` to compose complex validation conditions:

```
// Multi-signature: at least 2 of 3 signatures must be valid
<sig_check_1> <sig_check_2> <sig_check_3>
SHIFT[3] SOME[2] PASS

// Two conditions must both hold
<condition_A> <condition_B>
BOTH PASS

// Either condition suffices
<condition_A> <condition_B>
EITHER PASS
```
