# Stack Operations（栈操作指令）

## 来源构想

- `docs/conception/Instruction/3.栈操作指令.md`：定义 opcode `24-34`。
- `docs/proposal/Instruction/0.Base-Constraints.md`：定义数据栈、返回值和取参模型。

## 范围

栈操作指令直接读写数据栈：

- `NOP`
- `PUSH`
- `POP`
- `POPS`
- `TOP`
- `TOPS`
- `PEEK`
- `PEEKS`
- `SHIFT`
- `CLONE`
- `VIEW`

## 规范要求

- 数据栈为 LIFO。
- 栈顶、栈底和索引方向必须在 Plan 中统一。
- 栈高度受 `MaxStackHeight` 限制。
- 单个栈项受 `MaxStackItem` 限制，复合值大小计算仍未决。
- 数量不足、越界、非法负索引必须失败。
- `0` 附参表示“全部”的指令必须列入白名单，不能通用化。
- 多值返回的指令应声明是否自动展开。
- `CLONE` 默认浅复制；深复制由工具指令处理。

## 风险

- `POPS[0]`、`TOPS[0]`、`SHIFT[0]` 等“全部”语义若不统一，会导致实现差异。
- 浅复制 Slice/Dict 可能产生共享引用副作用。
- `POP` 在无截取时的返回/丢弃语义需明确，避免脚本作者误判。

## 未决问题

- 栈索引是否支持负数以及负数含义。
- 复合值栈项大小计算方式。
- 浅复制对象是否允许在公共验证中被后续修改。
- `VIEW` 的完整可见范围。

## 对后续 Plan 的影响

- 栈操作测试必须覆盖空栈、数量不足、越界、全部操作、浅复制和资源限制。
