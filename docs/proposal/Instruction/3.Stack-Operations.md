# Proposal: Stack Operations [24-34]（提案：栈操作 [24-34]）

> **Prerequisite:** 本规范基于 [Base Constraints](0.Base-Constraints.md) 文档，该文档定义了统一使用的指令格式、运行时模型与记号约定。


## 1. Overview（概览）

栈操作占用 opcode 范围 **[24-34]**（11 条指令）。它们提供对 data stack 的直接操控能力：push、pop、引用、克隆、查看条目。这些指令构成栈机的机械基础，使数据可在栈内重排、复制，并通过捕获前缀在 stack 与其他运行时空间（argument space、local scope）之间传递。

### Design Principle（设计原则）

根据“返回多个值时的行为”，栈操作分为两类：

- **Auto-expanding instructions**（opcode 27-32）：返回多个成员时，会按原顺序逐个放入目标空间（stack、argument space 或 local scope）。
- **Slice-returning instructions**（opcode 33-35）：多个成员会先打包成一个 slice 值，作为目标空间中的单个条目。

该区分对正确管理数据流非常关键。

### Argument Consumption Pattern（参数消费模式）

部分栈操作可从 argument space 获取位置参数（如 `PEEK`、`PEEKS`、`VIEW`）。它们遵循标准取参规则：argument space 优先，data stack 回退。若位置值本身来自 stack，要注意该位置值会先被消费（pop），再访问目标条目。


## 2. Instruction Definitions（指令定义）

### 2.1 No Operation（空操作）

```
24      NOP         No operation
```

- **Auxiliary params:** None
- **Arguments:** Variable count（可为 0）。
- **Return value:** None
- 不执行计算，但会 **reads and discards** argument space 中的全部值，等价于清空 argument space。
- **Examples:**
  - `@SHIFT[] NOP`：先将全部栈项提取到 argument space，再丢弃（等价清空整个栈）。
  - `@SHIFT NOP`：同上；辅助参数为 0 时可省略方括号。
  - `@POP NOP`：弹出并丢弃栈顶项。
- **Design note:** 其可变参数计数包含 0，因此 NOP 永远不需要访问 data stack，它只是 argument space 的纯粹“汇点”（a sink）。

### 2.2 Push to Stack（压入栈）

```
25      PUSH        Push to data stack
```

- **Auxiliary params:** None
- **Arguments:** Variable count。
- **Return value:** None
- 将 argument space 中所有成员按原始 FIFO 顺序转移到 data stack。
- 若 argument space 为空，则为 no-op。
- **Examples:**
  - `$[0] $[1] $[2] PUSH`：将 3 个 local scope 值压回 data stack。
  - `@PEEK @INPUT PUSH`：先捕获栈底项与输入缓冲区数据，再统一压入栈。

### 2.3 Pop Single Item（弹出单项）

```
26      POP         Pop top item
```

- **Auxiliary params:** None
- **Arguments:** None（直接作用于栈顶）。
- **Return value:** 栈顶项。
- 移除并返回最顶层条目。
- 通常应与捕获前缀配合；若不加前缀，值会重新入栈（无实际意义）。
- **Examples:**
  - `@POP`：弹出栈顶到 argument space。
  - `$POP`：弹出栈顶到 local scope。
  - `POP`：无意义（弹出后又压回同一值）。

### 2.4 Pop Multiple Items（弹出多项）

```
27      POPS(1)     Pop multiple top items
```

- **Auxiliary params:** 1 byte，弹出数量（`[0-255]`），`0` 表示全部。
- **Arguments:** None。
- **Return value:** 按原顺序返回多个项（auto-expanding）。
- 从栈顶移除指定数量条目。若数量超过当前栈高，执行 **fails**（strict mode）。
- **Examples:**
  - `@POPS[4]`：弹出栈顶 4 项并展开到 argument space。
  - `$POPS`：弹出全部栈项并展开到 local scope。
  - `POPS[4]`：无意义（弹出后按同序压回）。

### 2.5 Reference Top Item（引用栈顶项）

```
28      TOP         Reference top item
```

- **Auxiliary params:** None
- **Arguments:** None。
- **Return value:** 栈顶项（引用/浅拷贝）。
- **not** 移除栈中原项；可带或不带捕获前缀使用。
- **Examples:**
  - `@TOP`：将栈顶引用放入 argument space（浅拷贝）。
  - `$TOP`：将栈顶引用放入 local scope。
  - `TOP`：复制栈顶项并压入栈。

### 2.6 Reference Multiple Top Items（引用多个栈顶项）

```
29      TOPS(1)     Reference multiple top items
```

- **Auxiliary params:** 1 byte，数量（`[0-255]`），`0` 表示全部。
- **Arguments:** None。
- **Return value:** 按原顺序返回多个项（auto-expanding）。
- 从栈顶引用（浅拷贝）指定数量条目，不移除原值。若数量超出栈高则执行失败。
- **Examples:**
  - `$TOPS[3]`：引用栈顶 3 项并展开到 local scope。
  - `@TOPS[]` 或 `@TOPS`：引用全部栈项到 argument space。
- **Distinction from REP:** `TOPS[n]` 复制的是栈顶 N 个“不同条目”（逐项浅拷贝）；`REP[n]`（算术指令）则是把“单个栈顶值”重复 N 次。

### 2.7 Peek at Arbitrary Position（按任意位置引用单项）

```
30      PEEK        Reference item at arbitrary position
```

- **Auxiliary params:** None
- **Arguments:** Optional，目标位置（默认 `0`，即栈底）。支持负索引（从栈顶反向计数）。
- **Return value:** 指定位置项（引用）。
- 位置从栈底开始计数（`0`）。负值从栈顶计数（`-1` = 栈顶）。
- 若起始位置越界，执行会失败。
- **Examples:**
  - `PEEK`：引用栈底项，并将副本压到栈顶。
  - `@{0} PEEK`：同上，显式给出位置。
  - `@PEEK`：将栈底项引用放入 argument space。
  - `@{-1} PEEK`：引用栈顶项（等价 `TOP`）。

### 2.8 Peek at Arbitrary Position Range（按任意位置引用多项）

```
31      PEEKS(1)    Reference items from arbitrary position
```

- **Auxiliary params:** 1 byte，数量。`0` 表示从起始位置到末尾全部。
- **Arguments:** Optional，起始位置（默认 `0`）。支持负索引。
- **Return value:** 按栈顺序返回多个项（auto-expanding）。
- 若起始位置越界，或请求数量超出现有可用项，执行失败。
- **Examples:**
  - `@PEEKS[5]`：引用栈底 5 项并展开到 argument space。
  - `@{1} @PEEKS[2]`：从位置 1 开始引用 2 项并展开到 argument space。
  - `PEEKS[5]`：引用栈底 5 项并展开压到栈顶（将底部片段克隆到顶部）。

### 2.9 Shift Out as Slice（移出并打包为 Slice）

```
32      SHIFT(1)    Shift out top items (as slice)
```

- **Auxiliary params:** 1 byte，数量（`[0-255]`），`0` 表示全部。
- **Arguments:** None。
- **Return value:** 单个 **slice**，包含了被移除条目。
- 从栈顶移除条目并打包为一个 slice 值。若数量超过栈高则执行失败。
- **Examples:**
  - `@SHIFT[4]`：移除栈顶 4 项，打包为 slice 放入 argument space。
  - `$SHIFT[4]`：同上，写入 local scope（新增 1 条）。
  - `SHIFT[0]` 或 `SHIFT`：移除全部栈项并打包为一个 slice，再压栈。

### 2.10 Clone as Slice（克隆并打包为 Slice）

```
33      CLONE(1)    Clone top items (as slice)
```

- **Auxiliary params:** 1 byte，数量（`[0-255]`），`0` 表示全部。
- **Arguments:** None。
- **Return value:** 单个 **slice**，包含了条目的浅拷贝。
- 不移除栈中原项，只将复制结果打包为一个 slice 值。
- **Examples:**
  - `@CLONE[3]`：将栈顶 3 项克隆为 slice 并写入 argument space。
  - `$CLONE[3]`：将栈顶 3 项克隆到 local scope（1 条）。
  - `CLONE[3]`：克隆栈顶 3 项为 slice 并压入栈。
  - `CLONE`：克隆全部栈项为一个 slice 并压入栈。

### 2.11 View as Slice (Arbitrary Position)（按任意位置查看为 Slice）

```
34      VIEW(1)     View items from arbitrary position (as slice)
```

- **Auxiliary params:** 1 byte，数量。`0` 表示从起始位置到末尾全部。
- **Arguments:** Optional，起始位置（默认 `0`）。支持负索引。
- **Return value:** 单个 **slice**，其中包含对目标条目的引用。
- 不移除条目，只将引用打包为 slice。
- 若位置越界，或数量超出可用项，执行失败。
- **Examples:**
  - `@VIEW[5]`：将栈底 5 项作为 slice 查看并写入 argument space。
  - `VIEW[5]`：将栈底 5 项作为 slice 查看并压入栈。
  - `@{-5} VIEW`：将栈顶 5 项作为 slice 查看（等价 `CLONE[5]`）。

> **注：**
> 脚本是只读逻辑，所以 VIEW 返回 slice 引用是安全的。
> 没有向切片中写入数据的指令。<br>
> 实现：
> 考虑安全冗余，VIEW 返回的 slice 成员可以是一个浅拷贝。


## 3. Behavioral Summary（行为汇总）

### Auto-Expanding vs. Slice-Returning（自动展开 vs Slice 返回）

| Instruction | Removes from Stack | Return Style | Count Param |
|------------|-------------------|-------------|-------------|
| POP | Yes | Single item | — |
| POPS(n) | Yes | Auto-expand | Auxiliary (0=all) |
| TOP | No | Single item | — |
| TOPS(n) | No | Auto-expand | Auxiliary (0=all) |
| PEEK | No | Single item | Argument (position) |
| PEEKS(n) | No | Auto-expand | Auxiliary + Argument |
| SHIFT(n) | Yes | Single slice | Auxiliary (0=all) |
| CLONE(n) | No | Single slice | Auxiliary (0=all) |
| VIEW(n) | No | Single slice | Auxiliary + Argument |

### Strict Mode（严格模式）

所有基于数量的操作都执行严格边界检查。请求数量若超过当前栈内条目，会直接执行失败（不会静默截断），以避免脚本逻辑中的隐蔽错误。

### Position Indexing Convention（位置索引约定）

针对位置类指令（`PEEK`、`PEEKS`、`VIEW`）：
- **Positive values** 从 stack **bottom** 计数（0 = 最早压入项）。
- **Negative values** 从 stack **top** 计数（-1 = 最近压入项）。
