# Stack Operations（栈操作）

Status: Draft proposal（草案）
Source conception: `conception/Instruction/3.栈操作指令.md`

## Prerequisites（前置条件）

有关栈模型、local/global frame、opcode 布局与指令编码约定，请参见
[`0.Base-Constraints.md`](./0.Base-Constraints.md)。
下面简要回顾本文会用到的概念：

- **Data stack** — 主要的 LIFO 区域，容量受 `MaxStackHeight` 限制。
- **Argument zone** — 由 capture 指令填充的临时缓冲区；默认作为下一条指令
  的运行时参数来源。
- **Capture instructions (`@`, `$`)** — 参见
  [`./2.Capture-Instructions.md`](./2.Capture-Instructions.md)；stack
  operations 通常与它们配合使用，以重定向返回结果。
- **Auto-expansion vs. packed return** — 该家族中的部分指令会把多项返回值
  按原顺序逐项展开写入目标位置；另一些则返回一个打包后的单一 slice。
  目录会明确标注每条指令采用哪种行为。

## Overview（概述）

Stack operations 直接操作 data stack：向栈中 push 条目，从栈中 pop 或 peek
条目，复制或搬移条目区间。它们构成了更高层指令家族（collection ops、
arithmetic、comparison 等）的机械基础。

该指令家族中会反复出现两类子模式：

1. **Auto-expanding returns** (`POP`, `POPS`, `TOP`, `TOPS`, `PEEK`,
   `PEEKS`)：当返回多个条目时，会按原始顺序逐个写入目标位置。
2. **Packed returns** (`SHIFT`, `CLONE`, `VIEW`)：多项结果会被封装为
   一个 slice，并将该 slice 作为返回值。

返回值写入目标由可选的前置 capture 决定（`@` -> argument zone，
`$` -> local frame）；没有 capture 时，结果会自动 push 回 data stack。

## Opcode Range（操作码范围）

`[24, 34]` — 11 个 opcode。

## Instruction Catalog（指令目录）

```go
opcode  mnemonic    short description
-----------------------------------------------------------
24      NOP         no-op（清空 argument zone）
        // 参数：可变数量（零个或多个）。
        // 栈效应：( -- )，并额外清空 argument zone。
        // 不执行计算，但会消费（并丢弃）当前完整 argument zone。
        // 源码层写法：
        //   @SHIFT[] NOP    先提取全部栈条目，再清空。
        //   @SHIFT NOP      同上；值为 0 的 immediate 可省略
        //                   （不写方括号）。
        //   @POP NOP        移除栈顶条目。
        // 设计说明：
        // 由于运行时参数数量无上限（且可为 0），NOP 自身
        // 不会触碰 data stack。

25      PUSH        将 argument zone 压入 data stack
        // 参数：可变数量。
        // 栈效应：( -- args... )
        // 将 argument zone 中的每个成员按原顺序压入 data stack。
        // 当 argument zone 为空时，PUSH 是（无意义的）no-op。
        // 源码层写法：
        //   $[0] $[1] $[2] PUSH   将 3 个 local frame 条目移动到
        //                         data stack。
        //   @PEEK @INPUT PUSH     先暂存栈底条目和 input buffer 负载，
        //                         然后 push。

// --- Auto-expanding returns ------------------------------------------------

26      POP         弹出栈顶条目
        // 栈效应：( a -- )      [无 capture]
        //   返回被弹出的值。若无前置 capture，该值会被重新 push 回
        //   data stack，因此没有实际意义。
        // 源码层写法：
        //   @POP    弹出到 argument zone。
        //   $POP    弹出到 local frame。
        //   POP     合法但无意义（会把同一个值再 push 回去）。

27      POPS(1)     弹出多个栈顶条目
        // Immediate：1 字节条目数量（< 256）。0 表示“全部条目”。
        // 栈效应：( a1 a2 ... an -- )
        //   按原顺序返回弹出的序列（无 capture 时会自动展开）。
        // 错误：
        //   - count 超过当前栈高度 -> FAIL（严格模式）。
        // 源码层写法：
        //   @POPS[4]   将 4 个条目弹出到 argument zone。
        //   $POPS      将整个栈弹出到 local frame。
        //   POPS[4]    合法但无意义（条目会被重新 push 回去）。

28      TOP         引用栈顶条目
        // 栈效应：( a -- a a' )    [无 capture，浅拷贝]
        // 读取（不移除）栈顶条目。
        // 源码层写法：
        //   @TOP    将栈顶引用写入 argument zone。
        //   $TOP    将栈顶引用写入 local frame。
        //   TOP     在 data stack 上复制栈顶条目（该引用会自动
        //           push 回栈）。

29      TOPS(1)     引用多个栈顶条目
        // Immediate：1 字节条目数量（< 256）。0 表示“全部条目”。
        // 栈效应：保留原条目；按原顺序返回被引用的条目（无
        //   capture 时自动展开）。
        // 错误：
        //   - count 超过当前栈高度 -> FAIL（严格模式）。
        // 源码层写法：
        //   $TOPS[3]   引用栈顶 3 项，并逐项追加到
        //              local frame。
        //   @TOPS[]    引用全部条目，并展开到 argument zone。
        //   @TOPS      同上；可省略值为 0 的 immediate。
        // 说明：
        // 与 arithmetic 家族中的 REP 对比
        // （见 ./9.Arithmetic-Instructions.md）：
        //   REP[n]   取栈顶 1 项，复制 n 次并展开。
        //   TOP[n]   引用栈顶 n 项（每项 1 份），并展开。

// --- Indexed read-by-position (still auto-expanding) ----------------------

30      PEEK        引用任意栈位置
        // 参数：目标位置（可选，默认 0 = 栈底）。
        // 栈效应：返回被引用条目。
        //   索引从栈底开始计数；负值从栈顶反向计数。默认目标是
        //   data stack（auto-push）。
        // 错误：
        //   - 位置超出当前栈高度 -> FAIL。
        // 源码层写法：
        //   PEEK         引用栈底条目并 auto-push（把栈底克隆到栈顶）。
        //   @{0} PEEK    同上，但显式指定位置 0。
        //   @PEEK        将栈底引用写入 argument zone。
        //   @{-1} PEEK   引用栈顶并 auto-push（= TOP）。

31      PEEKS(1)    引用一段栈区间
        // Immediate：1 字节数量。0 表示“从起始位置直到
        //   栈顶”。
        // 参数：起始位置（可选，默认 0）。
        // 栈效应：按原顺序返回被引用序列（无 capture 时
        //   自动展开）。
        // 错误：
        //   - 起始位置超出栈高度，或请求区间条目不足 -> FAIL。
        // 源码层写法：
        //   @PEEKS[5]        将栈底 5 项引用到 argument zone。
        //   @{1} @PEEKS[2]   从位置 1 开始引用 2 项到
        //                    argument zone。
        //   PEEKS[5]         引用栈底 5 项并 auto-push（把这 5 项
        //                    克隆到栈顶）。

// --- Packed returns (single slice; NO auto-expansion) ---------------------

32      SHIFT(1)    将栈顶条目移出栈
        // Immediate：1 字节条目数量（< 256）。0 表示“全部条目”。
        // 栈效应：移除栈顶 `n` 个条目。
        // 返回：一个单一 slice，按顺序打包被移除条目。
        // 错误：
        //   - count 超过当前栈高度 -> FAIL（严格模式）。
        // 源码层写法：
        //   @SHIFT[4]    移出栈顶 4 项，并打包到 argument zone。
        //   $SHIFT[4]    同上，但把该 slice 追加到 local frame。
        //   SHIFT[0]     移出整个栈，并将结果 slice 作为单一值
        //                push 回去。
        //   SHIFT        等同 SHIFT[0]。

33      CLONE(1)    克隆栈顶条目（浅拷贝）
        // Immediate：1 字节条目数量（< 256）。0 表示“全部条目”。
        // 栈效应：保留原条目；返回一个 slice，按顺序打包
        //   栈顶条目的浅拷贝。
        // 源码层写法：
        //   @CLONE[3]   对栈顶 3 项做浅克隆，并作为单一 slice
        //               写入 argument zone。
        //   $CLONE[3]   同上，写入 local frame。
        //   CLONE[3]    将该 slice push 回 data stack。
        //   CLONE       将整个栈浅克隆为一个 slice。

// --- Indexed packed return ------------------------------------------------

34      VIEW(1)     将栈区间作为 slice 引用
        // Immediate：1 字节数量。0 表示“从起始位置直到
        //   栈顶”。
        // 参数：起始位置（可选，默认 0）。
        // 栈效应：返回一个 slice，按顺序打包被引用条目
        //   （不自动展开）。
        // 错误：
        //   - 起始位置超出栈高度，或请求区间条目不足 -> FAIL。
        // 源码层写法：
        //   @VIEW[5]    将栈底 5 项作为 slice 引用到 argument
        //               zone。
        //   VIEW[5]     同上，但将该 slice push 到 data stack。
        //   @{-5} VIEW  将栈顶 5 项作为 slice 引用并 push
        //               （等价于 CLONE[5] 后 auto-push）。
```

## Design Notes（设计说明）

- **Default destination is the data stack.** 没有前置 capture 时，本家族每条
  指令都会把返回值写回 data stack。许多这类“无 capture”写法在语法上合法，
  但语义上无意义（如不带 `@` 或 `$` 的 `POP`），目录已显式标注。
- **Auto-expanding vs. packed returns.** expanding
  (`POP`/`POPS`/`TOP`/`TOPS`/`PEEK`/`PEEKS`) 与 packed
  (`SHIFT`/`CLONE`/`VIEW`) 的区分是刻意设计：expanding 形式便于下一条
  指令逐项消费；packed 形式便于下游将结果作为单一复合值处理。
- **Strict counts.** 所有带 `count` immediate 的指令，只要 count 超过当前
  栈高度即失败。这一严格不变量是有意设计，用于简化静态分析并避免静默截断。
- **Position semantics.** `PEEK`、`PEEKS`、`VIEW` 的索引均从栈
  **bottom** 开始；负值从栈顶计数。该约定与
  `./2.Capture-Instructions.md` 中 `$[n]` 的 local frame 索引规则一致。
- **`NOP` as an explicit drain.** 除了直观的“do nothing”用途外，`NOP`
  还是在 capture 预先准备好 argument zone 后，将其显式丢弃的标准原语。
- **`TOP[n]` vs. `REP[n]`.** 二者都会在栈上生成 `n` 个条目，但 `TOP[n]`
  引用的是 `n` 个不同的栈顶条目（每项 1 份），而 `REP[n]`（来自 arithmetic
  家族）是把单一值复制 `n` 次。参见
  [`./9.Arithmetic-Instructions.md`](./9.Arithmetic-Instructions.md)。
