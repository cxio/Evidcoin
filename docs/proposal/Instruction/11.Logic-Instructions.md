# Logic Instructions（逻辑指令）

## 来源构想

- `docs/conception/Instruction/11.逻辑指令.md`：定义 opcode `112-115`。
- `docs/proposal/Instruction/0.Base-Constraints.md`：定义取参和返回规则。

## 范围

逻辑指令包括：

- `BOTH`
- `EITHER`
- `EVERY`
- `SOME`

## 规范要求

- 输入应为 Bool 或 Bool 集合；是否自动调用 `BOOL` 转换仍未决。
- `BOTH` 表示两个条件均为 true。
- `EITHER` 表示至少一个条件为 true。
- `EVERY` 对集合执行全真判断；空集结果需固定，构想倾向 true。
- `SOME[n]` 表示至少 n 个 true；`SOME[0]` 构想为无条件 true，必须显式警告。
- 输入集合的顺序不影响结果，但集合来源必须确定性。

## 风险

- 空集逻辑可能被误用绕过验证。
- `SOME[0]` 是危险的无条件通过构造。
- 自动 Bool 转换会降低脚本可审计性。

## 未决问题

- 是否允许非 Bool 自动转换。
- `EVERY` 空集是否为 true。
- `SOME` 空集和 `SOME[0]` 的最终政策。
- `[]any` 中非 Bool 成员的处理。

## 对后续 Plan 的影响

- 逻辑测试必须覆盖空集、阈值 0、非 Bool 输入和集合成员混合。
