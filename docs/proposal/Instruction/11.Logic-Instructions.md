# Logic Instructions（逻辑指令）

Status: Draft proposal（提案草案）
Source conception: `conception/Instruction/11.逻辑指令.md`

## Prerequisites（前置条件）

关于栈模型、本地帧、opcode 布局、操作数来源与运行时参数解析，见 `0.Base-Constraints.md`。

下文会用到的概念回顾：

- 逻辑指令消费 `Bool` 运行时参数（或由 `Bool` 值组成的集合），并返回单个 `Bool`。
- 对于集合形态指令（`EVERY`、`SOME`），成员必须已经是 `Bool`（或 `any` 封装的 `Bool`）；否则需先通过 `ANYS{Bool}` 转换（见 `8.Conversion-Instructions.md`）。

## Overview（概述）

逻辑指令族覆盖两类布尔组合形式：

- **Pairwise**（`BOTH`、`EITHER`）— 两个 `Bool` 参数。
- **Set-wise**（`EVERY`、`SOME(n)`）— 一个由 `Bool` 参数构成的集合。

空集合语义遵循常规代数恒等式：空合取为 `true`，空析取为 `false`。`SOME[0]` 在定义上恒为 `true`。

## Opcode range（opcode 范围）

范围：`[112 – 115]`，共 **4** 个 opcode。

| Sub-group     | Range       | Count |
| ------------- | ----------- | ----- |
| Pairwise      | `112 – 113` | 2     |
| Set-wise      | `114 – 115` | 2     |

## Instruction Catalog（指令目录）

### `BOTH` — Logical AND（逻辑与，opcode 112）

```go
112     BOTH        // a && b
```

- **Type**: pairwise logical conjunction.
- **Operands**: two `Bool` runtime arguments.
- **Stack effect**: `[ … a b ] → [ … (a && b) ]`.
- **Description**: 当且仅当两个参数都为 `true` 时返回 `true`。
- **Errors**: 类型不匹配（出现非 `Bool` 操作数）。

```go
BOTH PASS       // 从栈中弹出两个 Bool；若二者都为 true 则通过
```

### `EITHER` — Logical OR（逻辑或，opcode 113）

```go
113     EITHER      // a || b
```

- **Operands**: two `Bool` runtime arguments.
- **Stack effect**: `[ … a b ] → [ … (a || b) ]`.
- **Description**: 当至少一个参数为 `true` 时返回 `true`。
- **Errors**: 类型不匹配。

```go
EITHER PASS     // 若栈顶两个 Bool 中任一为 true 则通过
```

### `EVERY` — All members true（全体为真，opcode 114）

```go
114     EVERY       // (&&) over a Bool collection
```

- **Operand**: one collection whose members are `Bool` (or `any`-wrapped
  `Bool`).
- **Stack effect**: `[ … coll ] → [ … bool ]`.
- **Description**:
  - 当且仅当集合中的每个成员都为 `true` 时返回 `true`。
  - **Empty collection ⇒ `true`**（真空真）。
- **Errors**: 存在成员既不是 `Bool`，也不是 `any` 封装的 `Bool`。

```go
SHIFT[3] ANYS[1] EVERY            // 打包栈顶 3 项，进行类型转换后再测试
NIL ANYS[1] EVERY                 // 空集合 -> true
TRUE FALSE SHIFT[2] EVERY         // 直接测试，无需转换
```

### `SOME(1)` — At least n members true（至少 n 个成员为真，opcode 115）

```go
115     SOME(1)     // (||) / partial (&&) over a Bool collection
```

- **Immediate**: `n` — minimum number of `true` members required.
- **Operand**: one collection of `Bool`s (same member-type rules as
  `EVERY`).
- **Stack effect**: `[ … coll ] → [ … bool ]`.
- **Description**:
  - 当且仅当至少有 `n` 个成员为 `true` 时返回 `true`。
  - `SOME[0]` 恒为 `true`（无条件要求）。
  - `SOME[1]` 即常见的“任一为真”析取形式，是最常用写法。
  - 对空集合而言，只有 `SOME[0]` 返回 `true`；所有更高的 `n` 都返回 `false`。
- **Errors**: 与 `EVERY` 相同。

```go
SHIFT[3] SOME[2]        // 栈顶 3 项中至少 2 项为 true
... SOME[1]             // 任一为真（常用形式）
... SOME[0]             // 恒为 true
NIL ANYS[1] SOME[1]     // 空集合 -> false
```

## Design notes（设计说明）

- **Two shapes (pairwise + set-wise)** 在脚本字节经济性与表达能力之间取得平衡。Pairwise opcode 可避免在常见双参场景中构建中间集合；Set-wise opcode 则可扩展到 multi-signature、k-of-n quorum 等批量校验场景。
- **Empty-set semantics** 遵循标准代数恒等式，使程序化生成的脚本（例如由中间件拼装条件列表）在输入列表为空时也能平稳退化。
- **Type discipline at the member level** 被有意设为严格模式——拒绝静默类型强转（例如将 `Int` 直接按 truthiness 解释）。需要显式转换时请使用 `ANYS{Bool}`。
- **`SOME` immediate uses 1 byte**，因此 `n` 的上限为 255。这对 k-of-n multi-signature 方案已完全充足。
