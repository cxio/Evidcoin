# Proposal: Logic Instructions [112–115]（提案：逻辑指令）

> **Prerequisite:** 本规范基于 [Base Constraints](0.Base-Constraints.md) 文档，后者定义了贯穿全文的指令格式、运行时模型与符号约定。


## 1. Overview（概述）

逻辑指令占用 opcode 范围 **[112–115]**（共 4 条）。它们对布尔值或布尔集合执行逻辑运算，并返回单一布尔结果。这类指令负责把“单点比较结果”组合成“复合校验条件”。

### Relationship to Other Boolean Operations（与其他布尔操作的关系）

- `NOT`（opcode 99，归类为 arithmetic）对单个布尔值取反，属于算术一元运算。
- `BOTH` / `EITHER` 只处理 **two** 个布尔值（二元逻辑）。
- `EVERY` / `SOME` 面向 **boolean collections**（N 元逻辑）。


## 2. Instruction Definitions（指令定义）

### 2.1 Both (Logical AND)（与运算）

```
112     BOTH        Both true (&&)
```

- **Arguments:** 两个 Bool 值。
- **Return value:** Bool —— 仅当两个参数都为 `true` 时返回 `true`。
- **Example:**
  ```
  BOTH PASS    // 取栈顶两个值，若二者都为 true 则通过
  ```

### 2.2 Either (Logical OR)（或运算）

```
113     EITHER      Either true (||)
```

- **Arguments:** 两个 Bool 值。
- **Return value:** Bool —— 只要至少一个参数为 `true` 即返回 `true`。
- **Example:**
  ```
  EITHER PASS  // 取栈顶两个值，任一为 true 即通过
  ```

### 2.3 Every (Universal Quantifier)（全称量化）

```
114     EVERY       All true (&&)
```

- **Arguments:** 一个布尔集合（`[]Bool` 或包含布尔兼容值的 `[]any`）。
- **Return value:** Bool —— 当成员 **all** 为 `true` 时返回 `true`。
- **Empty collection:** 返回 `true`（空真，符合全称量化的标准数学约定）。
- 成员既可为原生 Bool，也可为 `any` 包装的布尔值。若集合包含非布尔 `any` 值，先用 `ANYS{Bool}` 转换。
- **Examples:**
  ```
  SHIFT[3] ANYS[1] EVERY        // 打包栈顶 3 项，转为 []Bool，检查是否全为 true
  SHIFT[3] ANYS{Bool} EVERY     // 同上，使用友好别名 {Bool}
  NIL ANYS[1] EVERY             // 空集合 → true
  TRUE FALSE SHIFT[2] EVERY     // 不经 ANYS 也可直接使用
  ```

### 2.4 Some (Existential Quantifier with Threshold)（带阈值的存在量化）

```
115     SOME(1)     At least N true
```

- **Auxiliary params:** 1 byte —— 所需最小 `true` 成员数 `n`。
- **Arguments:** 一个布尔集合。
- **Return value:** Bool —— 当 `true` 成员数不少于 `n` 时返回 `true`。
- **Empty collection:** 返回 `false`（`n = 0` 时除外）。
- **Special cases:**
  - `SOME[0]` —— 无条件为 `true`（阈值为 0 永远满足）。
  - `SOME[1]` —— 等价于存在量化（“至少一个为 true”），也是最常用形式。
- 集合成员要求同 `EVERY` 指令。
- **Examples:**
  ```
  SHIFT[3] SOME[2]         // 3 个里至少 2 个为 true
  ... SOME[1]              // 至少 1 个为 true（常见）
  ... SOME[0]              // 永远为 true
  NIL ANYS[1] SOME[1]      // 空集合 → false
  ```


## 3. Semantics Summary（语义汇总）

| Instruction | Arity | Logic | Empty Collection |
|-------------|-------|-------|-----------------|
| BOTH | Binary (2 values) | AND | N/A |
| EITHER | Binary (2 values) | OR | N/A |
| EVERY | N-ary (collection) | Universal AND | `true` |
| SOME(n) | N-ary (collection) | Threshold OR | `false` (unless n=0) |

### Relationship to PASS/CHECK（与 PASS/CHECK 的关系）

逻辑指令常常紧挨 `PASS` 或 `CHECK` 使用，用来拼装复杂校验条件：

```
// Multi-signature: 3 个签名里至少 2 个有效
<sig_check_1> <sig_check_2> <sig_check_3>
SHIFT[3] SOME[2] PASS

// 两个条件必须同时成立
<condition_A> <condition_B>
BOTH PASS

// 两个条件满足其一即可
<condition_A> <condition_B>
EITHER PASS
```
