# System Instructions（系统指令）

## 来源构想

- `docs/conception/Instruction/15.系统指令.md`：定义 opcode `164-169`。
- `docs/proposal/Instruction/0.Base-Constraints.md`：定义公共/私有边界。
- `docs/proposal/14.Incentives-And-Coinbase-Rewards.md`：定义兑奖规则。
- `docs/proposal/06.Transaction-Model.md`：定义签名消息和输入输出。

## 范围

系统指令包括：

- `SYS_TIME`
- `SYS_AWARD`
- `SYS_CHKPASS`
- opcode `167-168` 保留。
- `SYS_NULL`

## 规范要求

- `SYS_TIME` 返回客户端真实当前时间，只能在 `END` 或隐式 `INPUT` 后的私有路径使用。
- `SYS_TIME` 在公共验证前出现必须失败。
- `SYS_AWARD` 仅可用于 Coinbase 输出脚本，检查兑奖比例、高度、确认数量和可兑奖金额。
- `SYS_CHKPASS` 启动系统内置验证，支持 Coin/Credit、单签/多签。
- `SYS_CHKPASS` 的签名数据来自环境而非普通数据栈，这是特例。
- `SYS_CHKPASS` 成功后记录签名状态，供 `SIGNED` 查询。
- `SYS_NULL` 设置 SOURCE 提取零点，可用于解锁脚本，是解锁脚本 opcode 限制例外。

## 风险

- `SYS_CHKPASS` 横跨交易、签名、状态和脚本，必须接口化。
- `SYS_TIME` 误入公共验证会导致分叉。
- `SYS_AWARD` 依赖 Coinbase 和经济规则，不能在脚本层自行计算全部经济逻辑。
- `SYS_NULL` 是解析器特殊例外，必须在解锁脚本白名单中单独处理。

## 未决问题

- `SYS_AWARD` 的附参比例和高度编码。
- `SYS_CHKPASS` 验证类别编号是否与输出配置低 4 bits 完全一致。
- 签名数据环境结构和剪枝格式。
- `SYS_TIME` 公共路径出现时静态拒绝还是运行时失败。
- `SYS_NULL` 对 SOURCE 多次设置时的行为。

## 对后续 Plan 的影响

- 系统指令实现必须通过接口调用交易、状态、签名和激励模块。
- 测试必须覆盖公共路径 `SYS_TIME` 拒绝、Coinbase 外 `SYS_AWARD` 拒绝、单签/多签 `SYS_CHKPASS` 和 `SIGNED` 状态。
