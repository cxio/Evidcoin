# System Instructions（系统指令）

Status: Draft proposal（状态：提案草案）
Source conception: `conception/Instruction/15.系统指令.md`（来源构想）

## Prerequisites（前置条件）

关于栈模型、本地帧、操作码布局、立即数记法 `(n)` 与关联数据记法 `{}`，参见 `0.Base-Constraints.md`。

相关概念回顾：

- **pass status** 是脚本的判定信号；`SYS_CHKPASS` 等系统操作码会参与其设置。
- 脚本的 **explicit `END`**（或在 `INPUT` 处的隐式结束）标记了可安全读取非确定性信息的边界点。
- **Coinbase outputs** 属于特殊用途的脚本上下文，校验规则更严格。

## Overview（概述）

系统指令是一组规模较小、具备特权的系统级操作。该分组中的助记符统一使用 `SYS_` 前缀，以区别于普通操作码。此分组包含：墙钟时间访问器（`SYS_TIME`，仅允许在 `END`/`INPUT` 之后执行）、coinbase 奖励校验器（`SYS_AWARD`，仅限 coinbase output 使用）、内建交易验证器（`SYS_CHKPASS`，用于 coin/credit 花费的规范 pass 授予点），以及源码提取零标记器（`SYS_NULL`）。

## Instruction Catalog（指令目录）

```go
Code   Mnemonic       Form
-----------------------------------------------------------
164    SYS_TIME(1){}           Wall-clock accessor (post-END only)
165    SYS_AWARD(1,~)          Coinbase award validator
166    SYS_CHKPASS             Built-in transaction verification
167-168                        (unused)
169    SYS_NULL                Source-extraction zero marker
```

### 164 — `SYS_TIME(1){}`

- **Type**: system，环境查询（non-deterministic）
- **Immediate**: 1 byte 字段选择器；花括号形式可按名称接受同一组取值（作为立即数别名）
  - `0`（默认）— 当前时间对象（`Time`）
  - `Stamp` — Unix 毫秒时间戳
  - `Year` — 四位年份
  - `Month` — 年内月份（1–12）
  - `YearDay` — 年内天数（1–365 / 366）
  - `Day` — 月内日期（1–31）
  - `WeekDay` — 周内日序（0–6）
  - `Hour` — 小时（0–23）
  - `Minute` — 分钟（0–59）
  - `Second` — 秒（0–59）
  - `Millisecond` — 秒内毫秒（0–999）
  - `Microsecond` — 秒内微秒（0–999999）
- **Operands**: 无
- **Stack effect**: `-> Time | Int`
- **Description**: 读取宿主机当前墙钟时间。所有数值字段均以 `Int` 返回。由于该值在不同节点间天然具有非确定性，**仅允许在脚本的 explicit `END` 或其在 `INPUT` 处的 implicit `END` 之后执行**。墙钟时间适用于私有（post-consensus）脚本区域，例如客户端展示或本地逻辑。
- **Errors**: 在 `END` / `INPUT` 之前调用——脚本以硬错误失败；字段选择器未知。

### 165 — `SYS_AWARD(1,~)`

- **Type**: system，仅限 coinbase 的校验器
- **Immediates**:
  1. 1 byte 奖励比例——整数百分比（如 `10` ⇒ 10%）
  2. varint —— 区块高度
- **Operands**: 无
- **Stack effect**: `-> Int`（计算后的可兑奖金额，已扣除可能的预留）
- **Description**: 验证奖励兑付是否符合协议奖励规则，并计算可兑付金额。内部检查包括区块高度确认深度与比例组合。
  校验规则：
  - **Permitted only inside the output script of a coinbase transaction**；在其他位置使用属于硬错误。
  - 提供的区块高度必须等于其所在 coinbase 交易的区块高度。
  - 操作码会检查给定区块高度之后的链段；基于该高度可推导链段终止高度，并与协议 `+29 .. +48` 确认窗口比对，从而快速判断资格。
  - 所有 coinbase outputs 的奖励比例集合之和必须为 100（即 100%）；每个比例必须符合协议规则手册。
- **Errors**: 非 coinbase output 调用；比例规则违规；区块高度不匹配；确认数不足。

### 166 — `SYS_CHKPASS`

- **Type**: system，内建交易验证器（pass-granting）
- **Operands**: 不从数据栈取操作数——见下文“隐私加固”
- **Stack effect**: `->`（不压栈数据；成功时内部设置 pass status，失败时脚本退出）
- **Description**: 执行规范化的内建交易验证，覆盖单签与多签花费。与 `PASS` 一样，它本身就是 **pass-granting** 操作码：失败时脚本直接退出且不设置 pass status。验证按 **verification class**（output 配置字节的低半字节）分派：
  - `0` — **coin**（必须提取 coin 金额）
  - `1` — **credit/proof**（必须校验转移逻辑，包括使用次数、有效期、可变更性规则）
  接收方地址必须采用规范（标准）格式。
  底层验证器参考签名：
  ```go
  // 概念签名；运行时参数来自执行环境，
  // 而非数据栈（见 Privacy-hardening）。
  SYS_CHKPASS(
      chk_type   byte,    // 0 coin，1 credit
      sig_type   byte,    // 1 单签，2 多签
      auth_flag  byte,    // 签名消息约束
      sigs       [][]byte,
      pubKeys    [][]byte,
      pubHashes  [][]byte, // 仅多签使用
  )
  ```
- **Errors**: 签名验证失败（脚本退出且不授予 pass）；verification class 非法；签名类型不匹配。

**Privacy-hardening.** 从 `sig_type` 开始，验证器运行时参数改为从 **execution environment** 提取，而不是来自数据栈。这是有意设计的特殊处理：可将 unlock script 的签名数据从脚本序列化形式中“抬出”，使签名无需参与 transaction-ID 计算，并可在确认后剥离。此后交易完整性由链自身的链式哈希结构保证，而非脚本内嵌签名。

**Placement freedom.** 因为 `SYS_CHKPASS` 是独立操作码，作者可将内建验证放在脚本中任意合适位置——包括通过 `GOTO`、`EMBED` 或 `SCRIPT` 引入的脚本内部。不存在固定位置要求。

**Implementation note.** 成功时，引擎会将验证结果（`true` / `false`）记录到系统层标志位，供后续操作码（如 `SIGNED`）查询。

### 167–168

未使用。保留给未来系统操作码，在当前系统活动范围内预留。

### 169 — `SYS_NULL`

- **Type**: system，源码位置标记器
- **Operands**: 无
- **Stack effect**: 无
- **Description**: 在当前脚本源码位置植入 `NULL` 标记，用于标记后续源码片段提取的 **start point**。主要由 `SOURCE` 消费。示例形态：
  ```go
  SYS_NULL                 // 锚定起始位置
  "Hai"                    // 压入字符串 "Hai"
  ...                      // 任意后续操作码序列
  SOURCE[2]                // 从 "Hai" 开始提取源码片段
                           // 直到并包含当前位置
  ```
  该指令用于 unlock script 片段，以提供灵活的 “unlock ↔ lock” 耦合。这是在整体只读源码提取模型中的有意特殊处理。
- **Errors**: 无。

## Design Notes（设计说明）

- **Determinism boundaries.** `SYS_TIME` 是 non-deterministic 值的典型示例；将其限制在 `END`/`INPUT` 边界之后，可在保持共识一致性的同时，为私有/post-consensus 场景提供墙钟时间数据访问。
- **Coinbase scoping for `SYS_AWARD`.** 将该操作码限制在 coinbase outputs 内，可让引擎执行深层结构校验（比例组合、高度窗口、链段确认）；这些校验在该上下文之外既不安全也无意义。
- **Signature lifting via `SYS_CHKPASS`.** 将签名材料从数据栈路径转移到环境层可实现两点：1）签名不进入 transaction-ID 预映像，支持后续移除；2）签名不保留在序列化脚本中，提升隐私性。
- **`SYS_NULL` as anchor.** 通过显式锚点操作码处理脚本源码提取，可在保持源码只读的同时，提供哈希耦合脚本所需的 unlock 侧灵活性。
