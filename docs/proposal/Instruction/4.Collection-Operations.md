# Collection Operations（集合操作）

Status: 提案草案
Source conception: `conception/Instruction/4.集合指令.md`

## Prerequisites（前置条件）

关于栈模型、argument zone、local frame 与操作码布局，请参见 `0.Base-Constraints.md`。简要说明如下：

- **data stack** 用于保存执行过程中产生的值；**argument zone** 用于暂存将喂给下一条指令的运行时参数；**local frame** 是按代码块划分、静态且不继承的作用域；**global frame** 是脚本级共享命名空间。
- 每条指令编码为 `opcode (1 byte) + immediate operand(s) + associated data`。其中只有 opcode 是必需的。
- 脚本源码在运行时是只读的。本指令族中的任何指令都不会修改底层脚本字节；变更仅针对运行时对象。

## Overview（概述）

该指令族为 VM 的两类可迭代复合类型提供标准操作：**slices**（有序序列）与 **dicts**（按插入顺序维护的 `map[string]any`）。覆盖能力包括结构重塑（sub-slicing、reversing、merging、extending、packing、spreading）、成员访问（按位置和按键）、按键赋值、对象/模块上的方法分发，以及大小查询。它有意不提供任何“原地修改 slice 成员”的指令，因为 slice 引用可能触达脚本常量区，从而破坏只读不变量。

## Opcode range（操作码范围）

`[35 – 45]`，共 11 条指令。

## Instruction Catalog（指令目录）

### 35 — `SLICE(1)` — sub-slice（子切片）

| Field | Value |
|---|---|
| Type | Structural（结构） |
| Immediate | 子切片长度（`< 256`）；`0` 表示“从起始索引到末尾” |
| Runtime args | `arg1` 目标 slice；`arg2` 起始索引（0-based；负值从尾部计数） |
| Stack effect | `( target start -- subslice )` |
| Returns | 子切片（源 slice 上的 *reference view*，不是拷贝） |

**Description.** 生成与源数据共享同一底层存储的子切片。若需要独立副本，请使用 `COPY`。

```go
// 给定目标 [1, 2, 3, 4, 5, 6, 7, 8]：
{2}  SLICE[3]   // [3, 4, 5]
{-4} SLICE[2]   // [5, 6]
{-3} SLICE      // [6, 7, 8]
{8}  SLICE      // [] 空切片
```

**Errors / constraints.** 严格模式下，起始位置或长度任何越界都会导致脚本失败。

### 36 — `REVERSE` — reverse a slice（反转切片）

| Field | Value |
|---|---|
| Type | Structural（结构） |
| Runtime args | `arg1` 目标 slice |
| Stack effect | `( slice -- reversed )` |
| Returns | 一个新 slice，成员顺序反转 |

```go
{1, 2, 3, 4, 5}
REVERSE   // [5, 4, 3, 2, 1]
```

---

### 37 — `MERGE` — merge slices（合并切片）

| Field | Value |
|---|---|
| Type | Structural（结构） |
| Runtime args | variadic —— argument zone 中的所有项都作为输入 slice |
| Stack effect | `( s1 s2 ... sN -- merged )` |
| Returns | 一个新的单一 slice，按从 argument zone 取出的顺序拼接各输入 |

```go
{1, 2, 3}
{10, 20}
@POP @POP MERGE   // [10, 20, 1, 2, 3]
```

---

### 38 — `EXTEND` — extend a slice（扩展切片）

| Field | Value |
|---|---|
| Type | Structural（结构） |
| Runtime args | variadic —— 第一个是目标 slice，其余是要追加的成员 |
| Stack effect | `( target m1 m2 ... mN -- extended )` |
| Returns | 一个新 slice |

```go
{1, 2, 3}
{10}
{20}
@POPS[3] EXTEND   // [1, 2, 3, 10, 20]
```

---

### 39 — `PACK` — pack members into a contiguous byte sequence（打包成员为连续字节序列）

| Field | Value |
|---|---|
| Type | Encoding（编码） |
| Runtime args | 单个 slice |
| Stack effect | `( slice -- bytes )` |
| Returns | 一个新的字节序列（`Bytes`）：将每个成员解码为其底层二进制后再拼接而成 |

```go
SHIFT[3] PACK   // 将栈顶三个元素打包为一个 Bytes 值
```

**Note.** 当成员类型是 `Any` 时，会先将每个成员解码为其底层二进制表示，再执行拼接。

---

### 40 — `SPREAD` — spread a slice（展开切片）

| Field | Value |
|---|---|
| Type | Structural（结构） |
| Runtime args | 单个 slice |
| Stack effect | 可变 —— 向目标区域产出 0..N 个值 |
| Returns | 可变数量的成员 |

通过前导 capture 指令选择目标区域：

```go
$SPREAD   // 将成员展开到 local frame
@SPREAD   // 将成员展开到 argument zone
SPREAD    // 将栈顶项按原顺序展开回 data stack
```

连续使用 `@~SPREAD` 链，可将栈中的多个 slice 压平并写入 argument zone。

---

### 41 — `INDEX` — slice member(s) by position（按位置取 slice 成员）

| Field | Value |
|---|---|
| Type | Access（访问） |
| Runtime args | `arg1` 目标 slice；`arg2` 索引或索引集合 |
| Stack effect | `( slice idx -- value-or-values )` |

负索引从尾部计数。若索引参数本身是集合，则结果也是集合。

```go
{1}    INDEX   // 第 2 个成员
{1, 3} INDEX   // 第 2 和第 4 个成员
```

**Design.** 该指令面向单值或非连续位置选择。这里刻意不提供对 slice 成员赋值的 opcode，因为 slice 成员可能由脚本驻留数据支撑。

---

### 42 — `ITEM` — keyed lookup on dict / object / module（在 dict / object / module 上按键查找）

| Field | Value |
|---|---|
| Type | Access（访问） |
| Runtime args | `arg1` 目标集合；`arg2` 键或键集合 |
| Stack effect | `( target key -- value-or-values )` |

缺失键会解析为零值（`nil`）。键集合会得到值集合。

```go
"AAA"        ITEM   // "AAA" 的值
{"AA", "BB"} ITEM   // "AA" 和 "BB" 的值
```

---

### 43 — `SET` — assign key/value pairs into a dict（向 dict 写入键值对）

| Field | Value |
|---|---|
| Type | Mutation（变更，仅限 runtime dict） |
| Runtime args | `arg1` 目标 dict；`arg2` 键（string）或键集合（`[]string`）；`arg3` 值或值集合 |
| Stack effect | `( dict key value -- dict )` |
| Returns | 同一个目标 dict |

若 key 和 value 都是集合，则长度必须一致（按位置配对）；否则会把单个 value 赋给每个 key。

**Safety.** dict 是运行时对象，其内容来自运行时参数。修改它不会改变任何脚本驻留值，因此可保持脚本源码只读不变量。

---

### 44 — `CALL(1)` — invoke a method on an object or module（调用 object 或 module 的方法）

| Field | Value |
|---|---|
| Type | Dispatch（分发） |
| Immediate | 需要传递的方法参数个数 |
| Runtime args | `arg1` 目标集合（object/module）；`arg2` 方法名（区分大小写）；`arg3..` 方法参数 |
| Stack effect | `( target name a1 .. aK -- result )` |
| Returns | 方法返回值；多返回值会被封装为单个 slice |

variadic 方法定义不是问题：immediate operand 只是在告诉解释器要弹出多少参数，这是调用侧决策，而非定义侧约束。

```go
DATE{2026-03-21T19:08:50Z}
TOP "Day"            // TOP 会把 date 保留在栈上
@CALL                // => 21
TOP "Clock"
@CALL                // => [19, 8, 50]
"Format" "15_04_05"  // 弹出 date 作为 receiver
@CALL[1]             // => "19_08_50"
PUSH                 // [21, [19, 8, 50], "19_08_50"]
```

```go
MO_MATH
TOP "Abs" {-10}
@CALL[1]             // => 10.0
"Pow" {2} {8}
@CALL[2]             // => 256.0
PUSH
SHIFT[2]
ANYS{Int}            // => [10, 256]
```

```go
MO_FMT
"Sprint"
DATE{1606227845000}
CALL[1]              // fmt.Sprint(date)
```

**Design.** 单一 dispatch opcode 将 VM 与方法表解耦：module 无需预声明 arity，且每个 module 的方法数量不受上限约束。

---

### 45 — `SIZE` — collection size（集合大小）

| Field | Value |
|---|---|
| Type | Query（查询） |
| Runtime args | 单个集合（slice 或 dict） |
| Stack effect | `( collection -- n )` |
| Returns | 整数 |

## Design notes（设计说明）

- **No in-place slice assignment.** slice 成员可能引用脚本驻留字面量；允许元素修改将使脚本能够伪造对自身源码的写入。需要按键变更时请使用仅运行时的 dict。
- **Reference vs. copy semantics.** `SLICE` 返回视图；`COPY`（参见 Stack Operations）是显式深拷贝原语。
- **`SPREAD` destination polymorphism.** 目标区域（data stack、argument zone、local frame）由前导 capture 指令（`$`、`@` 或无）选择；指令目录与 capture 语法保持正交。
- **`PACK` is the inverse of multi-push.** 它是把任意栈项压平为单个 `Bytes` 以便 hashing 或 signing 的最简方式。
- **`CALL` arity is a runtime concern.** immediate 仅是“弹出这么多参数”的提示，使方法定义无需硬编码 arity 约束。
- **Dict construction.** dict 不是首要集合类型，但通过 `SET` 与类型转换辅助（见 `8.Conversion-Instructions.md`）可以由两条并行 slice 构建 dict。
