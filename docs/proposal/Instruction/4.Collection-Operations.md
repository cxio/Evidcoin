# Collection Operations（集合指令）

## 来源构想

- `docs/conception/Instruction/4.集合指令.md`：定义 opcode `35-45`。
- `docs/proposal/Instruction/0.Base-Constraints.md`：定义实参和返回值模型。

## 范围

集合指令处理 Slice、Dict、Object 和 Module：

- `SLICE`
- `REVERSE`
- `MERGE`
- `EXTEND`
- `PACK`
- `SPREAD`
- `INDEX`
- `ITEM`
- `SET`
- `CALL`
- `SIZE`

## 规范要求

- Dict 必须保留插入顺序，不能使用实现语言 map 遍历顺序。
- `PACK` 必须明确运行时值转 Bytes 的编码来源；未固定前不能用于协议 Hash 输入。
- `SPREAD` 显式展开集合成员。
- `INDEX` 必须固定负索引、索引集合和越界行为。
- `ITEM` 对不存在键返回 `nil` 还是失败必须固定。
- `SET`、`DEL`、`CLEAR` 等修改操作必须明确是否原地修改。
- `CALL` 属于半确定性取参，实参数量由附参决定。
- 模块方法多返回值应封装为 Slice，避免自动展开歧义。

## 风险

- Dict 原地修改和浅复制会产生副作用。
- `CALL` 动态方法分派会破坏静态审计和成本估算。
- `PACK` 若使用非规范编码，会导致跨实现差异。

## 未决问题

- `PACK` 采用脚本值编码、基础规范编码还是专用编码。
- Dict/Module/Object 的比较、复制和 Hash 规则。
- `CALL` 方法名大小写、未知方法和版本治理。
- `SET` 是否允许修改共享引用对象。

## 对后续 Plan 的影响

- 集合测试必须覆盖顺序、原地修改、浅复制副作用、未知键、越界索引和 `CALL` 取参数量。
