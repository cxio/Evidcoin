# Interaction Instructions（交互指令）

## 来源构想

- `docs/conception/Instruction/5.交互指令.md`：定义 opcode `46-50`。
- `docs/conception/6.脚本系统.md`：定义 INPUT/OUTPUT 缓存区和外部监听。
- `docs/proposal/Instruction/0.Base-Constraints.md`：定义公共/私有边界。

## 范围

交互指令连接脚本和外部环境：

- `INPUT`
- `OUTPUT`
- `BUFDUMP`
- `PRINT`
- opcode `49` 保留。

## 规范要求

- INPUT 缓存区和 OUTPUT 缓存区独立，不互通。
- `INPUT` 在公共验证中视作隐式 `END`。
- 私有节点执行 `INPUT` 时，缓存区无值或数量不匹配应正常失败，不影响调用者既有 pass 状态的精确语义仍需确认。
- `OUTPUT` 非阻塞，可多次累积。
- `BUFDUMP` 将 OUTPUT 缓存区数据转出并清空。
- `PRINT` 若保留，应定位为调试或私有输出，不得影响共识。
- GOTO/EMBED 可共享缓存区，但缓存区不得成为 GOTO 返回数据通道。

## 风险

- `INPUT` 若进入公共验证，会引入非确定性。
- `OUTPUT/BUFDUMP` 容易被误认为链上状态。
- 私有监听和公共验证混在一起会导致节点行为差异。

## 未决问题

- 私有 `INPUT` 失败是否保留 pass 状态。
- `PRINT` 是否允许在公共路径执行。
- `BUFDUMP` 标识值格式和监听器接口。
- 交互缓存区大小和成本。

## 对后续 Plan 的影响

- 执行器必须能以公共模式和私有模式运行。
- 测试必须覆盖公共模式遇到 `INPUT`、私有模式缓存不足、OUTPUT 累积和 BUFDUMP 清空。
