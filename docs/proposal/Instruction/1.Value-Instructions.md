# Value Instructions（值指令）

## 来源构想

- `docs/conception/Instruction/1.值指令.md`：定义 opcode `0-18` 的运行时值。
- `docs/proposal/Instruction/0.Base-Constraints.md`：提供指令元规则。
- `docs/proposal/09.Script-System.md`：提供运行时值集合。

## 范围

值指令 opcode 范围为 `0-18`，作为具体值存在，执行时直接入栈或构造值。

| opcode | 指令 | 类型 | 规格边界 |
|--------|------|------|----------|
| 0 | `NIL` | Nil | 无值。 |
| 1-2 | `TRUE` / `FALSE` | Bool | opcode 直接表达 true/false。 |
| 3 | byte literal | Byte | 单字节值。 |
| 4 | rune literal | Rune | Unicode code point，4 bytes。 |
| 5 | int literal | Int | signed varint 未决。 |
| 6 | bigint literal | BigInt | 长度小于 256 bytes，规范符号编码未决。 |
| 7 | float literal | Float | float64，NaN/Inf/-0 规则未决。 |
| 8 | string literal | String | UTF-8 字节串，长度 varint 未决。 |
| 9 | value set | Values | 成员类型可提升。 |
| 10 | `DATA` | Bytes | 字节序列。 |
| 11 | regexp literal | RegExp | RE2 语义，最大 255 bytes。 |
| 12 | `DATE` | Time | Unix milliseconds 或 RFC3339。 |
| 13 | `DICT` | Dict | 有序 map，使用实参构造。 |
| 14-15 | reserved | - | 保留。 |
| 16 | `CODE` | Script | 字面脚本值，不执行。 |
| 17 | `SCRIPT` | Script | 引用外部输出脚本，不执行。 |
| 18 | `VALUE` | Any | 通用取值。 |

## 规范要求

- 值指令执行后产生运行时值，不代表该值自动具备规范化链上编码。
- `CODE` 和 `SCRIPT` 入栈后不执行。
- `SCRIPT` 需要外部脚本获取接口，获取失败必须导致确定性失败或策略性拒绝。
- `DICT` 是值指令中需要运行时实参构造值的特殊指令。
- RegExp 必须使用 RE2 等确定性、无回溯爆炸风险的语义。
- Time 不得在公共验证中隐式使用本地当前时间；`DATE` 字面值必须固定为确定性值。

## 风险

- BigInt、Int、Float 若无规范编码，会导致跨实现不一致。
- Float 的 NaN、Inf 和 -0 会破坏比较和 Hash 稳定性。
- `SCRIPT` 可能降低交易验证优先级，需限制外部脚本查询成本。
- 字符串转义若绑定 Go 语法，需确保协议版本长期稳定。

## 未决问题

- signed varint 具体算法。
- BigInt 符号、零值和最短编码。
- Float 是否允许 NaN/Inf，及规范化 bit pattern。
- Time 的内部时区和 RFC3339 小数秒规范。
- `SCRIPT` 在解锁脚本中的政策。
- `VALUE` 可取值表和与 `ENV` 的边界。

## 对后续 Plan 的影响

- 值指令测试必须覆盖每种字面量的解析、入栈值、非法编码和大小限制。
- 在 signed varint 未决前，不得生成 Int/Date 最终字节测试向量。
- RegExp 测试必须验证非 RE2 特性被拒绝。
