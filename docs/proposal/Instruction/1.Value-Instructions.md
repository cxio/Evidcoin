# Proposal: Value Instructions [0-18]（提案：值指令 [0-18]）

> **Prerequisite:** 本规范基于 [Base Constraints](0.Base-Constraints.md) 文档，该文档定义了统一使用的指令格式、运行时模型与记号约定。


## 1. Overview（概览）

值指令占用 opcode 范围 **[0-18]**（共 19 条）。这类指令表示字面值，执行后会自动将自身结果压入 data stack。它们是脚本系统中最基础的数据生产指令。

执行环境中的数据引入主要依赖此类指令：常量、类型字面量、内联数据、代码块、外部脚本引用，都经由该类别进入运行时。

### Design Principle（设计原则）

每条值指令都具备自描述性：opcode 决定类型，辅助参数（若存在）决定尺寸，associated data 承载具体负载，因此其产出类型不存在歧义。

数字字面量统一写在 `{}` 中（而不是裸数字），以保持脚本“指令序列”范式的一致可读性，使每个元素边界清晰。


## 2. Instruction Definitions（指令定义）

### 2.1 Nil Value（Nil 值）

```
0       NIL         Null value: nil
```

- **Type:** Nil
- **Auxiliary params:** None
- **Associated data:** None
- 仅用 opcode 字节即可表示该值。执行时向 data stack 压入 `nil`。
- 用作通用的“无值”标记。

### 2.2 Boolean Values（布尔值）

```
1       TRUE        Boolean true
2       FALSE       Boolean false
```

- **Type:** Bool
- **Auxiliary params:** None
- **Associated data:** None
- 仅 opcode 字节编码布尔值，运行时表示为 `true` 或 `false`。
- bytecode 示例：`0x01` -> `true`，`0x02` -> `false`。

### 2.3 Byte Value（Byte 值）

```
3       ''          Single byte: byte [0, 255]
```

- **Type:** Byte
- **Auxiliary params:** None
- **Associated data:** 1 byte，表示 uint8 值。
- 是 uint8 的别名，适用于小正整数运算。
- 语法上使用单引号包裹字符字面量。
- **Examples:**
  - `'A'`：字符 A（值 65），编译后：`0x03 0x41`。
  - `'\n'`：换行转义序列（值 10）。
  - `'\x41'`：A 的十六进制转义。

### 2.4 Rune Value（Rune 值）

```
4       ''          Rune (Unicode code point): rune [0, 0x10FFFF]
```

- **Type:** Rune
- **Auxiliary params:** None
- **Associated data:** 4 bytes，表示 uint32 的 Unicode code point。
- 表示单个 Unicode code point。
- **Examples:**
  - `'嗨'`：单引号中的中文字符。
  - `'\u4e16'`：Unicode 转义形式（`\uhhhh` 或 `\Uhhhhhhhh`）。
  - `'\u0041'`：以 Rune 类型存储字符 A（4 字节，与 Byte 区分）。
- **Disambiguation:** 编译器根据 code point 范围判断单引号字面量应编译为 Byte（opcode 3）还是 Rune（opcode 4）。默认 `<= 255` 使用 Byte；显式使用 `\u` / `\U` 或超出 byte 范围时使用 Rune。

### 2.5 Integer Value（整数值）

```
5       {}          Integer: int64 [-1<<63, 1<<63 - 1]
```

- **Type:** Int
- **Auxiliary params:** None
- **Associated data:** 变长整数（varint 编码）。
- 标准有符号 64 位整数。
- **Examples:**
  - `{12345}`：整数 12345。
  - `{123_45}`：同一数值；允许使用单下划线作视觉分隔。

### 2.6 Big Integer Value（大整数值）

```
6       {}(1)       Big integer: big.Int
```

- **Type:** BigInt
- **Auxiliary params:** 1 byte，表示后续大整数数据的字节长度。
- **Associated data:** 采用 big.Int 编码的大整数值。
- 最大 255 字节（受 1 字节长度前缀限制）。
- **Examples:**
  - `{0x8000ffff_8000ffff_8001}`：超出 int64 范围，按 BigInt 解析。
- 通常使用十六进制书写，允许下划线分隔。

### 2.7 Float Value（浮点值）

```
7       {}          Float: float64
```

- **Type:** Float
- **Auxiliary params:** None
- **Associated data:** 8 bytes，IEEE 754 float64。
- **Examples:**
  - `{3.14159}`：浮点字面量。
  - `{2.16e10}`：科学计数法。
  - `{123.}`：末尾小数点用于区分整数 `{123}`。
- **Disambiguation:** 是否包含小数点（`.`）或指数记号（`e`/`E`）用于区分 Float 与 Int。

### 2.8 String Values（字符串值）

```
8       ""(~)       String: string
```

- **Type:** String
- **Auxiliary params:** varint，表示字符串字节长度。
- **Associated data:** UTF-8 编码字符串内容。
- 双引号字符串支持 Go 风格转义。
- 反引号语法（`` `...` ``）表示 raw string，不做转义处理。
- **Examples:**
  - `"Hello world"`：普通双引号字符串。
  - `"Hello\nWorld"`：带转义序列的字符串。
  - `` `Hello\nworld` ``：raw string，`\n` 为两个普通字符，不是换行。

### 2.9 Value Set (Typed Array Literal)（Value Set：类型化数组字面量）

```
9       {...}(~)    Value set: [...]
```

- **Type:** Typed slice（例如 `[]Int`、`[]Float`、`[]String`）
- **Auxiliary params:** varint，成员值序列的总字节长度。
- **Associated data:** 值指令序列，每个成员都采用自身 opcode 格式编码。
- 用于内联构造同质类型数组的便捷语法。
- **Type uniformity requirement:** 所有成员类型必须一致，或至少可 **promotable** 到同一目标类型。提升在运行时执行：
  - `Byte -> Rune -> Int -> BigInt`
  - `Byte -> Rune -> Int -> Float`
  - Float、BigInt 与 String 为终止类型，不能再提升。
- **Examples:**
  - `{TRUE, FALSE}`：`[]Bool`
  - `{'A', 'B'}`：`[]Byte`
  - `{'你', '好'}`：`[]Rune`
  - `{2, 4, 8}`：`[]Int`
  - `{100, }`：单成员集合 `[100]`，必须保留尾逗号。
  - `{0x8000ffff_8000ffff_8001, 0xffff}`：`[]BigInt`
  - `{3.14, 5}`：`[]Float`（Int `5` 提升为 Float）。
  - `{"Hello", "World"}`：`[]String`
- **Error case:** `{}` 在无类型上下文时为错误。
- **Design note:** 类型提升发生在运行时而非编译时。它属于值指令中的特殊机制，不是独立类型，只是便捷的内联构造方式。

### 2.10 Byte Data (Binary Blobs)（字节数据/二进制块）

```
10      DATA{}(~)   Bytes: []byte
```

- **Type:** Bytes
- **Auxiliary params:** varint，表示数据长度。
- **Associated data:** 原始字节内容。
- 使用无 `0x` 前缀的十六进制写法。
- **Examples:**
  - `DATA{46af3fb4}`：4 字节，长度自动计算。
  - `DATA[8]{46af3fb4}`：显式指定 8 字节长度，尾部不足部分补零。
- **Compiled bytecode example:** `DATA{414243}`（内容为 "ABC"）->
  - `0x0A`（opcode 10）、`0x03`（长度 3）、`0x41 0x42 0x43`（数据）。

### 2.11 Regular Expression（正则表达式）

```
11      /.../(1)    Regular expression
```

- **Type:** RegExp
- **Auxiliary params:** 1 byte，表示内容字节长度（最大 255）。
- **Associated data:** UTF-8 编码的正则表达式。
- 表达式必须合法，具体语法由实现采用的正则引擎定义。
- 书写形式为斜杠包裹。
- **Examples:**
  - `/(?i)[a-z0-9]+/`：大小写不敏感的 ASCII 字母数字匹配。
  - `/(?i)Zip-(\d+)/`：大小写不敏感并包含捕获组。
- **Size limit:** 表达式文本最多 255 字节。
- **Design note:** 这是静态数据，因此编译时会验证表达式是否合法。用户有机会检查修改。可能采用 RE2 引擎（不支持回溯，安全且性能可预测）。

### 2.12 Date/Time Value（日期时间值）

```
12      DATE{}      Time object
```

- **Type:** Time
- **Auxiliary params:** None
- **Associated data:** 有符号变长整数，单位为毫秒的 UNIX timestamp。
- 花括号语法接受 RFC 3339 格式日期时间（编译为 timestamp）。
- 方括号语法接受原始 UNIX 毫秒时间戳。
- **Examples:**
  - `DATE{2020-11-24T15:04:05Z}`：RFC 3339 UTC。
  - `DATE{2020-11-24T15:04:05.456+08:00}`：带时区的 RFC 3339。
  - `DATE[1606227845000]`：原始 UNIX 毫秒时间戳。

### 2.13 Construct Dict（构造 Dict）

```
13      DICT        Construct a Dict (map[string]any)
```

- **Type:** Dict（有序 map[string]any）
- **Auxiliary params:** None
- **Associated data:** None
- **Arguments:**
  1. Key 序列（`[]string`）。
  2. Value 序列（`[]any`）。
- **Pairing rules:**
  - Key 与 Value 按位置配对。友好支持两个序列长度不同：
  - Key 多于 Value 时，多出的 Key 对应 `nil`。
  - Value 多于 Key 时，多出的 Value 会被**忽略**。
- **Example:**
  ```
  {"one", "two", "123"}
  {100} {200} NIL {1.0}
  DICT
  // 结果：{"one": 100, "two": 200, "123": nil}
  // 1.0 被忽略（只有 3 个 key）
  ```
- **Iteration order:** 成员按插入顺序迭代（确定性）。
- **Design note:** Dict 无法通过关联数据创建，是唯一需要 **Arguments** 的值指令。

### 2.14 Reserved（保留）

```
14      (Reserved, unused)
15      (Reserved, unused)
```

预留 opcode，供未来扩展新的值类型。

### 2.15 Code Block (Inline Instruction Sequence)（代码块/内联指令序列）

```
16      CODE{}(~)   Code/instruction sequence
```

- **Type:** Script
- **Auxiliary params:** varint，表示封装指令序列的字节长度。
- **Associated data:** 编译后的指令序列（opcode + 辅助参数 + associated data）。
- 内部指令属于 **inert data**：遇到时不会立即执行。整个 CODE 块会作为 Script 类型值压入栈。
- 可与 `EVAL` 配合做动态执行，或作为可执行逻辑提供给第三方扩展。
- **Example:**
  ```
  CODE{
       TOP FN_HASH256 DATA{<hashResult>} EQUAL PASS
  }
  ```
  这是一个以数据形式保存的哈希屏障校验序列。
- **Compiled bytecode:** `0x10`（opcode 16）+ varint 长度 + 编译后的内部序列。

### 2.16 Script Reference (External)（外部脚本引用）

```
17      SCRIPT(~,48,~)  Script reference
```

- **Type:** Script
- **Auxiliary params:**
  1. varint：交易年份（由交易 timestamp 计算）。
  2. 48 bytes：transaction ID。
  3. varint：输出索引（0-based）。
- **Associated data:** None
- 从指定交易输出读取 lock script，并将其作为 Script 类型值压入栈。
- 与 `GOTO`/`EMBED` 不同，本指令 **not** 执行目标脚本，只是加载为数据。
- **Example:**
  ```
  SCRIPT[2026, 0xab93cf..., 0] FN_HASH256 ...
  ```
  读取 2026 年某交易的第一个输出脚本，再对其做哈希。
- **Use case:** 用于引用存证交易中的识别脚本，以做校验或比对。

### 2.17 Generic Value Accessor（通用值访问器）

```
18      VALUE(1){}      Retrieve a named/indexed value
```

- **Type:** Any
- **Auxiliary params:** 1 byte，值索引。
- **Associated data:** None
- 通用值检索指令，与 `ENV{}` 有部分重叠，但访问能力更广。
- **Examples:**
  - `VALUE{someName}`：按名称检索（友好）。名称只是索引的别名，实际索引由附参表达。
  - `VALUE[0]`：直接用值索引检索（不友好）。
- `VALUE{name}` 不携带关联数据，编译后仍只保存 1 字节索引附参。
- **Design note:** 单独设立该指令，是为保证某些关键值可在 unlock script（opcode [0-50]）中使用。


## 3. Type System Summary（类型系统汇总）

值指令引入如下类型层级：

| Type | Opcode | Size | Promotable To |
|------|--------|------|---------------|
| Nil | 0 | 0 bytes | — |
| Bool | 1, 2 | 1 byte (opcode only) | — |
| Byte | 3 | 1 byte | Rune, Int, BigInt, Float |
| Rune | 4 | 4 bytes | Int, BigInt, Float |
| Int | 5 | 1-10 bytes (varint) | BigInt, Float |
| BigInt | 6 | 1-255 bytes | — |
| Float | 7 | 8 bytes | — |
| String | 8 | variable | — |
| Typed slice | 9 | variable | — |
| Bytes | 10 | variable | — |
| RegExp | 11 | 2-256 bytes | — |
| Time | 12 | 1-10 bytes (varint) | — |
| Dict | 13 | variable | — |
| Script | 16, 17 | variable | — |
| Any | 18 | variable | — |

### Promotion Rules（提升规则）

类型提升适用于 value set（opcode 9），并在运行时计算：

```
Byte -> Rune -> Int -> BigInt
Byte -> Rune -> Int -> Float
```

Float、BigInt 与 String 是终止类型，不存在继续提升路径。BigInt 与 Float 都由 Int 分叉提升，因此同一 value set 中不能同时出现 BigInt 与 Float 成员（两者路径互斥）。


## 4. Compilation Notes（编译说明）

- 编译器会自动将字面量语法映射为合适的 opcode（如 `'A'` 按 code point 映射到 Byte 或 Rune；`{123}` -> Int，`{123.}` -> Float）。
- 辅助参数值（如字符串长度、数据长度、varint 整数）在编译期计算。
- value set 成员的类型兼容性会尽可能在编译期校验；跨类型场景通过运行时提升处理。
