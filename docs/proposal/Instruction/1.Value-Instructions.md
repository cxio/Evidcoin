# Value Instructions（值指令）

Status: Draft proposal（草案提案）
Source conception: `conception/Instruction/1.值指令.md`

## Prerequisites（前置条件）

关于栈模型、本地/全局帧、opcode 布局与指令编码约定，请参见
[`0.Base-Constraints.md`](./0.Base-Constraints.md)。
下面是本文会用到概念的简要回顾：

- **Data stack** — 主 LIFO 区域。值指令会自动将结果压入该栈
  （唯一例外是 `DICT`，它从运行时参数构造结果）。
- **Immediate operand `(n)`** — 紧跟 opcode 的编译期字节；
  宽度为 `n` 字节，`~` 表示 varint。
- **Associated data `{}`** — 内联负载，即将要产生的值的字面内容。
- **Runtime argument** — 不属于编码内容；在本指令族中只有 `DICT`
  会消费运行时参数。

## Overview（概述）

值指令是主要的字面量生产指令族。该范围内每个 opcode 都编码一个
单一的类型化常量（或小型构造器），其结果会**automatically pushed onto the data stack**，
无需额外胶水逻辑。它们也是其他指令族的基础构件：比较、算术、哈希、
脚本引用等都消费由本指令族引入的值。

## Opcode Range（Opcode 范围）

`[0, 18]` — 19 个 opcode（`14` 与 `15` 预留供未来使用）。

## Instruction Catalog（指令目录）

```go
opcode  mnemonic    short description
-----------------------------------------------------------
0       NIL         nil 值
        // Type: nil
        // Stack effect: ( -- nil )
        // 该值由 opcode 本身编码；无 immediate operand，
        // 无 associated data。是通用“无值”标记，凡是可接受
        // 缺失输入的指令都可使用。

// --- 基础标量类型 ---

1       TRUE        布尔 true
2       FALSE       布尔 false
        // Type: Bool
        // Stack effect: ( -- bool )
        // opcode 数值本身即运行时值：
        //   opcode 1 -> true
        //   opcode 2 -> false
        // 注意：交易脚本是这些 opcode 编译后的字节序列，
        // 不是源码文本。

// --- 携带 immediate operands 或 associated data 的基础类型 ---

3       ''          [0, 255] 范围的 byte 字面量
        // Type: Byte (uint8 alias)
        // Data: 1 byte 存储该值。
        // Stack effect: ( -- byte )
        // 适用于小型无符号整数运算。
        // Source-level forms:
        //   'A'      单引号字符（值为 65）
        //   '\n'     转义序列（值为 10）
        //   '\x41'   十六进制转义（值为 65）
        // Encoding example: 'A' 编译为 0x03 0x41
        //   0x03 = opcode (Byte), 0x41 = byte 本体。

4       ''          [0, 0x10FFFF] 范围的 rune 字面量
        // Type: Rune (Unicode code point, uint32)
        // Data: 4 bytes（big-endian）存储 code point。
        // Stack effect: ( -- rune )
        // Source-level forms:
        //   '嗨'        单引号 Unicode 字符
        //   '\u4e16'    \uhhhh 转义（CJK '世'）
        //   '\u0041'    以 Rune 存储的 'A'（区别于 Byte 'A'）

5       {}          [-2^63, 2^63 - 1] 范围的 int64 字面量
        // Type: Int
        // Data: 有符号 varint。varint 自分隔，
        // 因此不需要 immediate 长度操作数（节省 1 byte）。
        // Stack effect: ( -- int )
        // Source-level forms:
        //   {12345}     字面量 12345
        //   {123_45}    同值；允许使用下划线分隔符。
        // Design note:
        // 保留花括号是为了强化视觉边界，延续栈式脚本
        // “指令序列”风格。

6       {}(1)       任意精度整数（arbitrary-precision）字面量
        // Type: BigInt (Go math/big.Int, length-bounded < 256 bytes)
        // Immediate: 1 byte 表示 payload 的字节长度。
        // Data: big-endian 幅值字节（符号按引擎 BigInt 编码打包）。
        // Stack effect: ( -- bigint )
        // Source-level form（通常为十六进制）:
        //   {0x8000ffff_8000ffff_8001}
        // 超过 int64 范围的字面量会被解析为 BigInt。

7       {}          float64 字面量
        // Type: Float
        // Data: 8 bytes IEEE-754 binary64。
        // Stack effect: ( -- float )
        // Source-level forms:
        //   {3.14159}   普通十进制
        //   {2.16e10}   科学计数法
        //   {123.}      尾随小数点用于与 Int 区分
        // 必须包含小数点（或指数），这样编译器才能
        // 将 Float 与 Int 区分开。

8       ""(~)       UTF-8 string 字面量
        // Type: String
        // Immediate: payload 字节长度，varint。
        // Data: UTF-8 bytes。
        // Stack effect: ( -- string )
        // 支持 Go 风格转义；也支持反引号 raw 形式。
        // Source-level forms:
        //   "Hello world"
        //   "Hello\nWorld"
        //   `Hello\nworld`   raw，不做转义处理

// --- 复合值类型 ---

9       {...}(~)    同构值集合
        // Type: []Bool, []Byte, []Rune, []Int, []BigInt,
        //       []Float, []String 之一（运行时选择）。
        // Immediate: 总 payload 字节长度，varint。
        // Data: 一段值指令编码序列。
        // Stack effect: ( -- slice )
        // Source-level forms（会编译为 opcode 9，包裹内部
        // value-instruction sequence）:
        //   {TRUE, FALSE}                    -> []Bool
        //   {'A', 'B'}                       -> []Byte
        //   {'你', '好'}                     -> []Rune
        //   {2, 4, 8}                        -> []Int
        //   {3.14, 5}                        -> []Float
        //   {0x8000ffff_8000ffff_8001, 0xffff} -> []*big.Int
        //   {"Hello", "World"}               -> []String
        //   {100, }                          -> 单成员集合 [100]
        // Errors:
        //   - 空字面量 `{}` 在编译期被拒绝
        //     （无法推断元素类型）。
        //   - 无法统一的混合类型会在运行时触发 FAIL。
        // Type promotion rules (runtime):
        //   Byte  => Rune => Int => BigInt
        //   Byte  => Rune => Int => Float
        // BigInt、Float、String 位于提升链顶端。
        // Design note:
        // 类型统一发生在执行期而非编译期。这是 value family
        // 中唯一一个复合构造“捷径”；唯一依赖参数驱动的
        // 构造器请参见 DICT。

10      DATA{}(~)   原始字节序列
        // Type: Bytes ([]byte)
        // Immediate: payload 字节长度，varint。
        // Data: 原始 bytes（源码层：不带 0x 前缀的十六进制）。
        // Stack effect: ( -- bytes )
        // 允许显式长度；尾部不足位用 0 填充。
        // Source-level forms:
        //   DATA{46af3fb4}      自动计算为 4-byte payload
        //   DATA[8]{46af3fb4}   8-byte payload，后补 0
        // Encoding example: DATA{414243} -> 0x0A 0x03 0x41 0x42 0x43.

11      /.../(1)    RE2 正则表达式
        // Type: RegExp
        // Immediate: 1 byte payload 长度（< 256）。
        // Data: 两个斜杠之间的 UTF-8 源串。
        // Stack effect: ( -- regexp )
        // Source-level forms:
        //   /(?i)[a-z0-9]+/      大小写不敏感 ASCII 字母数字
        //   /(?i)Zip-(\d+)/      大小写不敏感并带 capture group
        // Reference: github.com/google/re2/wiki/Syntax
        // Design note:
        // 255-byte 上限是有意设计——实际脚本通常不需要更长表达式，
        // 而小 immediate 可降低校验成本。

12      DATE{}      时间戳值
        // Type: Time
        // Data: 以毫秒为单位的有符号 varint UNIX timestamp。
        // Stack effect: ( -- time )
        // Source-level forms:
        //   DATE{2020-11-24T15:04:05Z}                RFC 3339
        //   DATE{2020-11-24T15:04:05.456+08:00}       带 TZ 的 RFC 3339
        //   DATE[1606227845000]                       原始毫秒时间戳
        // 文本形式仅接受 RFC 3339 语法；
        // 完整解析规则见 MO_TIME。

13      DICT        有序字典构造器
        // Type: Dict (insertion-ordered map[string]any)
        // Args:
        //   arg #1: key 序列 ([]string)
        //   arg #2: value 序列 ([]any)
        // Stack effect: ( keys values -- dict )
        // 键值按位置配对：
        //   - 若 len(keys) > len(values)，缺失值默认补 nil；
        //   - 若 len(keys) < len(values)，多余值会被丢弃。
        // Source-level example:
        //   {"one", "two", "123"}     // keys
        //   {100} {200} NIL {1.0}     // values
        //   DICT
        // 结果为：
        //   { "one": 100, "two": 200, "123": nil }   // (1.0 被丢弃)
        // Design note:
        // 迭代顺序遵循插入顺序，而非 Go 原生 map 的随机顺序。
        // DICT 是值指令中唯一需要运行时参数来构造结果的指令。

14-15   (reserved)

16      CODE{}(~)   已编译指令序列（字面量）
        // Type: Script
        // Immediate: payload 字节长度，varint（自动计算）。
        // Data: 一段已编译完成的指令序列。
        // Stack effect: ( -- script )
        // Source-level form:
        //   CODE{
        //       TOP FN_HASH256 DATA{<hashResult>} EQUAL PASS
        //   }
        // Security:
        // CODE 携带的字节是纯字面数据；在求值 CODE 本身时
        // 不会执行。只有传给 EVAL 或第三方扩展并被其选择解释时，
        // 才会变为可执行内容。

17      SCRIPT(~,48,~)   外部脚本引用
        // Type: Script
        // Immediates:
        //   #1 (varint) 交易年份（由 timestamp 推导）
        //   #2 (48 B)   transaction ID (TxID, 48 bytes)
        //   #3 (varint) 输出索引（0-based）
        // Stack effect: ( -- script )
        // 加载被引用输出的 locking script，并将其作为
        // Script 值压栈。
        // Source-level form:
        //   SCRIPT[2026, 0xab93cf..., 0] FN_HASH256 ...
        // 典型用法：引用既有 notarization（proof）交易的
        // recognition script。

18      VALUE(1){}      通用索引值查找
        // Type: Any
        // Immediate: 1-byte 值索引。
        // Data: 花括号内可选 symbolic alias（仅别名，
        // 不改变编码）。
        // Stack effect: ( -- any )
        // Source-level forms:
        //   VALUE[0]              数字索引
        //   VALUE{someName}       当 `someName` 是索引 0 的
        //                         注册别名时，与 VALUE[0] 等价。
        // 注意语法：此处花括号是 alias 容器，不是
        // associated-data 载体；规范签名是
        // `VALUE(1){}`，不是 `VALUE{}(1)`。
        // Design note:
        // 该指令作为通用 accessor，覆盖 unlock 侧 ENV 未覆盖的
        // 取值场景（与 ENV 存在重叠是有意设计）。同时也为前瞻性的
        // pre-evaluation 模式保留空间（例如量子时代需求）。
```

## Design Notes（设计说明）

- **Auto-push semantics.** 除 `DICT` 外，每条值指令都会把结果直接
  压入 data stack，无需辅助拼接逻辑。这使值指令能与所有消费者
  统一互操作。
- **Runtime type promotion in `{...}`.** 类型统一发生在执行期而非
  编译期。编译器按原样发出字面字节；引擎再选择能表示全部成员的
  最小类型。提升链为
  `Byte → Rune → Int → BigInt` 与 `Byte → Rune → Int → Float`。
- **Brace-vs-bracket convention.** 在值指令中，花括号 `{}` 通常表示
  *associated data is present*；`VALUE(1){}` 是唯一例外，此处花括号用于
  引入 alias。方括号 `[]` 始终引入数字 immediate operands。
- **Encoding compactness.** `Int`（opcode 5）刻意省略 immediate 长度
  操作数，因为 varint 自分隔；每个整数可节省 1 byte。
- **`CODE` is data, not code.** `CODE{...}` 中字节是惰性字面量；
  在传递给 `EVAL` 或其他显式选择求值的消费者前，不会执行。
