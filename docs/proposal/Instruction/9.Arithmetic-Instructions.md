# Proposal: Arithmetic Instructions [80–103]（提案：算术指令 [80–103]）

> **Prerequisite:** 本规范建立在 [Base Constraints](0.Base-Constraints.md) 文档之上；该文档定义了全局采用的指令格式、运行时模型与记号约定。


## 1. Overview（概览）

算术指令占用 opcode 范围 **[80–103]**（共 24 条），提供两种不同形态的计算能力：

1. **Symbol instructions**（opcode 81–85）：运算符号（`*`、`/`、`%`、`+`、`-`），仅可在表达式内部使用。
2. **Named instructions**（opcode 86–103）：单词形式指令，可在表达式内外使用。

这种双形态设计让脚本既能在数学表达中使用自然的中缀写法，也能在通用计算中保留基于栈的指令模型。

### Expression Model（表达式模型）

表达式指令 `()`（opcode 80）会创建独立且自包含的算术求值上下文：

- 表达式中的所有数值会**自动转换为 Float (float64)**。
- 表达式返回**单个 Float 值**。
- 可通过子表达式 `((...))` 嵌套。
- 在表达式内部，value instructions 无需花括号，数字可按自然写法书写。
- 表达式内指令产生的值是操作数，不会作为栈条目入栈。

**表达式内支持的输入类型：**

| 类型 | 自动转换为 Float | 说明 |
|------|----------------|------|
| Byte | ✓ | 直接扩宽 |
| Rune | ✓ | 直接扩宽 |
| Int | ✓ | 标准转换；绝对值超过 2⁵³ 时精度有损 |
| Float | ✓（无需转换） | 直接使用 |
| Bool | ✗ | **不支持**，执行失败 |
| String | ✗ | **不支持**，执行失败 |
| BigInt | ✗ | **不支持**，执行失败（范围不可预测，禁止隐式转换）|
| 其他复合类型 | ✗ | **不支持**，执行失败 |

> **Design decision (confirmed):** 表达式始终返回 Float (float64)。这是有意设计：表达式仅用于浮点计算。若需整数精确计算，应在表达式外使用 `MUL`、`ADD`、`SUB` 等 named instructions。

> **⚠️ 精度警告：** Int 转 Float 时，`float64` 仅有 53 位有效尾数。当 Int 的绝对值超过 **2⁵³（约 9007 万亿）** 时，转换结果会静默丢失精度。如需对大整数进行精确计算，请在表达式外使用 named instructions（`MUL`、`ADD` 等）。


## 2. Expression Instruction（表达式指令）

### 2.1 Expression Wrapper（表达式封装）

```
80      ()(1)       Expression encapsulation / grouping
```

- **Auxiliary params:** 1 byte —— 表达式字节长度（最大 255 bytes）。
- **Arguments:** 无。
- **Return value:** Float（float64）。
- 创建一个独立计算单元。表达式内部：
  - 普通指令可取值并计算，但其返回值是**操作数**，不会自动压入栈，也不能被 `@` 或 `$` 捕获。
  - 子表达式 `((...))` 用于优先级分组，其结果同样是操作数。
  - 顶层表达式结果会自动压入栈（也可在外层 `(` 上加 prefix 捕获到其他目标空间）。
- **Complexity limit:**
  - 该 1-byte 辅助参数（非 varint）将表达式长度限制在 255 bytes，以约束复杂度。
  - 技巧：如果用户希望计算较多的数据，可以先向栈中或局域存入值，然后在表达式中使用取值类指令。
- **Result conversion:** 表达式返回 Float，若后续指令需要 Int，可紧跟 `INT` 指令截断转换：
  ```
  ( POP * 1.5 ) INT         // 表达式结果截断为 Int
  ```
- **Examples:**
  ```
  ( POP + 100 )             // 弹出栈顶值并加 100
  ((POP + $[0]) * 10)       // 带局部作用域访问的子表达式
  @( POP + 100 )            // 顶层，可将结果捕获到 argument space
  ( POP * 1.5 ) INT         // 浮点计算后转回整数
  ```


## 3. Symbol Instructions (Expression-Only)（符号指令，仅表达式内可用）

这些运算符**仅**在 `()` 表达式内部有效。遵循标准优先级：`*`、`/`、`%` 高于 `+`、`-`；子表达式 `((...))` 可覆盖默认优先级。

### 3.1 Multiply（乘法）

```
81      *           Multiply
```

### 3.2 Divide（除法）

```
82      /           Divide
```

### 3.3 Modulo（取模）

```
83      %           Modulo
```

### 3.4 Add（加法）

```
84      +           Add (unary or binary)
```

### 3.5 Subtract / Negate（减法/取负）

```
85      -           Subtract (binary) or Negate (unary)
```

- 所有符号运算都基于 Float 操作数（在表达式上下文中自动转换）。
- `+` 与 `-` 同时支持**一元**（单操作数）和**二元**（双操作数）用法。
- **Examples:**
  ```
  ( -100 )                  // 一元取负：-100.0
  ( 100 + POP * 1.5 )       // 二元：100 + (top_value * 1.5)
  ```


## 4. Named Instructions — Standard Arithmetic（命名指令：标准算术）

### 4.1 Binary Numeric Operations（二元数值运算）

这些指令接收两个参数，并返回与输入一致的类型（Int 或 Float）。

```
86      MUL         Multiply: MUL(x, y) → x * y
87      DIV         Divide:   DIV(x, y) → x / y
88      ADD         Add:      ADD(x, y) → x + y
89      SUB         Subtract: SUB(x, y) → x - y
90      MOD         Modulo:   MOD(x, y) → x % y (Int) or math.Mod(x, y) (Float)
91      POW         Power:    POW(x, y) → math.Pow(x, y)
```

- **Supported types:** Int、Float；返回类型与输入一致。
- **ADD extended behavior:**
  - String + String → 拼接（`"Hello" + "World"` → `"HelloWorld"`）
  - Bytes + Bytes → 拼接
  - Dict + Dict → 合并（键冲突时以第二个 dict 覆盖第一个）
- **Examples:**
  ```
  {2} {8} POW       // → 256 (Int)
  {2.0} {8} POW     // → 256.0 (Float)
  "Hello" "World" ADD   // → "HelloWorld"
  ```

### 4.2 Bitwise Operations (Integer-Only)（位运算，仅 Int）

这些指令只接受 Int，并返回 Int。

```
92      LMOV        Left shift:   LMOV(x, y) → x << y
93      RMOV        Right shift:  RMOV(x, y) → x >> y
94      AND         Bitwise AND:  AND(x, y)  → x & y
95      ANDX        Bit clear:    ANDX(x, y) → x &^ y
96      OR          Bitwise OR:   OR(x, y)   → x | y
97      XOR         Bitwise XOR:  XOR(x, y)  → x ^ y
```

### 4.3 Unary Operations（一元运算）

```
98      NEG         Negate: -x
```

- **Arguments:** 单个数值（Int 或 Float）。
- **Return value:** 同类型的取负值。
- **Example:** `PEEK NEG` —— 读取栈底值并返回其相反数。

```
99      NOT         Logical NOT: !v
```

- **Arguments:** 单个 Bool 值。
- **Return value:** Bool（取反后）。


## 5. Named Instructions — Quasi-Arithmetic（命名指令：准算术）

这些指令并非严格数学运算，但出于组织简洁归入算术类。它们**不能在表达式内部使用**。

### 5.1 Divide and Modulo（除法与取模）

```
100     DIVMOD      Divide and modulo: DIVMOD(x, y) → x/y, x%y
```

- **Arguments:** 两个值（Int 或 Float）。
- **Return value:** 两个值 —— 商与余数，自动展开到目标空间。

### 5.2 Repeat (Duplicate)（重复/复制）

```
101     REP(1)      Repeat (duplicate)
```

- **Auxiliary params:** 1 byte —— 复制份数。
- **Arguments:** 任意类型的单个值。
- **Return value:** 指定数量的副本，自动展开。
- 对参数值执行多份浅拷贝。
- **Edge cases:**
  - `REP[0]` —— 0 份副本：等效于**移除**栈顶项（反直觉，需谨慎）。
  - `REP[1]` —— 1 份副本：返回原值（几乎无实际意义）。
  - `REP[2]` —— 2 份副本：等价于 `TOP`（复制栈顶）。
  - `REP[3]` —— 3 份副本：通常才开始体现实用价值。
- **Distinction from TOPS:** `TOPS[n]` 是引用 N 个不同栈项；`REP[n]` 是把同一个栈项复制 N 次。

### 5.3 Delete Dict Entry（删除 Dict 条目）

```
102     DEL         Delete dict entry
```

- **Arguments:**
  1. 目标 dict。
  2. Key（string）或 key 序列（`[]string`）。
- **Return value:** 原 dict 引用（原地修改后）。
- **Examples:**
  ```
  "abc" DEL           // 删除 key "abc"
  {"A", "B"} DEL      // 删除 key "A" 与 "B"
  ```

### 5.4 Clear Dict（清空 Dict）

```
103     CLEAR       Clear dict contents
```

- **Arguments:** 目标 dict。
- **Return value:** 原 dict 引用（此时为空）。


## 6. Type Rules Summary（类型规则摘要）

| Category | Instructions | Input Types | Return Type |
|----------|-------------|-------------|-------------|
| Expression symbols | `*`, `/`, `%`, `+`, `-` | 自动转换为 Float | Float |
| Expression wrapper | `()` | — | Float |
| Standard binary | MUL, DIV, ADD, SUB, MOD, POW | Int or Float | Same as input |
| ADD (extended) | ADD | String, Bytes, Dict | Same as input |
| Bitwise | LMOV, RMOV, AND, ANDX, OR, XOR | Int only | Int |
| Unary numeric | NEG | Int or Float | Same as input |
| Unary logic | NOT | Bool | Bool |
| Divide+mod | DIVMOD | Int or Float | Two values, same type |
| Repeat | REP | Any | Same as input |
| Dict operations | DEL, CLEAR | Dict | Dict |
