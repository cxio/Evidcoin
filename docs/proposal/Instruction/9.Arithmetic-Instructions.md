# Arithmetic Instructions（运算指令）

Status: 草案提案
Source conception: `conception/Instruction/9.运算指令.md`

## Prerequisites（前置条件）

关于栈模型、本地帧、opcode 布局、操作数来源，以及 immediate 操作数与关联数据的编码方式，参见 `0.Base-Constraints.md`。

下文使用概念回顾：

- **Expression form** `( … )`：一个自包含的算术单元，其内部节点对操作数运算（而不是直接操作数据栈）。
- **Named form**：常规 opcode，其运行时参数来自参数区（或默认来自数据栈）。
- 在 expression 内部，所有数值都会统一为 `Float`（`float64`）；若需要精确整数运算，必须使用 named form。

## Overview（概述）

Evidcoin 的算术层提供两种互补语法：

1. **Symbolic instructions** —— 使用中缀运算符书写，且仅在 expression block 内合法，例如 `(POP + 100)`。
2. **Named instructions** —— 传统 opcode，如 `ADD`、`MUL`、`POW`，既可在 expression 内使用，也可在 expression 外使用。

expression block 是一个轻量、无副作用的算术子环境：可以从栈或本地帧**读取**值并组合为一个浮点结果，但**不允许在 expression 内部回写到栈或本地帧**。顶层 expression 结果的行为与普通指令返回值一致（自动压入数据栈，或被捕获到参数区）。

另有一小组“准算术”指令（`DIVMOD`、`REP`、`DEL`、`CLEAR`）也收录于此，原因是结构形态相近，尽管其语义并非严格数值运算。

## Opcode range（操作码范围）

范围：`[80 – 103]`，共 **24** 个 opcode。

| Sub-group              | Range     | Count |
| ---------------------- | --------- | ----- |
| Expression block       | `80`      | 1     |
| Symbolic operators     | `81 – 85` | 5     |
| Named numeric (binary) | `86 – 91` | 6     |
| Named bitwise (binary) | `92 – 97` | 6     |
| Named unary            | `98 – 99` | 2     |
| Quasi-arithmetic       | `100–103` | 4     |

## Instruction Catalog（指令目录）

### `()` — Expression block (opcode 80)（表达式块）

```go
80      ()(1)       // 表达式包装器 / 分组
```

- **Type**: 结构型。
- **Operands**: 1 字节 immediate，表示内部 expression 的字节长度。该 1 字节宽度是有意设置，用于限制 expression 复杂度。
- **Stack effect**: 不会隐式弹栈；操作数取自 expression 体内部。会向数据栈压入一个 `Float` 结果（若前缀为 `@` / `$`，则写入捕获目标）。
- **Description**:
  - 构建一个隔离的算术单元。在 block 内部，子表达式 `( … )` 作为优先级分组，其返回值是单个操作数。
  - 数值字面量直接裸写（不使用 `{}` 包裹），与常见表达式语法一致。
  - 所有数值在计算时都会被强制为 `Float`（`float64`）。
  - 在 expression 内可调用普通指令来**读取**值，但这些返回值只会作为操作数，不会写回栈，且*不能*被拦截（`@`、`$`）。
  - 仅**top-level** expression 结果会自动流向数据栈或捕获目标。
- **Errors**:
  - 长度 immediate 超过 1 字节上限（255）。
  - 在 expression 体内使用任何回写类指令。
  - 当 expression 仅允许 Float 运算时，出现不受支持的（非数值）中间值。

示例：

```go
( POP + 100 )           // 弹出栈顶并加 100
((POP + $[0]) * 10)     // 子表达式，混合读取栈与本地帧
@( POP + 100 )          // 捕获顶层表达式结果
```

> 未来扩展：支持 expression 内整数算术（遇到 Float 时自动提升）。当前版本为简化实现，限制 expression 仅使用 Float；需要精确整数运算请使用 named form。

### Symbolic operators (opcodes 81 – 85)（符号运算符）

**仅在 expression block 内有效**。可为单操作数或双操作数，见下。

```go
81      *           // 乘法（双目）
82      /           // 除法（双目）
83      %           // 取模（双目）
84      +           // 加法（单目或双目）
85      -           // 减法（单目或双目）
```

- **Stack effect**: 纯操作数级重写；不与数据栈交互。
- **Description**: 对 `Float` 操作数执行标准浮点算术。
- **Errors**:
  - 除零/模零。
  - 单目形式位置非法。

```go
( -100 )                // 单目负号
( 100 + POP * 1.5 )     // 双目 +、双目 *
```

### Named binary numeric (opcodes 86 – 91)（命名二元数值运算）

接受 `Int` / `Float` 运行时参数，并返回相同类型。`ADD` 对序列类类型做了重载。

```go
86      MUL         // x * y
87      DIV         // x / y
88      ADD         // x + y；也用于拼接（String、Bytes）
                    // 以及字典合并（map[string]any）
89      SUB         // x - y
90      MOD         // x mod y；Float 情况下 => math.Mod(x, y)
91      POW         // math.Pow(x, y)
```

- **Operands**: 两个运行时参数（来自参数区，或默认来自数据栈）。
- **Stack effect**: `[ … x y ] → [ … (x op y) ]`。
- **Errors**:
  - 类型不匹配（例如 `Int` 与 `String` 执行 `ADD`）。
  - 除零/模零。
  - 定宽 `Int` 溢出（遵循平台整数规则）。

```go
"Hello" "World" ADD     // "HelloWorld"
{2}   {8}   POW         // 256          (Int)
{2.0} {8}   POW         // 256.0        (Float)
( 100 + POW )           // POW 从 expression 外部拉取其参数
```

### Named binary bitwise (opcodes 92 – 97)（命名二元按位运算）

仅支持 `Int`；返回 `Int`。

```go
92      LMOV        // x << y
93      RMOV        // x >> y
94      AND         // x & y
95      ANDX        // x &^ y          （位清除）
96      OR          // x | y
97      XOR         // x ^ y
```

- **Errors**: 非整数操作数；移位量为负数。

### Named unary (opcodes 98 – 99)（命名单目运算）

```go
98      NEG         // -x       (Int, Float)
99      NOT         // !v       (Bool)
```

- **Stack effect**: `[ … v ] → [ … (op v) ]`。
- **Errors**: 类型不匹配（对非数值使用 `NEG`，或对非 Bool 使用 `NOT`）。

### Quasi-arithmetic (opcodes 100 – 103)（准算术指令）

出于结构邻近性收录于此；**不可在 expression block 内使用**。

```go
100     DIVMOD      // (x, y) -> (x/y, x%y)
```

- **Operands**: 两个 `Int`/`Float` 运行时参数。
- **Stack effect**: 产生 2 个返回值；自动展开到目标区（数据栈或参数区）。
- **Errors**: 除数为零。

```go
101     REP(1)      // 重复
```

- **Immediate**: 复制次数（`n`）。
- **Operand**: 一个运行时值（待重复项）。
- **Stack effect**: 输出输入值的 `n` 份副本，并展开写入接收区。
- **Notes**:
  - `REP[1]` 重复一次（等效无操作，保留原值）。
  - `REP[2]` ≡ `TOP` 语义（在顶部多一份副本）。
  - `REP`（无 immediate）会重复 *zero* 次 → 输入被消费且不输出任何值；等效于删除栈顶。
    **Counter-intuitive — handle with care.**
  - 复制为浅拷贝。

```go
102     DEL         // 从 Dict (map[string]any) 删除条目
```

- **Operands**:
  1. 目标字典；
  2. key（`String`）或 key 序列。
- **Returns**: 返回该字典（已被修改）的引用。

```go
"abc"      DEL       // 删除 key "abc"
{"A","B"}  DEL       // 删除 key "A" 和 "B"
```

```go
103     CLEAR       // 清空 Dict 的全部条目
```

- **Operand**: 目标字典。
- **Returns**: 返回该字典（现已清空）的引用。

## Design notes（设计说明）

- **Why a 1-byte length immediate on `()`** — 用于限制 expression 复杂度。较长算术链应改写为 named form，借助栈与本地帧保存中间状态。
- **No store-back inside expressions** — 保持 expression 的引用透明性与可序列化性，也使“无副作用”语义可在编译期强约束。
- **Float-only inside expressions** — 避免逐操作数类型跟踪；顶层结果再回流到常规的强类型指令流。
- **`ADD` overloading** — 这是有意的人机工程折中。脚本引擎必须根据第一个参数的运行时类型标签进行分派。
- **`REP` zero-form pitfall** — 在此特别标注，便于实现方在 `REP` 未显式给出次数时提供更友好的编译期告警。
