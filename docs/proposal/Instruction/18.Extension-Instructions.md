# Extension Instructions（扩展指令）

## 来源构想

- `docs/conception/Instruction/18.扩展指令.md`：定义 opcode `251-253`。
- `docs/proposal/Instruction/0.Base-Constraints.md`：定义公共/私有边界和 opcode 保留。

## 范围

扩展指令包括：

- `EXT_MO`
- opcode `252` 保留。
- `EXT_PRIV`
- opcode `254-255` 系统保留。

## 规范要求

- `EXT_MO` 用于公共扩展模块，必须有注册、版本、成本和确定性规则。
- `EXT_PRIV` 仅用于私有路径。
- `EXT_PRIV` 在 `END` 或隐式 `INPUT` 前出现必须失败或静态拒绝。
- 未识别公共扩展必须失败，不能忽略。
- 未识别私有扩展在公共路径不能执行；私有节点可按本地扩展处理。
- 扩展指令不得修改脚本原始 bytecode。

## 风险

- 私有扩展误入公共验证会导致共识分歧。
- 公共扩展无注册表会造成实现碎片化。
- 扩展弃用和升级若无策略，会破坏长期脚本可执行性。

## 未决问题

- `EXT_MO` 注册表治理。
- `EXT_PRIV` 是否允许公共节点反编译显示但不执行。
- 未识别扩展是失败、忽略还是仅私有路径可忽略。
- 扩展模块的版本兼容和弃用策略。

## 对后续 Plan 的影响

- 扩展实现应默认禁用，先提供解析和拒绝测试。
- 公共扩展必须在注册表和测试向量存在后才能启用。
