# Proposal: Extension Instructions [251–253]（扩展指令）

> **Prerequisite:** 本规范建立在 [Base Constraints](0.Base-Constraints.md) 文档之上，该文档定义了全局使用的指令格式、运行时模型与记号约定。


## 1. Overview（概览）

扩展指令占用操作码区间 **[251–253]**（3 条）。它们提供通用扩展机制，用于在基础指令集、函数集、模块集之外继续增加功能。

> **Note:** 操作码 **250** 为标准模块扩展包 `MO_XX`，归属于模块指令（[Module Instructions](17.Module-Instructions.md)）。

> **Note:** 操作码 **254** 与 **255** 为系统保留值，不属于指令集，绝不可出现在脚本 bytecode 中。

> **Note:** 本区指令有待进一步思考和定义，为未来预留空间，当前规范先定义地址框架与目标使用场景。

定义了如下扩展类型：

| Opcode | Instruction | Category | Purpose |
|--------|-------------|----------|---------|
| 251 | EXT_MO | Module | 扩展模块包（64K 大空间，超出 MO_XX 的 256 槽位） |
| 252 | — | — | 保留未用 |
| 253 | EXT_PRIV | Private | 私有扩展（非共识场景） |


## 2. Instruction Definitions（指令定义）

### 2.1 Module-Type Extension（模块型扩展）

```
251     EXT_MO(2){}  Module-type extension
```

- **Auxiliary params:** 2 bytes——目标模块索引（`[0–65535]`），64K 指令位。
- **Associated data:** 可选"目标模块名称"（使用 `{}` 记法时）。
- **Arguments:** 由目标扩展模块构造函数决定（与普通模块一样，可传递实参）。
- **Return value:** 目标扩展模块对象（后续通过 `CALL` 调用其成员方法）。
- **Design notes:**
  - 将模块地址空间扩展到 `MO_XX`（opcode 250）256 子模块之外。
  - 专门的扩展模块空间，提供 64K 指令位，可容纳大规模模块扩展需求。
  - 沿用相同模块模式：模块对象创建后通过 `CALL` 指令调用成员方法。
  - `{}` 记法支持按名称引用扩展模块。
- **Example:**
  ```
  EXT_MO{Storage}           // 引入 Storage 扩展模块
  "Put" <key> <value>
  CALL[2]                   // 调用 Put 方法
  ```

### 2.2 Reserved（保留）

```
252     （未用）
```

操作码 **252** 当前保留未用，为未来扩展预留。

### 2.3 Third-Party Private Extension（第三方私有扩展）

```
253     EXT_PRIV(2){}   Third-party private extension
```

- **Auxiliary params:** 2 bytes——目标扩展索引（`[0–65535]`）。
- **Associated data:** 可选目标名称（使用 `{}` 记法时）。
- **Arguments:** 由第三方扩展决定。
- **Return value:** 由第三方扩展决定。
- **Design notes:**
  - 专用于 **private, non-consensus** 功能。
  - 为专门的私有需求提供，自由定制。
  - 公共校验节点**不会处理**这类指令——若在 `END` 或隐性 `END`（`INPUT`）之前出现，脚本会被视为错误而失败。
  - 适用于私有应用、企业部署、实验性功能等不参与公共链共识的场景。
  - 第三方可在该命名空间内独立开发与注册扩展，而不影响公共网络。
- **Example:**
  ```
  EXT_PRIV{AuditLog}    // 私有审计日志（公共校验节点忽略）
  EXT_PRIV[100]         // 索引 100 的私有扩展
  ```


## 3. Design Notes（设计说明）

### Address Space Summary（地址空间总览）

| Mechanism | Aux Size | Slots | Typical Use |
|-----------|----------|-------|-------------|
| `FN_X` (opcode 224) | 1 byte | 256 | 常用扩展函数 |
| `MO_XX` (opcode 250) | 1 byte | 256 sub-modules | 常用扩展模块 |
| `EXT_MO` (opcode 251) | 2 bytes | 65,536 modules | 大规模模块扩展 |
| `EXT_PRIV` (opcode 253) | 2 bytes | 65,536 | 私有/非共识扩展 |

### Consensus Boundary（共识边界）

`EXT_MO` 参与共识，公共校验节点必须执行并将其效果纳入状态迁移。`EXT_PRIV` 明确排除在共识之外，用于私链或应用层功能，而不增加公共网络负担——这是基础共识边界设计。

### System-Reserved Opcodes（系统保留操作码）

操作码 **254** 与 **255** 为系统永久保留。它们作为哨兵值，不得出现在合法脚本 bytecode 中。实现应将其出现视为 bytecode 格式错误。
