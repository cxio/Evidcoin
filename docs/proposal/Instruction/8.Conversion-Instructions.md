# Conversion Instructions（转换指令）

## 来源构想

- `docs/conception/Instruction/8.转换指令.md`：定义 opcode `67-79`。
- `docs/proposal/Instruction/0.Base-Constraints.md`：定义运行时值和失败边界。

## 范围

转换指令包括：

- `BOOL`
- `BYTE`
- `RUNE`
- `INT`
- `BIGINT`
- `FLOAT`
- `STRING`
- `BYTES`
- `RUNES`
- `TIME`
- `REGEXP`
- `ANYS`

## 规范要求

- 每个转换必须有源类型到目标类型的显式规则表。
- 不允许隐式宽松转换掩盖错误。
- 溢出、非法 UTF-8、非法 Rune、NaN/Inf、负数转无符号等必须失败或明确定义。
- `STRING` 转换需固定进制、浮点格式和 Time 格式。
- `BYTES` 转换需固定 big-endian 或专用编码规则。
- `TIME` 转换使用 Unix milliseconds 和 RFC3339，具体规范未决。
- 不支持 Bytes 直接转 Script，除非未来安全规则明确允许。
- `ANYS` 类型标识必须与值指令 opcode 或独立类型表一致。

## 风险

- 自动转换过宽会降低脚本可审计性。
- Float 到 Int 的截断和溢出容易跨实现不同。
- 字符串解析若依赖 Go `strconv`，需避免版本差异。
- REGEXP 编译成本需计入预算。

## 未决问题

- 转换是否允许 Bool/Int/String 间宽松规则。
- Float NaN/Inf 是否允许转换。
- Time 的零值、时区和小数秒规范。
- BYTES 对 BigInt/Int/Float 的编码。
- ANYS 类型标识表。

## 对后续 Plan 的影响

- 转换测试必须采用表驱动，覆盖所有源类型、成功转换、失败转换和边界值。
