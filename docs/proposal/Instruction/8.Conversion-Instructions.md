# Conversion Instructions（转换指令）

状态：提案草稿
来源构想：`conception/Instruction/8.转换指令.md`

## Prerequisites（前置条件）

关于栈模型、参数区、本地帧与操作码布局，参见 `0.Base-Constraints.md`。简述如下：

- **data stack** 承载中间值；**argument zone** 为下一条指令暂存输入；**local frame** 与 **global frame** 保存作用域变量。
- 每条指令由 `opcode + immediate + associated data` 组成；只有 opcode 是必需项。
- 这些转换会返回一个*新的*运行时值；不仅类型标记会变化，底层字节内容也可能变化。

## Overview（概述）

该指令族用于将运行时值规范化为 VM 的标准类型之一：`Bool`、`Byte`、`Rune`、`Int`、`BigInt`、`Float`、`String`、`Bytes`、`Runes`、`Time`、`RegExp`，以及切片元素转换器 `ANYS`（在 `[]any` 与 `[]T` 之间转换）。出于安全考虑，**conversion targeting the `Script` type is intentionally not provided** ——基础类型之间可自由转换；与复合类型之间仅提供部分双向支持。

## Opcode range（操作码范围）

`[67 – 79]`，共 13 条指令（其中 1 个槽位保留）。

## Conversion summary（转换汇总）

| Opcode | Mnemonic | Target type |
|---|---|---|
| 67 | `BOOL` | `Bool` |
| 68 | `BYTE` | `Byte` |
| 69 | `RUNE` | `Rune` |
| 70 | `INT` | `Int` |
| 71 | `BIGINT` | `BigInt` |
| 72 | `FLOAT` | `Float` |
| 73 | `STRING(1){}` | `String` |
| 74 | `BYTES` | `Bytes` |
| 75 | `RUNES` | `[]Rune` |
| 76 | `TIME` | `Time` |
| 77 | `REGEXP` | `RegExp` |

## Instruction Catalog（指令目录）

### 67 — `BOOL` — to `Bool`（转换为 `Bool`）

| Source | Result |
|---|---|
| `nil` | `false` |
| `Byte` | `0 → false`，否则 `true` |
| `Rune` | `0 → false`，否则 `true` |
| `Int` | `0 → false`，否则 `true` |
| `BigInt` | `0 → false`，否则 `true` |
| `Float` | `x ≤ math.SmallestNonzeroFloat64 → false`，否则 `true` |
| `String` | `"" → false`，否则 `true` |

**Tip.** 判断 dict 或 slice 是否非空，可用：`SIZE BOOL`。

---

### 68 — `BYTE` — to `Byte`（转换为 `Byte`）

| Source | Result |
|---|---|
| `nil` | `0` |
| `Bool` | `false→0`，`true→1` |
| `Rune` | 当 Unicode 值 < 256 时正常转换，否则报错 |
| `Int` | 当值 < 256 时正常转换，否则报错 |
| `BigInt` | 与 `Int` 相同规则 |
| `Float` | 先截断为 int，再按相同规则处理 |

**Note.** 不支持与 `String` 的互转。

---

### 69 — `RUNE` — to `Rune`（转换为 `Rune`）

| Source | Result |
|---|---|
| `nil` | `0` |
| `Bool` | `false→0`，`true→1` |
| `Byte` | 直接转换，值保持不变 |
| `Int` | 当值 ≤ `0x10FFFF` 时正常转换，否则报错 |
| `BigInt` | 相同规则 |
| `Float` | 先截断为 int，再按相同规则处理 |

**Note.** 不支持与 `String` 的互转。

---

### 70 — `INT` — to `Int`（转换为 `Int`）

| Source | Result |
|---|---|
| `nil` | `0` |
| `Bool` | `false→0`，`true→1` |
| `Byte` | 直接转换 |
| `Rune` | 直接转换 |
| `BigInt` | 当值 ≤ `1<<63 - 1` 时正常转换，否则报错 |
| `Float` | 截断小数部分后，再按相同规则处理 |

**Convenience sources:**

- `Time` ——提取 Unix 毫秒时间戳。
- `String` ——接受任意 Go 合法整数字面量：十进制、`0b|0B` 二进制、`0o|0O` 八进制、`0x|0X` 十六进制，且允许使用单下划线作数字分隔。

**Note.** 字符串按 `strconv` 规则解释。

---

### 71 — `BIGINT` — to `BigInt`（转换为 `BigInt`）

| Source | Result |
|---|---|
| `nil` | `BigInt(0)` |
| `Bool` | `false→0`，`true→1` |
| `Byte` | 直接转换 |
| `Rune` | 直接转换 |
| `Int` | 直接转换 |

**Convenience source:**

- `Bytes` ——按大端无符号整数解码（`math/big`）。

### 72 — `FLOAT` — to `Float`（转换为 `Float`）

| Source | Result |
|---|---|
| `nil` | `0.0` |
| `Bool` | `false→0.0`，`true→1.0` |
| `Byte` | 数值转换 |
| `Rune` | 数值转换 |
| `Int` | 数值转换 |

**Convenience source:**

- `String` ——任意合法浮点或科学计数字面量。

### 73 — `STRING(1){}` — to `String`（转换为 `String`）

| Field | Value |
|---|---|
| Immediate | base 或 format 标志（1 byte） |
| Runtime args | 源值 |

**Flag semantics:**

- `2..36` ——数值进制，应用于整数/大整数来源。用方括号写法表示：
  ```go
  STRING[16]   // 十六进制字符串
  ```
- `b, e, E, f, g, G, x, X` ——浮点格式标志，应用于浮点来源。用花括号写法表示：
  ```go
  STRING{E}    // 科学计数字符串
  ```

| Source | Result |
|---|---|
| `nil` | `""` |
| `Bool` | `"true"` / `"false"` |
| `Byte` | UTF-8 字节 → 可打印字符或十六进制形式 |
| `Rune` | code point → 可打印字符或 `\uhhhh` / `\Uhhhhhhhh` 形式 |
| `Int` | 按 base 标志转换（`strconv`） |
| `BigInt` | 按 base 标志转换 |
| `Float` | 按 format 标志转换 |

**Convenience sources:**

- `Bytes` ——按 UTF-8 字节序列解释。
- `Runes`（`[]rune`）——编码为 UTF-8（`[]rune → string`）。

**Design.** `STRING{E}` 中的花括号表示*flag literal*，不是 associated data。因此该助记符写作 `STRING(1){}`（1 字节 flag immediate + 花括号语法 flag）——这与以 `{}` 表示 payload bytes 的指令不同。format-flag 最小 ASCII 值为 `E`（69），与 numeric-base 取值范围 `2..36` 不重叠，因此两种编码不会冲突。

### 74 — `BYTES` — to `Bytes`（转换为 `Bytes`）

| Source | Result |
|---|---|
| `nil` | 空 `[]byte` |
| `Byte` | 包装为单字节切片 |
| `Rune` | UTF-8 编码 |
| `Int` | 大端序，固定 8 字节 |
| `BigInt` | 大端序，长度可变 |

**Convenience sources:**

- `String` ——按 UTF-8 编码。
- `Runes` ——按 UTF-8 编码。
- `Script` ——克隆源字节序列并重标记为 `Bytes`。

**Note.** 出于安全考虑，`Bytes → Script` **not** supported。

---

### 75 — `RUNES` — to `[]Rune`（转换为 `[]Rune`）

这是一个特例：rune 切片在某些处理场景更方便。

| Source | Result |
|---|---|
| `nil` | 空 `[]rune` |
| `Rune` | 包装为单 rune 切片 |
| `String` | UTF-8 解码为 `[]rune` |
| `Bytes` | UTF-8 解码；非法序列产出 `\uFFFD` |

---

### 76 — `TIME` — to `Time`（转换为 `Time`）

| Source | Result |
|---|---|
| `Int` | 视为 Unix 毫秒时间戳 |
| `String` | 必须符合 RFC 3339 |
| `nil` | 零值时间 |

```go
{1608883593536}                           TIME  // => DATE{2020-12-25T08:06:33.536Z}
"2020-12-25T08:06:33.536+08:00"           TIME  // => DATE{2020-12-25T08:06:33.536+08:00}
NIL                                       TIME  // => DATE{1970-01-01T00:00:00.000Z}
```

---

### 77 — `REGEXP` — to `RegExp`（转换为 `RegExp`）

| Source | Result |
|---|---|
| `String` | 编译为正则表达式对象 |

仅接受字符串来源，且来源必须是语法合法的 pattern。

### 79 — `ANYS(1){}` — slice element-type conversion (`[]any` ↔ `[]T`)（切片元素类型转换）

用于在 typed slice 与 `[]any` 之间双向转换。

| Field | Value |
|---|---|
| Immediate | conversion flag（1 byte） |
| Runtime args | 源切片，或 `nil` |

**Flag table:**

| Flag | Direction | Result |
|---|---|---|
| `0` | `nil` or `[]xxx` → `[]any` | `[]any` |
| `1` or `2` | `nil` or `[]any` → `[]Bool` | `[]Bool` |
| `3` | `nil` or `[]any` → `[]Byte`（别名 `Bytes`） | `[]Byte` |
| `4` | `nil` or `[]any` → `[]Rune` | `[]Rune` |
| `5` | `nil` or `[]any` → `[]Int` | `[]Int` |
| `6` | `nil` or `[]any` → `[]*big.Int` | `[]BigInt` |
| `7` | `nil` or `[]any` → `[]Float` | `[]Float` |
| `8` or `9` | `nil` or `[]any` → `[]String` | `[]String` |

仅有 `NIL` 作为来源时，会转换为目标类型的空切片。空 `[]any` 同样会转换为空的 typed slice。

**Friendly form.** 除 `0` 外，flag 值与对应 value-type 指令的 opcode 编号完全一致。也支持带类型名的花括号语法：

```go
NIL ANYS                          // 空 []any
{1,3,5} ANYS                      // []Int -> []any

"AB" "XY" SHIFT[2] ANYS[8]        // []any -> []String
"AB" "XY" SHIFT[2] ANYS{String}   // []any -> []String（等价）
```

压栈成员天然是 `any`，因此元素强制转换发生在转换过程中：

```go
{0} NIL TRUE SHIFT[3] ANYS{Bool}
// => [false, false, true]
```

**Note.** Dict 成员类型为 `any`，所以在填充 dict 前，有时需要先将 typed slice 转为 `[]any`。

## Design notes（设计说明）

- **No `Script` target.** 若允许任意 bytes（或其他运行时值）重标记为 script，VM 将可能执行攻击者提供的 opcode 流，破坏脚本源码只读不变式。因此 `Bytes ↔ Script` 仅允许单向（`Script → Bytes`）。
- **Strict overflow.** 缩窄转换（`Int → Byte/Rune`、`BigInt → Int` 等）在溢出时会触发错误，不会静默截断。
- **Float→Int truncates.** 小数部分会被丢弃；结合上面的严格规则，越界 float 仍会报错。
- **Convenience sources are documented per opcode.** 它们覆盖了常见需求（字符串解析、时间转换、大端字节编解码），同时避免类型矩阵膨胀。
- **`STRING` flag encoding is unambiguous.** numeric base（2..36）与 float-format 字母（`b, e, E, f, g, G, x, X`，最小 ASCII 为 69）占用互不重叠的字节区间，因此单字节 immediate 足够且无歧义。
- **`ANYS` flag aliases mnemonic-numbers.** 复用类型指令 opcode 作为 conversion flag，可形成易记、可自解释的编码。
