# Proposal: Environment Instructions [128–137]（提案：环境指令）

> **Prerequisite:** 本规范基于 [Base Constraints](0.Base-Constraints.md) 文档，后者定义了贯穿全文的指令格式、运行时模型与符号约定。


## 1. Overview（概述）

环境指令占用 opcode 范围 **[128–137]**（共 10 条，其中 opcode 137 保留）。这组指令用于从嵌套执行环境层级中只读获取上下文信息，同时提供全局变量管理与脚本源码提取能力。

### Environment Hierarchy（环境层级）

脚本运行时工作在五层嵌套域中，外层是内层的父域：

```
System Domain  ──►  Transaction Domain  ──►  Validation Domain  ──►  Script Domain  ──►  Block Domain
```

1. **System Domain** —— 节点运行时环境，提供系统级信息，如实时时钟、当前链高度与区块时间。
2. **Transaction Domain** —— 被校验交易所在域，包含其输入、输出，以及这些输入所引用的源输出。
3. **Validation Domain** —— 当前输出的校验上下文，包含 `GOTO`/`EMBED` 引入脚本校验的状态信息。
4. **Script Domain** —— 脚本自身执行域，涵盖顶层脚本与 `GOTO`/`EMBED` 子脚本。数据栈与参数空间属于该域。通过 `GOTO` 调用的子脚本拥有独立的数据栈和参数空间。
5. **Block Domain** —— 语法块级局部作用域。各 Block Domain 相互隔离，不能访问彼此局部变量。

### Design Principles（设计原则）

大多数环境指令不需要显式参数 —— 通过 auxiliary parameter 指定目标条目即可读取；`SETVAR` 是例外，它需要接收一个赋值参数。

这些指令暴露的环境信息也应同步提供给 middleware 层，便于外部系统感知脚本执行状态。


## 2. Instruction Definitions（指令定义）

### 2.1 Environment Variable Retrieval（环境变量读取）

```
128     ENV(1){}    Environment variable retrieval
```

- **Auxiliary params:** 1 byte —— 目标标识，指定要读取的环境条目。
- **Arguments:** 无。
- **Return value:** 对应环境条目的值。
- **System domain entries:**
  - `Timestamp` —— 当前实时时间（Unix 时间戳，毫秒）。
  - `TipHeight` —— 区块链末端高度（交易验证时最新已确认区块）
  - `BlockTime` —— 当前区块时间戳。
- **Transaction domain entries:**
  - `TxTime` —— 交易时间戳。
  - `TxID` —— 交易标识。
  - **Note:** 当脚本作为输入源被引用时，这些条目指向的是**源交易所在区块**，不是待处理交易。
- **Validation domain entries:**
  - `Passed` —— Bool，当前通关状态（主要体现 CHECK 指令结果）。
  - `InGoto` —— Bool，表示当前是否位于 `GOTO` 目标脚本内。
  - `InEmbed` —— Bool，表示当前是否位于 `EMBED` 目标脚本内。
  - `InScript` —— Bool，表示当前是否位于 `SCRIPT` 子脚本内。
  - `InCode` —— Bool，表示当前是否位于 `CODE` block 内。
  - `Gotos` —— 截至当前 `GOTO` 进入的深度。
- **Examples:**
  ```
  ENV[Timestamp]        // 读取当前系统时间
  ENV[TxID]             // 读取源交易标识
  ENV[InGoto]           // 检查当前是否处于 GOTO 目标脚本内
  ```

### 2.2 Input Item Retrieval（输入项读取）

```
129     IN(1){}     Input item retrieval
```

- **Auxiliary params:** 1 byte —— 目标标识，指定要读取的输入条目。
- **Arguments:** 无。
- **Return value:** 指定输入条目的值。
- **Domain:** Validation domain。读取“未来花费”的输入部分信息，即引用当前正在被校验输出的那个输入。
- **Entries:**
  - `Index` —— 该输入在交易内的索引。
  - `Amount` —— 该输入携带的金额。
  - `Account` —— 输入方公钥哈希（等价于源输出中的 `Receiver` 字段：`INOUT:Receiver`）。
  - `Remarks` —— 附加备注数据。
- **Examples:**
  ```
  IN[Amount]            // 读取输入金额
  IN[Account]           // 读取输入公钥哈希
  ```

### 2.3 Output Item Retrieval（输出项读取）

```
130     OUT(~,1){}  Output item retrieval
```

- **Auxiliary params:**
  1. Varint —— 待处理交易中的输出索引（0-based）。
  2. 1 byte —— 目标标识，指定要读取的输出条目。
- **Arguments:** 无。
- **Return value:** 指定输出条目的值。
- **Domain:** Transaction domain。面向当前待处理交易的输出集合。
- **Entries:**
  - `Amount` —— 输出金额。
  - `Receiver` —— 接收方地址（公钥哈希）。
  - `Remarks` —— 备注数据。
  - `Creator` —— 创建者标识。
  - `Title` —— 标题字段。
  - `Description` —— 描述字段。
  - `Content` —— 内容字段。
  - `Count` —— 凭信转移次数。
  - `Attachment` —— 附件标识。
  - `Locked` —— 输出脚本字节内容（即锁定脚本）。
  - `Timestamp` —— 交易时间戳。
- **Examples:**
  ```
  OUT[0, Amount]        // 读取第 1 个输出的金额
  OUT[2, Receiver]      // 读取第 3 个输出的接收地址
  OUT[1, Attachment]    // 读取第 2 个输出的附件 ID
  ```

### 2.4 Source Output Item Retrieval（源输出项读取）

```
131     INOUT(~,1){}    Source output item retrieval
```

- **Auxiliary params:**
  1. Varint —— 输入所引用源交易输出在其输出集合中的位置索引（0-based）。
  2. 1 byte —— 目标标识，指定要读取的条目。
- **Arguments:** 无。
- **Return value:** 指定源输出条目的值。
- **Domain:** Transaction domain。读取输入源交易输出集合中的“同级输出”信息，即与当前输入来源输出并列存在的其他输出项。
- **Entries:** 与 `OUT` 相同（Amount、Receiver、Remarks、Creator、Title、Description、Content、Count、Attachment、Source、Timestamp）。
- **Design note:** 访问特定输出项需要系统查询来源交易，涉及额外存储访问。但交易输出集采用二元哈希树结构构造交易 ID，因此验证并不困难。
- **Examples:**
  ```
  INOUT[0, Amount]      // 读取源交易第 1 个输出金额
  INOUT[1, Timestamp]   // 通过第 2 个输出读取源交易创建时间
  ```

### 2.5 Source Script Info Retrieval（源脚本信息读取）

```
132     XFROM(1){}  Source script info retrieval
```

- **Auxiliary params:** 1 byte —— 目标标识，指定要读取的源脚本条目。
- **Arguments:** 无。
- **Return value:** 指定条目的值。
- **Domain:** Transaction domain。针对通过 `GOTO` 或 `EMBED` 加载的外部脚本，读取其来源交易信息。
- **Entries:**
  - `InSize` —— 源交易输入数量。
  - `InAmount` —— 源交易输入总额。
  - `OutSize` —— 源交易输出数量。
  - `OutAmount` —— 源交易输出总额。
  - `Timestamp` —— 源交易时间戳。
  - `Amount` —— 被引用具体输出的金额。
  - `Account` —— 被引用输出关联账户（公钥哈希）。
  - `PayType` —— 支付类型：`Coin`、`Credit` 或 `Proof`。
  - `Locked` —— 被引用输出的锁定脚本（字节内容）。
  - `Source` —— 源脚本从起点到跳转点的指令序列（不含解锁部分）。
- **Examples:**
  ```
  XFROM[PayType]        // 检查源脚本来源交易的支付类型
  XFROM[InSize]         // 读取源交易输入数量
  XFROM[Amount]         // 读取被引用输出的金额
  ```

### 2.6 Global Variable Retrieval（全局变量读取）

```
133     VAR(1)      Global variable retrieval
```

- **Auxiliary params:** 1 byte —— 变量位置索引 `[0–255]`。
- **Arguments:** 无。
- **Return value:** 指定全局变量位置里的值。
- **Domain:** Script domain。与仅追加的 block-local scope 不同，全局变量可通过 `SETVAR` 进行赋值与覆盖。
- **Sharing semantics:** 全局变量不会跨 `GOTO` 边界共享（每个 GOTO 目标拥有独立 Script Domain），但 `EMBED` 目标与调用者共享同一全局变量空间。
- **Example:**
  ```
  VAR[0]                // 读取位置 0 的全局变量
  VAR[100]              // 读取位置 100 的全局变量
  ```

### 2.7 Global Variable Assignment（全局变量赋值）

```
134     SETVAR(1)   Global variable assignment
```

- **Auxiliary params:** 1 byte —— 变量位置索引 `[0–255]`。
- **Arguments:** 任意类型的单个值。
- **Return value:** 无。
- **Domain:** Script domain。把参数值写入指定全局变量槽位，覆盖旧值。
- **Example:**
  ```
  {42} SETVAR[0]        // 将整数 42 存入全局变量槽位 0
  "hello" SETVAR[5]     // 将字符串 "hello" 存入全局变量槽位 5
  ```

### 2.8 Script Source Extraction（脚本源码提取）

```
135     SOURCE(1)   Script source extraction
```

- **Auxiliary params:** 1 byte —— 提取标识，指定要提取的源码片段。
- **Arguments:** 无。
- **Return value:** 被提取的脚本源码片段（Bytes）。
- **Domain:** Script domain。
- **Extraction identifiers:**
  - `0` —— 当前 block 的完整代码。
  - `1` —— 当前 block 已执行代码，截止并包含当前 `SOURCE` 指令。
  - `11` —— 当前 block 已执行代码，截止但不包含当前 `SOURCE` 指令。
  - `-1` —— 从当前位置到 block 末尾的后续代码（不含 `SOURCE`）。
  - `2` —— 从最近 `SYS_NULL` 点开始的已执行代码（不含 NULL 标记，含 SOURCE）。
  - `12` —— 从最近 `SYS_NULL` 点开始的已执行代码（不含 NULL 标记，也不含 SOURCE）。
  - `-2` —— 从当前位置到最近 `SYS_NULL` 点的后续代码（不含 NULL 和 SOURCE）。
- **NULL points:** 由 `SYS_NULL` 指令（opcode 169）标记，作为 `SOURCE` 片段提取边界。
- **Design note:** 外部脚本中的 SOURCE 指令只针对其脚本自身（与上级引入者无关）。
- **Examples:**
  ```
  SOURCE[0]             // 提取当前 block 完整代码
  SOURCE[1]             // 提取截至当前（含 SOURCE）的已执行代码
  SOURCE[-1]            // 提取到 block 末尾的剩余代码
  ```

### 2.9 Signature Placement Confirmation（签名位置确认）

```
136     SIGNED(1)   Signature placement confirmation
```

- **Auxiliary params:** 1 byte —— 公钥哈希列表中的目标位置（0-based）。
- **Arguments:** 无。
- **Return value:** Bool
  - 若目标位置公钥已签名且验证合法，返回 `true`。
  - 若目标位置公钥未参与签名或签名验证非法，返回 `false`。
- **Domain:** Validation domain。常用于多签验证时，检查特定参与者（公钥）是否执行签名。也可用于单签，读取签名验证结果，此时序位只能是0值。
- **Visibility:** 该指令在 `GOTO` 目标脚本中可见，使委托脚本可以检查调用方上下文的签名状态。
- **Design note:** 该指令不验证签名本身的有效性，仅读取指定位置公钥是否参与签名或签名验证的结果。主要用于按不同签名者执行不同逻辑，或定制验证中任意位置检查签名验证结果。
- **Example:**
  ```
  SIGNED[0]             // 检查单签或多签列表位置[0]是否参与签名或签名验证的结果
  SIGNED[2] PASS        // 要求位置 2 对应的参与者已签名且正确，否则失败
  ```

### 2.10 Reserved（保留）

```
137     (Reserved, unused)
```


## 3. Domain Access Summary（域访问汇总）

| Instruction | Domain | Description |
|-------------|--------|-------------|
| ENV | System / Transaction / Validation | 通用环境变量读取 |
| IN | Validation | 未来花费输入项信息 |
| OUT | Transaction | 待处理交易输出条目 |
| INOUT | Transaction | 源交易同级输出条目 |
| XFROM | Transaction | 源脚本来源交易信息（用于 GOTO/EMBED） |
| VAR | Script | 全局变量读 |
| SETVAR | Script | 全局变量写 |
| SOURCE | Script | 脚本源码片段提取 |
| SIGNED | Validation | 签名参与者位置和结果核实 |

### Design Notes（设计说明）

- **ENV** 通过单指令配合域内条目标识，统一访问 System、Transaction、Validation 三域信息。这样既保持 opcode 空间紧凑，也提供了广泛环境可见性。
- **Global variables**（VAR/SETVAR）为脚本执行提供可变状态，与仅追加的局部作用域机制互补。GOTO 隔离边界可避免独立脚本执行间的非预期状态泄漏。
- **SOURCE extraction** 支持脚本对自身指令序列进行检查和推理，可用于 verification、code-signing 等元编程场景。
