# Environment Instructions（环境指令）

## 来源构想

- `docs/conception/Instruction/13.环境指令.md`：定义 opcode `128-137`。
- `docs/proposal/Instruction/0.Base-Constraints.md`：定义环境、脚本域和公共验证边界。
- `docs/proposal/06.Transaction-Model.md`：定义交易、输入、输出和签名消息边界。
- `docs/proposal/08.UTXO-UTCO-State.md`：定义状态和 Credit 验证边界。

## 范围

环境指令包括：

- `ENV`
- `IN`
- `OUT`
- `INOUT`
- `CREDIT`
- `XFROM`
- `SIGNED`
- `VAR`
- `SETVAR`
- `SOURCE`

## 环境层级

环境层级为：

```text
系统域 > 交易域 > 校验域 > 脚本域 > 块域
```

规则：

- 系统域提供确定性链上环境和非确定私有环境；公共验证只能读取确定性字段。
- 交易域提供当前交易、输入、输出和源输出信息。
- 校验域提供当前输入验证上下文、GOTO/EMBED 状态和签名验证记录。
- 脚本域提供数据栈、实参区和全局变量。
- 块域提供局部域。

## 指令要求

- `ENV` 读取系统/交易/校验环境字段。
- `IN` 读取当前输入来源信息。
- `OUT` 读取当前交易输出项。
- `INOUT` 读取输入来源交易的兄弟输出项，成本较高。
- `CREDIT` 仅用于 Credit 输入来源的锁定脚本，否则失败。
- `XFROM` 仅用于 `GOTO` 或 `EMBED` 目标脚本。
- `SIGNED` 查询签名验证状态，依赖 `FN_CHECKSIG`、`FN_MCHECKSIG` 或 `SYS_CHKPASS` 标记。
- `VAR/SETVAR` 访问当前脚本域全局变量，`EMBED` 共享，`GOTO` 不共享。
- `SOURCE` 提取脚本片段副本，不得修改脚本。

## 风险

- `ENV{Timestamp}` 若是本机时间会破坏公共验证；应与 `SYS_TIME` 区分。
- `INOUT` 查询兄弟输出成本高。
- `SOURCE` 可用于模式验证，提取范围必须精确。
- `SIGNED` 依赖签名验证状态，必须防止未验证即读取为 true。

## 未决问题

- `ENV{Source0}`、`ENV{Source1}` 等示例字段是否正式化。
- `ENV{Timestamp}` 是节点真实时间还是确定性链上时间。
- `BlockTime`、`Height` 对“当前即将创建区块”的定义。
- `IN{Unlock}` 是否包含签名数据。
- 多签 `SIGNED` 序位、补全集和失败记录规则。
- `SOURCE` 标识位和跨块提取规则。

## 对后续 Plan 的影响

- 环境字段必须有注册表，包含类型、确定性、可用域、成本和错误规则。
- 公共模式测试必须拒绝读取非确定系统字段。
