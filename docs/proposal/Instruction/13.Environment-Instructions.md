# Environment Instructions（环境指令）

Status: Draft proposal（状态：提案草稿）
Source conception: `conception/Instruction/13.环境指令.md`

## Prerequisites（前置条件）

关于 stack model、local frame、opcode 布局等基础约束，请参见 `0.Base-Constraints.md`。

核心回顾：

- **data stack** 与 **argument zone** 归属于 script scope；child script 拥有自己的 data stack 与 argument zone。
- **local frame** 是 block-local 的，不在同级或嵌套 block 间共享；**global frame** 是 script-scoped 的。
- Immediate operand 是编译期选择器，associated data 与 opcode 一并编码；runtime argument 按基础规则来自 argument zone 或 data stack。

## Overview（概述）

环境指令向脚本暴露确定性的运行时上下文。它们从 system、transaction、verification、script、block 等 scope 读取值，不会修改 transaction 本身或 script source。该族中多数 opcode 不接收 runtime argument；`SETVAR` 是例外，因为它会把一个 runtime 值写入当前 script-scoped 的全局变量表。

该指令族涵盖五类：

- 环境与交易查询：`ENV`、`IN`、`OUT`、`INOUT`、`XFROM`；
- Credit 专用的来源与转移查询：`CREDIT`；
- 已验证签名槽检查：`SIGNED`；
- script-scoped 全局变量：`VAR`、`SETVAR`；
- 脚本源码片段提取：`SOURCE`。

关键环境数据 SHOULD 可供 middleware 获取，以便其观测系统状态并取得确定性值，尤其是 `ENV`、`IN`、`OUT`、`INOUT` 暴露的成员值。

## Scope hierarchy（作用域层级）

脚本执行运行在嵌套环境中。前一层 scope 是后一层 scope 的父级：

```text
System domain > transaction domain > verification domain > script domain > block domain
```

1. **System domain**：节点运行环境，以及节点可提供的系统级数据。
2. **Transaction domain**：当前被校验的交易，包括 input 集、output 集，以及 input 对应的 source-output 信息。
3. **Verification domain**：当前 output 的验证上下文，包含由 `GOTO` 引入脚本的验证状态。
4. **Script domain**：顶层脚本及由 `GOTO`、`SCRIPT` 或其他脚本加载机制引入脚本的环境。
5. **Block domain**：语法层面的 block-local 作用域，包含顶层与嵌套子 block；局部 domain 彼此不可见。

data stack 与 argument zone 归属于 script domain。由于 script domain 可以嵌套，child script 同样拥有自己的 data stack 与 argument zone。这一点会影响 `VAR` / `SETVAR`、`SOURCE`，以及通过 `GOTO` 跳入或通过 `EMBED` 嵌入的脚本行为。

## Instruction Catalog（指令目录）

```go
Code   Mnemonic       Form
-----------------------------------------------------------
128    ENV(1){}       读取系统、交易或验证域字段
129    IN(1){}        读取当前输入字段
130    OUT(~,1){}     读取当前交易输出字段
131    INOUT(~,1){}   读取输入来源交易中的同级输出字段
132    CREDIT(1){}    读取 Credit 专用来源字段或转移目标字段
133    XFROM(1){}     从 GOTO/EMBED 目标读取来源交易数据
134    SIGNED(1)      检查签名槽是否已通过验证
135    VAR(1)         读取 script-scoped 全局变量
136    SETVAR(1)      写入 script-scoped 全局变量
137    SOURCE(1)      提取脚本源码片段
```

### 128 — `ENV(1){}`

- **Type**：环境查询，覆盖 system / transaction / verification scope。
- **Operands**：
  - **Immediate**：1 byte 目标选择器。
  - **Associated data**：可选 `{}` 形式符号选择器名，是 immediate selector 的别名。
  - **Runtime args**：无。
- **Stack effect**：`-> Any`。
- **Description**：读取一个环境项并将其值压栈。以下名称在本提案中是规范定义，但具体实现可以在其基础上扩展。

  System domain 字段：

  | Field | Meaning |
  |-------|---------|
  | `Timestamp` | 当前真实时间戳，单位毫秒。 |
  | `TipHeight` | 区块链 tip 高度；在交易验证期间表示最新已确认区块高度。 |
  | `BlockTime` | 当前区块 Unix 毫秒时间戳。当前区块指尚未创建、将打包该交易的区块。 |
  | `Height` | 当前区块高度，与 `BlockTime` 严格对应；`TipHeight + 1 = Height`。 |

  Transaction domain 字段：

  | Field | Meaning |
  |-------|---------|
  | `TxID` | 当前交易 ID。 |
  | `TxTime` | 当前交易 Unix 毫秒时间戳。 |

  Verification domain 字段：

  | Field | Meaning |
  |-------|---------|
  | `Passed` | 当前通过状态（`true` / `false`）。 |
  | `InGoto` | 当前执行是否位于由 `GOTO` 进入的脚本内。 |
  | `InEmbed` | 当前执行是否位于由 `EMBED` 进入的脚本内。 |
  | `InScript` | 当前执行是否位于由 `SCRIPT` 引入的脚本内。 |
  | `InCode` | 当前执行是否位于由 `CODE` 定义的脚本内。 |
  | `Gotos` | 到当前执行位置为止的 `GOTO` 跳转深度。 |

  ```go
  // 如果当前区块高度大于 9999，则通过验证。
  ENV{Height} {9999} GT PASS
  ```

  在需要时，实现可额外暴露 `Locked`、`Unlock`、`Source` 以便进行模式校验。
- **Errors-Constraints**：未知 selector；当前执行模式下环境字段不可用；实现 MUST 保证返回值在共识相关执行中具备确定性。

### 129 — `IN(1){}`

- **Type**：交易输入查询，verification scope。
- **Operands**：
  - **Immediate**：1 byte 输入字段选择器。
  - **Associated data**：可选 `{}` 形式符号选择器名。
  - **Runtime args**：无。
- **Stack effect**：`-> Any`。
- **Description**：读取当前正在验证的输入字段。站在 lock script 的源位置看，这些值会出现在未来花费交易的 input 部分。

  Input 字段：

  | Field | Meaning |
  |-------|---------|
  | `Timestamp` | 输入来源交易的 Unix 毫秒时间戳。 |
  | `TxID` | 该输入的来源交易 ID。 |
  | `Height` | 包含该来源交易的区块高度。 |
  | `Amount` | 输入中的 Coin 金额。 |
  | `Account` | 输入账户，以公钥哈希表示。 |
  | `Remarks` | 附加在输入账户上的备注。 |
  | `Locked` | 锁定脚本序列。 |
  | `Unlock` | 解锁脚本序列，通常不含签名。 |
  | `Source` | 脚本源码，由 unlock script 与 lock script 组合而成。 |
  | `Index` | 该输入在交易 input 集中的序号。 |
- **Errors-Constraints**：未知字段选择器；当前验证上下文中无活动输入；请求的来源数据不可用。

### 130 — `OUT(~,1){}`

- **Type**：交易输出查询，transaction scope。
- **Operands**：
  - **Immediate**：varint 表示输出索引（从 `0` 开始）+ 1 byte 字段选择器。
  - **Associated data**：可选 `{}` 形式符号选择器名。
  - **Runtime args**：无。
- **Stack effect**：`-> Any`。
- **Description**：读取当前被验证交易的 output 集字段。它使 lock script 能约束未来花费者的输出，比如定向转账或遗嘱式规则。

  Output 字段：

  | Field | Meaning |
  |-------|---------|
  | `Amount` | Coin 金额。 |
  | `Receiver` | 接收方。 |
  | `Remarks` | 支付备注。 |
  | `Creator` | Credit / proof 创建者。 |
  | `Title` | Credit / proof 标题。 |
  | `Description` | Credit 描述。 |
  | `Content` | Proof 内容。 |
  | `Count` | Credit 转移计数。 |
  | `Attachment` | 附件 ID。 |
  | `Locked` | 以 bytes 表示的输出脚本。 |

  ```go
  // 当前交易第一个输出的 Coin 金额。
  OUT{0, Amount}
  ```
- **Errors-Constraints**：输出索引越界；未知字段选择器；该输出类型不携带所请求字段。

### 131 — `INOUT(~,1){}`

- **Type**：来源交易同级输出查询，transaction scope。
- **Operands**：
  - **Immediate**：varint 表示同级输出索引（从 `0` 开始）+ 1 byte 字段选择器。
  - **Associated data**：可选 `{}` 形式符号选择器名。
  - **Runtime args**：无。
- **Stack effect**：`-> Any`。
- **Description**：读取当前输入的来源交易 output 集字段。目标输出是该来源交易中的同级输出。

  字段与 `OUT` 相同：

  | Field | Meaning |
  |-------|---------|
  | `Amount` | Coin 金额。 |
  | `Receiver` | 接收方。 |
  | `Remarks` | 支付备注。 |
  | `Creator` | Credit / proof 创建者。 |
  | `Title` | Credit / proof 标题。 |
  | `Description` | Credit 描述。 |
  | `Content` | Proof 内容。 |
  | `Count` | Credit 转移计数。 |
  | `Attachment` | 附件 ID。 |
  | `Locked` | 以 bytes 表示的输出脚本。 |

  该 opcode 的资源开销相对较高，因为系统必须查询或加载输入来源交易中的同级输出。来源构想中指出，交易 output 集被设计为二叉 hash-tree 结构用于构建 transaction ID。
- **Errors-Constraints**：来源交易不可用；同级输出索引越界；未知字段选择器；加载来源输出时资源预算超限。

### 132 — `CREDIT(1){}`

- **Type**：Credit 专用字段查询，verification scope。
- **Operands**：
  - **Immediate**：1 byte Credit 字段选择器。
  - **Associated data**：可选 `{}` 形式符号选择器名。
  - **Runtime args**：无。
- **Stack effect**：`-> Any`。
- **Description**：读取与 Credit 输入来源交易相关的专用信息。其逻辑类似 `INOUT`，但仅限于 Credit 输入的来源输出，以及该 Credit 的转移目标。

  来源 Credit 字段：

  | Field | Meaning |
  |-------|---------|
  | `Creator` | Credit 创建者。 |
  | `Receiver` | Credit 接收者。 |
  | `Title` | Credit 标题。 |
  | `Description` | Credit 描述。 |
  | `Config` | Credit 配置，2 bytes。 |
  | `IsNew` | Credit 是否为新建。 |
  | `Modifiable` | Credit 是否可修改。 |
  | `Modified` | Credit 在当前转移中是否已修改。 |
  | `Count` | Credit 转移计数。 |
  | `Deadline` | Credit 高度截止条件。 |
  | `OutIndex` | Credit 被转移到的输出索引。 |
  | `Attachment` | 附件 ID。 |
  | `Locked` | 以 bytes 表示的锁定脚本。 |

  转移目标字段：

  | Field | Meaning |
  |-------|---------|
  | `ToReceiver` | 目标输出的接收者。 |
  | `ToDescription` | 目标输出的 Credit 描述。 |
  | `ToConfig` | 目标输出的 Credit 配置，2 bytes。 |
  | `ToModifiable` | 目标输出 Credit 是否可修改。 |
  | `ToLocked` | 以 bytes 表示的目标输出锁定脚本。 |

  该 opcode 主要用于采集 Credit 逻辑脚本所需数据。
- **Errors-Constraints**：该 opcode 仅在 Credit 输入的 lock script 内有效；在其他位置使用会导致验证失败；未知字段选择器或 Credit 转移数据格式错误也会失败。

### 133 — `XFROM(1){}`

- **Type**：目标脚本外部来源查询，transaction scope。
- **Operands**：
  - **Immediate**：1 byte 来源字段选择器。
  - **Associated data**：可选 `{}` 形式符号选择器名。
  - **Runtime args**：无。
- **Stack effect**：`-> Any`。
- **Description**：允许外部跳入脚本检查跳入它的来源交易。例如，`GOTO` 目标脚本可能需要上层执行跳转的来源交易数据。

  来源交易字段：

  | Field | Meaning |
  |-------|---------|
  | `InSize` | 来源交易的输入数量。 |
  | `InAmount` | 来源交易输入总金额。 |
  | `OutSize` | 来源交易的输出数量。 |
  | `OutAmount` | 来源交易输出总金额。 |
  | `Account` | 来源脚本账户，以公钥哈希表示。 |
  | `Amount` | 来源脚本花费的 Coin 金额。 |
  | `PayType` | 来源脚本的花费类型：coin、credit 或 proof。 |
  | `Locked` | 以 bytes 表示的来源脚本 lock script。 |
  | `Source` | 来源脚本指令序列，范围从 `Locked` 开始到跳转点。 |
  | `Timestamp` | 来源交易创建时间戳。 |

  这里的 **source transaction** 指的是：当前被验证、且在花费一个输出后跳入该外部目标的那笔交易。
- **Errors-Constraints**：仅在 `GOTO` 或 `EMBED` 目标脚本内有效；在此类目标之外使用会验证失败；未知选择器或来源交易上下文缺失也会失败。

### 134 — `SIGNED(1)`

- **Type**：签名槽验证查询，verification scope。
- **Operands**：
  - **Immediate**：1 byte 签名槽选择器。
  - **Associated data**：无。
  - **Runtime args**：无。
- **Stack effect**：`-> Bool`。
- **Description**：检查目标签名槽上的公钥是否参与签名，且签名验证已通过。单签场景仅允许槽位 `0`；多签场景下，槽位是公钥哈希列表中的从 `0` 开始序号。

  返回值：

  | Condition | Result |
  |-----------|--------|
  | 目标槽位已签名且签名验证成功。 | `true` |
  | 目标槽位未签名，或签名验证失败。 | `false` |

  其主要用途是区分签名者后进行分支或决策。在多签脚本中最常见，也可用于自定义验证流程：脚本调用 `FN_CHECKSIG` / `FN_MCHKSIGN` 后不立即 `PASS`。

  ```go
  // 如果多签集合中的第二个槽位已签名，则通过验证。
  SIGNED[1] PASS

  // 替代模式：三个槽位中至少需要两个签名。
  SIGNED[0] SIGNED[1] SIGNED[2] SHIFT[3] SOME[2] PASS
  ```

  `SIGNED` 属于 verification scope，因此在通过 `GOTO` 到达的脚本中也可见。

  `FN_CHECKSIG`、`FN_MCHKSIGN`、`SYS_CHKPASS` 会在系统层记录签名验证状态。`SYS_CHKPASS` 仅记录成功状态，因为失败会终止脚本且不会授予通过。
- **Errors-Constraints**：单签模式使用了非 `0` 的非法槽位；多签槽位越界；当前验证上下文中签名状态表不可用。

### 135 — `VAR(1)`

- **Type**：script-scoped 全局变量读取。
- **Operands**：
  - **Immediate**：1 byte 全局变量索引，范围 `[0, 255]`。
  - **Associated data**：无。
  - **Runtime args**：无。
- **Stack effect**：`-> Any`。
- **Description**：从当前 script domain 的全局变量表读取一个值，压入 data stack；若使用 capture form，则追加到 argument zone。

  ```go
  // 将全局变量槽位 0 读入 argument zone。
  @VAR[0]

  // 将全局变量槽位 10 读入 data stack。
  VAR[10]
  ```

  与 local frame 不同，这是一套可变变量机制，值可被 `SETVAR` 覆盖。该 opcode 属于 script scope，不跨 `GOTO`。通过 `EMBED` 嵌入的脚本共享同一环境，因此可经共享全局变量表相互影响。
- **Errors-Constraints**：索引超出 `[0, 255]`；若实现将未设置全局变量视为非法而非 `nil`，读取未设置槽位会失败。

### 136 — `SETVAR(1)`

- **Type**：script-scoped 全局变量写入。
- **Operands**：
  - **Immediate**：1 byte 全局变量索引，范围 `[0, 255]`。
  - **Associated data**：无。
  - **Runtime args**：任意类型的一个值。
- **Stack effect**：`Any ->`。
- **Description**：将一个 runtime 值写入当前 script domain 的全局变量表，与 `VAR` 配套使用。

  ```go
  // 将栈顶值 123 存入全局变量槽位 1。
  {123} SETVAR[1]

  // 将浮点数 1.5 存入全局变量槽位 255。
  {1.5} SETVAR[255]

  // 将先前存储的 123 读入当前 local frame。
  $VAR[1]
  ```

  赋值会覆盖同槽位旧值。其作用域限于当前 script domain，不跨 `GOTO`；由于 `EMBED` 共享同一环境，`EMBED` 参与方共享该变量表。
- **Errors-Constraints**：索引超出 `[0, 255]`；缺失 runtime argument；值超出实现对栈项或变量存储的限制。

### 137 — `SOURCE(1)`

- **Type**：脚本源码片段提取，script scope。
- **Operands**：
  - **Immediate**：1 byte 源码片段掩码。
  - **Associated data**：无。
  - **Runtime args**：无。
- **Stack effect**：`-> Bytes`。
- **Description**：返回脚本源码指令序列的一个副本。Immediate mask 控制提取是 block-local 还是 script-level，以及提取完整源码、已执行前缀、自最近 `NULL` 标记后的已执行前缀，还是剩余后缀。提取前缀时包含 `SOURCE` 指令本身；提取后缀时不包含 `SOURCE` 指令本身。提取结果永不包含 `NULL` 标记本身。

  源码掩码：

  | Mask | Scope | Fragment |
  |------|-------|----------|
  | `0x00` | current block | 当前 block 的完整代码。 |
  | `0x01` | current block | 包含 `SOURCE` 的已执行前缀。 |
  | `0x02` | current block | 自最近 `NULL` 标记以来的已执行前缀，不含该标记。 |
  | `0x04` | current block | 当前 block 剩余后缀直到结尾。 |
  | `0x80` | script-level | 含 unlock 部分的完整脚本。 |
  | `0x81` | script-level | script-level 已执行前缀，包含 unlock script。 |
  | `0x82` | script-level | script-level 自最近 `NULL` 标记以来的已执行前缀，不含该标记。 |
  | `0x84` | script-level | script-level 剩余后缀直到脚本结尾。 |

  附加规则：

  - 脚本的顶层 block domain 包含 unlock script。
  - 外部脚本中的 `SOURCE` 仅作用于该脚本自身，与引入它的上层脚本无关。
  - `NULL` 标记由 `SYS_NULL` 写入。
  - 在所有提取模式中，`NULL` 标记本身都被排除。

  示例：

  ```go
  DATA{556e6c6f636b}  // 假设这里是全部 unlock 数据。
  SYS_NULL            // NULL.1
  ENV{Height} {<Height>}
  GT
  IF{
      SIGNED[1]
      SYS_NULL        // NULL.2
      PASS
      <Source.1>      // 提取点 1
      NOP
      EXIT
  }
  <Source.2>          // 提取点 2
  SIGNED[0] CHECK END
  ```

  `<Source.1>` 处结果：

  | Invocation | Extracted fragment |
  |------------|--------------------|
  | `SOURCE` | `SIGNED` 到 `EXIT`。 |
  | `SOURCE[0]` | 与 `SOURCE` 相同：当前 block 全段，即 `SIGNED` 到 `EXIT`。 |
  | `SOURCE[1]` | `SIGNED` 到 `<Source.1>`。 |
  | `SOURCE[2]` | `PASS` 到 `<Source.1>`，不含 `NULL.2`。 |
  | `SOURCE[4]` | `NOP` 到 `EXIT`。 |
  | `SOURCE[0x80]` | `DATA` 到 `END`，即完整脚本序列。 |
  | `SOURCE[0x81]` | `DATA` 到 `<Source.1>`。 |
  | `SOURCE[0x82]` | `ENV` 到 `<Source.1>`，不含 `NULL.1`。 |
  | `SOURCE[0x84]` | `NOP` 到 `END`。 |

  `<Source.2>` 处结果：

  | Invocation | Extracted fragment |
  |------------|--------------------|
  | `SOURCE[0]` | 在 block 逻辑下，`DATA` 到 `END` 的完整脚本序列。 |
  | `SOURCE[1]` | `DATA` 到 `<Source.2>`。 |
  | `SOURCE[2]` | `ENV` 到 `<Source.2>`。 |
  | `SOURCE[0x82]` | `PASS` 到 `<Source.2>`；此时 block 已被剖分，取局部片段。 |

  设计限制：`SOURCE` 不支持提取“未来未执行后缀且以 `NULL` 标记为结束点”的模式。若支持该模式，需要额外的前瞻扫描。

  实现要求：引擎需要两个 `NULL` 游标：

  - block-local cursor：存储在各 block domain 中，block 结束即丢弃；
  - script-global cursor：从 unlock script 头部开始，记录并更新最新 script-level `NULL` 标记位置。
- **Errors-Constraints**：未知 mask；当前 script domain 中无法寻址目标源码片段；提取会超出字节大小或资源限制。返回 bytes MUST 是副本，而不是 script source 的可变视图。

## Design notes（设计说明）

- **Read-only source model**：环境指令只检查执行上下文与脚本源码，不会修改链上不可变源码。
- **Scope-sensitive visibility**：`ENV` 覆盖 system、transaction、verification；`IN`、`OUT`、`INOUT` 面向交易；`VAR`、`SETVAR`、`SOURCE` 属于 script scope；源码提取与 local frame 受 block 局部性影响。
- **Source-output loading cost**：`INOUT` 有意比 `OUT` 更昂贵，因为它可能需要加载当前输入来源交易中的同级输出。
- **Credit isolation**：`CREDIT` 被刻意限制在 Credit 输入的 lock script 中，使 Credit 转移逻辑可暴露细粒度字段，同时不削弱非 Credit 路径的验证。
- **External-target introspection**：`XFROM` 让 `GOTO` / `EMBED` 目标脚本可访问调用它们的交易上下文，但在其他位置无效。
- **Signature-slot state**：`SIGNED` 本身不是签名验证器；它读取的是 `FN_CHECKSIG`、`FN_MCHKSIGN`、`SYS_CHKPASS` 记录的状态。
- **Variable sharing boundary**：script-global 变量不跨 `GOTO`；而 `EMBED` 共享同一环境，因此可通过 `VAR` / `SETVAR` 产生相互副作用。
- **Two `NULL` cursors**：`SOURCE` 需要同时跟踪 block-local 与 script-global 两类 `NULL` 标记，才能区分局部 block 片段与完整脚本片段。
