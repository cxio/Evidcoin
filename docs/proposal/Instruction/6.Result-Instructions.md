# Result Instructions（结果指令）

## 来源构想

- `docs/conception/Instruction/6.结果指令.md`：定义 opcode `51-57`。
- `docs/proposal/09.Script-System.md`：定义执行状态、GOTO/EMBED/END 边界。
- `docs/proposal/Instruction/0.Base-Constraints.md`：定义错误和 pass 状态基础。

## 范围

结果指令包括：

- `PASS`
- `CHECK`
- `GOTO`
- `EMBED`
- `EXIT`
- `RETURN`
- `END`

## 规范要求

- `PASS true` 设置通关状态并继续执行。
- `PASS false` 失败并结束脚本。
- `CHECK` 设置或覆盖通关状态但不结束。
- `GOTO` 跳转到外部输出脚本，在独立环境中执行，传入实参区数据作为初始数据栈，不返回栈残留。
- `EMBED` 嵌入外部输出脚本，在当前环境中执行，共享数据栈、实参区和全局域。
- `EXIT` 结束脚本，并按当前 pass 状态决定验证结果。
- `RETURN` 仅用于 `MAP/FILTER` 等函数式子块。
- `END` 结束公共验证；私有节点可忽略并继续执行后续私有逻辑。

## 外部脚本限制

- `GOTO` 顶层跳转次数 `<= 2`。
- `GOTO` 跳转深度 `<= 3`。
- `EMBED` 次数 `<= 4`。
- `EMBED` 深度 `== 0`，嵌入脚本不可再包含 `EMBED` 或 `GOTO`。
- 外部脚本获取失败必须导致确定性失败或验证延迟策略，不能默默通过。

## 风险

- `GOTO/EMBED` 循环引用和递归。
- `EMBED` 共享环境副作用强。
- `CHECK` 覆盖 pass 状态可能被误用为 OR 逻辑。
- `END` 后私有错误是否影响公共结果必须固定。

## 未决问题

- 初始 pass 状态。
- `CHECK` 是否允许 true 覆盖为 false，或 false 覆盖为 true。
- `EXIT` 返回值是否被系统处理。
- GOTO 目标脚本中的 `END` 如何传播到顶层状态。
- 外部脚本不可得时是否允许低优先级或延迟验证。

## 对后续 Plan 的影响

- 结果指令测试必须覆盖 pass 状态机、GOTO 环境隔离、EMBED 副作用、深度限制和 END 公私模式差异。
