# Module Instructions（模块指令）

Status: 草案提案
Source conception: `conception/Instruction/17.模块指令.md`

## Prerequisites（前置条件）

关于栈模型、本地帧、opcode 布局、即时操作数记法 `(n)` 以及关联数据记法 `{}`，
请参见 `0.Base-Constraints.md`。

相关概念回顾：

- 模块 opcode 使用 `MO_` 前缀。
- 执行模块 opcode 会调用其构造器，并将 **module object** 压入数据栈；
  模块暴露的方法随后统一通过通用 `CALL` opcode 分发。
- 模块构造器与其他 opcode 一样，可以接收运行时参数。
- **Convention.** 当模块方法返回多个值时，结果会被打包为一个 slice ——
  模块方法返回值不进行 auto-spread。

## Overview（概览）

模块区块将主题相关的能力归组到具名模块对象之后。构造器 opcode 会生成一个
不透明的模块对象值；方法分发统一通过 `CALL` 完成，方法名以字符串提供，
调用参数个数由 `CALL` 的 immediate 携带。这里先定义两个基础模块
（`MO_MATH`、`MO_FMT`），随后是按字节索引的标准模块入口（`MO_XX`），
可寻址最多 256 个标准模块。

## Opcode Range（操作码范围）

| Range     | Count | Notes                                                   |
|-----------|-------|---------------------------------------------------------|
| `225..226`| 2     | 基础模块（`MO_MATH`、`MO_FMT`）                         |
| `227..249`| 23    | 未使用                                                  |
| `250`     | 1     | `MO_XX` —— 标准模块引用（256 个子索引）                 |
| **Total** | **26**| 区块 `[225..250]`                                       |

## Instruction Catalog（指令目录）

```go
Code   Mnemonic       Form
-----------------------------------------------------------
225    MO_MATH                 数学模块（构造器）
226    MO_FMT                  格式化模块（Go fmt 子集）
227-249                        （未使用）
250    MO_XX(1){}              标准模块引用
```

### 225 — `MO_MATH`

- **Type**: module constructor，数学运算
- **Operands**: none（构造器不接收运行时参数）
- **Stack effect**: `-> Module`
- **Description**: 构造 math 模块对象——提供一组基础算术区块未覆盖的
  高阶算术操作。该对象只能通过 `CALL` 使用。暴露接口固定了方法集合
  （因此也构成模块契约）。成员包括（非穷尽）：
  ```go
  Abs  (x float64) float64
  Ceil (x float64) float64
  Floor(x float64) float64
  Pow  (x, y float64) float64
  Max  (x, y float64) float64
  Min  (x, y float64) float64
  Mod  (x, y float64) float64
  // ...
  ```
- **Examples**:
  ```go
  MO_MATH "Abs" {-100} CALL[1]      // => 100.0
  MO_MATH "Max" {-1} {99} CALL[2]   // => 99.0
  ```
- **Errors**: `CALL` 中方法名未知；参数形状不匹配。

### 226 — `MO_FMT`

- **Type**: module constructor，数据格式化（Go `fmt` 子集）
- **Operands**: none
- **Stack effect**: `-> Module`
- **Description**: 构造 formatting 模块——对 Go `fmt` 标准库中选定成员的
  轻量封装。暴露成员：
  ```go
  Sprint  (a ...any) string
  Sprintf (format string, a ...any) string
  Sprintln(a ...any) string
  ```
- **Errors**: `CALL` 中方法名未知；format/argument 不匹配会以底层调用的
  运行时错误形式抛出。

### 227–249

未使用。保留用于当前模块活动范围内未来的基础模块。

### 250 — `MO_XX(1){}`

- **Type**: module constructor，分发到标准模块集合
- **Immediate**: 1 byte 子模块索引（花括号形式可接受模块符号名作为别名）
- **Operands**: 由解析后的目标模块构造器定义
- **Stack effect**: `-> Module`
- **Description**: 加载给定索引对应的标准子模块。标准模块由协议作者定义并
  发布，在单字节索引空间内寻址（最多 256 个模块）。若模块空间超出 256，
  参见 `18.Extension-Instructions.md` 中的 `EXT_MO`，其提供 64K 槽位的
  扩展模块地址空间。
- **Examples**:
  ```go
  MO_XX{crypto}     // 加载标准 "crypto" 模块
  ```
- **Errors**: 子模块索引未定义。

## Design Notes（设计说明）

- **Constructor + `CALL` dispatch.** 将模块构造与方法调用拆分，可保持
  opcode 空间紧凑（按模块占 1 byte，而非按方法占 1 byte），同时允许各模块
  的方法接口独立演进。传给 `CALL` 的方法名是模块契约定义的字符串标识符。
- **No auto-spread for module returns.** 多返回值方法结果会打包为一个
  slice。这里有意偏离引擎全局 auto-spread 规则，以保持模块 ABI 跨版本稳定：
  调用方在需要单个值时显式解构，且方法返回元组新增项不会静默重排下游栈。
- **Module-object opacity.** 模块对象值是 *opaque*；其唯一合法消费方是
  `CALL`。可以将其存入 `VAR` 或经通用操作符传递，但都按不透明 payload 处理。
- **Standard-set sizing.** 单字节索引将标准模块集合上限设为 256 项；
  进一步扩展使用 `EXT_MO` 的 16-bit 地址空间。
