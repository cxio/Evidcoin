# Proposal: Module Instructions [210–249]（模块指令）

> **Prerequisite:** 本规范建立在 [Base Constraints](0.Base-Constraints.md) 文档之上，该文档定义了全局使用的指令格式、运行时模型与记号约定。


## 1. Overview（概览）

模块指令占用操作码区间 **[210–249]**（40 条）。模块提供 **categorized functionality**：每个模块在单一 opcode 下聚合一组相关操作，并通过 1-byte 辅助参数选择具体成员函数。

### Module Design Pattern（模块设计模式）

每条模块指令遵循统一模式：

- **Opcode** 标识模块本体。
- **Auxiliary param**（1 byte）在模块内索引成员函数，每个模块最多支持 256 个成员。
- 若模块需要创建 **object representing itself**，首个成员（`Create`，index 0）作为构造器；后续成员函数将该对象作为第一个参数。
- 模块成员函数返回多个值时，被封装为一个切片，不自动展开。

### Notation Conventions（记法约定）

模块成员可使用两种等价记法：

| Notation | Example | Description |
|----------|---------|-------------|
| Brace syntax | `MO_MATH{Abs}` | 在花括号中写成员名 |
| Dot syntax | `MO_MATH.Abs` | 用点号连接模块与成员名 |

两种记法在脚本源码中可互换，最终编译为相同 bytecode。

> **Note:** 本节尚未完全定稿。下列成员列表为初步设计，后续会随系统成熟继续细化与扩展。


## 2. Instruction Definitions（指令定义）

### 2.1 Math Module（数学模块）

```
210     MO_MATH(1){}    Math operations module
```

- **Auxiliary params:** 1 byte——成员函数标识。
- **Members:**

  | Name | Signature | Description |
  |------|-----------|-------------|
  | Abs | `(x float64) → float64` | 绝对值 |
  | Ceil | `(x float64) → float64` | 向上取整到最近整数 |
  | Floor | `(x float64) → float64` | 向下取整到最近整数 |
  | Pow | `(x, y float64) → float64` | 计算 `x` 的 `y` 次幂 |
  | Max | `(x, y float64) → float64` | 两值中较大者 |
  | Min | `(x, y float64) → float64` | 两值中较小者 |
  | Mod | `(x, y float64) → float64` | 浮点余数 |
  | ... | — | 其余成员待定 |

- **Examples:**
  ```
  {-3.14} MO_MATH{Abs}       // → 3.14
  {2.0} {10.0} MO_MATH{Pow}  // → 1024.0
  {3.7} MO_MATH.Ceil         // → 4.0
  ```
- **Design notes:**
  - Math 模块没有 `Create` 构造器——所有成员均为无状态函数。
  - 对齐 Go `math` package 的子集，优先覆盖链上计算常见需求。

### 2.2 Time Module（时间模块）

```
211     MO_TIME(1){}    Time module
```

- **Auxiliary params:** 1 byte——成员函数标识。
- **Members:**

  | Name | Signature | Description |
  |------|-----------|-------------|
  | Create | `(sec int64) → Time` | 基于 UNIX timestamp（毫秒）构造 Time 对象 |
  | Now | `() → Time` | 当前时间（共识上下文需 deterministic） |
  | Add | `(tm Time, d int64) → Time` | 对时间增加时长（毫秒） |
  | Clock | `(tm Time) → (hour, min, sec int)` | 提取 [时、分、秒] |
  | Date | `(tm Time) → (year, month, day int)` | 提取 [年、月、日] |
  | ... | — | 其余成员待定 |

- **Examples:**
  ```
  {1608883593} MO_TIME{Create}     // 通过 UNIX timestamp 构造 Time
  MO_TIME.Now                      // 获取当前时间
  <tm> {3600000} MO_TIME{Add}      // 增加 1 小时（毫秒）
  <tm> MO_TIME.Clock               // → [hour, min, sec]（三个值的切片）
  ```
- **Design notes:**
  - `Create`（index 0）是构造器，产生可供其他成员操作的 `Time` 对象。
  - `Clock`、`Date` 等多个返回成员会被封装为一个切片。
  - 注意 `Now` 返回的值应当不用于共识场景，但如何使用由用户自己控制。

### 2.3 Regular Expression Module（正则表达式模块）

```
212     MO_RE(1){}      Regular expression module
```

- **Auxiliary params:** 1 byte——成员函数标识。
- **Members:**

  | Name | Signature | Description |
  |------|-----------|-------------|
  | Create | `(str string) → *Regexp` | 将正则模式字符串编译为 Regexp 对象 |
  | Find | `(re *Regexp, b []byte) → []byte` | 在字节切片中查找首个匹配 |
  | FindAll | `(re *Regexp, b []byte, n int) → [][]byte` | 查找全部匹配（最多 `n` 个；`-1` 表示无限） |
  | Split | `(re *Regexp, s string, n int) → []string` | 按模式拆分字符串（最多 `n` 段；`-1` 表示无限） |
  | String | `(re *Regexp) → string` | 返回原始模式字符串 |
  | ... | — | 其余成员待定 |

- **Examples:**
  ```
  "^[A-Z]{2}\\d+" MO_RE{Create}         // 编译 regex
  <re> <data> MO_RE.Find                // 查找首个匹配
  <re> <data> {-1} MO_RE{FindAll}       // 查找全部匹配
  <re> "a-b-c-d" {-1} MO_RE{Split}      // → ["a", "b", "c", "d"]
  ```
- **Design notes:**
  - 使用 RE2 引擎（线性时间保证、无回溯），与 Go `regexp` package 一致，可防止病态模式导致的拒绝服务。
  - `Create` 在编译失败时会立即终止执行。

### 2.4 Reserved (213–248)（保留）

操作码 **213–248**（36 个槽位）保留给未来模块扩展，当前未分配，实现中不得使用。

未来模块候选包括但不限于：字符串工具、密码学原语、数据序列化、集合算法。


## 3. Instruction Definition — Standard Module Extension（指令定义——标准模块扩展）

### 3.1 Extended Module Reference（扩展模块引用）

```
249     MO_X(1){}   Extended module reference
```

- **Auxiliary params:** 1 byte——目标子模块索引（`[0–255]`）。
- **Associated data:** 可选“目标名称+成员名”（使用 `{}` 记法时）。
- **Arguments:** 由目标子模块成员函数决定。
- **Return value:** 由目标子模块成员函数决定。
- **Design notes:**
  - 作为扩展模块表的分发网关，最多支持 **256** 个子模块槽位。
  - 每个子模块使用自身 1-byte 辅助参数管理成员函数，模式与内建模块一致。
  - `{}` 记法支持点式成员访问：`MO_X{Exp.Create}` 指向扩展模块 `Exp` 的 `Create` 成员。
  - 当需超过 256 个扩展模块时，使用扩展指令中的 `EX_MO`（opcode 251），其提供 2-byte（64K）地址空间。
- **Example:**
  ```
  <args...> MO_X{Crypto.Sign}    // 调用扩展模块 "Crypto" 的 Sign 成员
  <args...> MO_X[5]              // 调用索引 5 的扩展模块（默认成员）
  ```


## 4. Design Notes（设计说明）

### Object-Oriented Pattern（面向对象模式）

通过 `Create` 产出对象的模块，等效于在栈式脚本系统中引入轻量面向对象模式。对象在后续成员调用时作为第一个参数传入，模拟 method invocation，而无需完整 class 体系。

### Determinism Requirement（确定性要求）

所有模块成员函数在相同输入下都必须给出 **deterministic results**。这是共识关键约束——任何非确定性行为（如系统随机数、wall-clock 时间）都会导致节点间校验分叉。

### Extensibility（可扩展性）

保留区间（213–248）提供了充足增长空间。`MO_X` 进一步扩展至 256 个子模块，每个最多 256 成员——在需要 `EX_MO` 之前，扩展成员总容量可达 65,536。
