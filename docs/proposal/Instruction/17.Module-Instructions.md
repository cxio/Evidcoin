# Module Instructions（模块指令）

## 来源构想

- `docs/conception/Instruction/17.模块指令.md`：定义 opcode `225-250`。
- `docs/proposal/Instruction/0.Base-Constraints.md`：定义 `CALL`、模块对象和成本边界。

## 范围

模块指令包括标准模块引用：

- `MO_MATH`
- `MO_FMT`
- `MO_XX`
- 其它保留或标准模块位。

## 规范要求

- 模块指令返回 Module/Object 值。
- 模块方法只能通过白名单方法表调用。
- `CALL` 调用模块方法时，方法名、实参数量、返回类型和错误规则必须固定。
- 多返回值应封装为 Slice。
- 模块方法不得直接等同 Go 标准库全量 API。
- 模块版本、弃用和新增必须由注册表治理。

## 风险

- 直接映射 Go 标准库会引入版本差异。
- `MO_FMT` 格式化结果可能随实现变化。
- 动态字符串方法名难以静态审计。
- 模块无限扩展会破坏共识兼容。

## 未决问题

- 标准模块完整列表。
- 方法名大小写和别名。
- 模块注册表治理。
- Module/Object 是否可复制、比较、序列化。
- MO_FMT 是否允许公共验证。

## 对后续 Plan 的影响

- 模块实现必须从最小白名单开始，每个方法单独测试确定性、成本和错误规则。
