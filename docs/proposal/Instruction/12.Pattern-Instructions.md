# Pattern Instructions（模式指令）

## 来源构想

- `docs/conception/Instruction/12.模式指令.md`：定义 opcode `116-127`。
- `docs/proposal/Instruction/0.Base-Constraints.md`：定义脚本只读、返回值和成本规则。

## 范围

模式指令用于匹配脚本或字节序列：

- `MODEL`
- `_`
- `_[n]`
- `?{}`
- `^?`
- `#`
- `!?Type`
- 整数范围。
- 浮点范围。
- `RE`
- `&`
- `...`

## 规范要求

- `MODEL` 目标可以是 Script 或 Bytes，必须明确解析模式。
- 无取值时返回 Bool；有取值时返回 `[]any, Bool`。
- `MODEL` 附参最高位表示是否取值，最大子块候选 32KB。
- 严格匹配、附参匹配、关联数据匹配和深匹配必须有不同语义。
- 通配符和可选片段必须有确定的非贪婪/贪婪策略。
- 指令取值队列必须按匹配顺序稳定返回。
- 正则匹配必须使用 RE2 语义。
- 模式匹配必须有最大步数和成本预算。

## 风险

- `...`、正则和可选片段组合可能造成高成本。
- 模式语言复杂，解析歧义会导致跨实现差异。
- 模式可用于中间件自动识别，误匹配风险高。
- Hash 匹配算法未明确时不能用于协议级证明。

## 未决问题

- MODEL 哈希匹配使用何种 Hash 算法。
- 最大步数和成本公式。
- 子块长度 32KB 是否最终固定。
- 取值返回结构的精确类型。
- 可选片段嵌套规则。

## 对后续 Plan 的影响

- 模式指令测试必须覆盖成功匹配、失败匹配、取值顺序、成本上限和恶意复杂模式。
