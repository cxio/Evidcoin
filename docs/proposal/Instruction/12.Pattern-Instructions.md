# Pattern Instructions（模式指令）

Status: Draft proposal
Source conception: `conception/Instruction/12.模式指令.md`

## Prerequisites（前置条件）

关于栈模型、本地帧、opcode 布局等基础约束，参见 `0.Base-Constraints.md`。

本文使用的核心概念：脚本指令按 opcode、立即数操作数与关联数据进行比较；运行时参数来自参数区或数据栈；嵌套代码块体默认视为关联数据，除非该指令语义明确进入子块。

## Overview（概述）

模式指令定义了一个基于模型的匹配子系统，用于验证目标脚本序列是否符合预期逻辑形态。主入口为 `MODEL{}(2)`，它会打开一个模式匹配子环境，并将目标 `Script` 或 `Bytes` 序列与模型块进行比对。

当脚本在执行动作前需要先被检查时，模式匹配非常有用。典型场景包括：跨链交换中校验对手方预付款脚本是否符合预期形状、在中间件中从脚本提取结构化数据、以及无人值守自动化流程中对脚本形态进行无需人工复核的验证。

在 `MODEL{}` 内，普通指令表示严格全量匹配：目标指令必须在 opcode、立即数操作数和关联数据上完全一致。该指令族中的模式专用指令在此基础上提供单指令通配、可选序列、片段捕获、类型测试、区间测试、正则检索与指令段通配能力。

被广泛约定的脚本流程可预先发布，并作为 `GOTO` 或 `EMBED` 的目标被引用；随后可用 `MODEL{}` 检查是否符合这些已知流程。

## Opcode range（操作码范围）

Range: `[116-127]`, total **12** opcodes.

| Opcode | Mnemonic | Purpose |
|--------|----------|---------|
| `116` | `MODEL{}(2)` | 创建模式匹配区域 |
| `117` | `_` | 单指令通配符 |
| `118` | `_(~)` | 定长指令段通配符 |
| `119` | `?{}(~)` | 可选指令序列 |
| `120` | `^?(1)` | 后续指令的通配指示器 |
| `121` | `#(1)` | 指令片段捕获 |
| `122` | `!?(1)Type` | 类型匹配或可选类型匹配 |
| `123` | `>{}(~,~)` | 整数类数值区间匹配 |
| `124` | `>{}(8,8,4)` | 浮点数值区间匹配 |
| `125` | `RE{[INST]/.../gGS}(1,1,1)` | 正则匹配/检索 |
| `126` | `&(1)` | 正则结果捕获 |
| `127` | `...` | 任意指令段通配符 |

## Instruction Catalog（指令目录）

### `MODEL{}(2)` — Create a pattern-matching region（创建模式匹配区域，opcode 116）

```go
116     MODEL{}(2)      // 创建模式模型块
```

- **Type**: 模式块执行器。
- **Operands**:
  - **Immediate**: `blockLength`（2 字节）。最高位标记模型是否包含捕获指令；其余 15 位编码模型子块长度，因此模型代码上限为 32 KB。
  - **Associated data**: 编译后的模式子块。
  - **Runtime args**: 待测试目标序列，可为 `Script` 或 `Bytes`。
- **Stack effect**:
  - 无捕获时：`( target -- bool )`。
  - 有捕获时：`( target -- captures bool )`，其中 `captures` 为切片。
- **Description**: 打开解析子环境，并用模式块匹配目标序列。若模型不含捕获，仅返回表示成功与否的 `Bool`。若模型至少含一个捕获，返回 `(Slice, Bool)`；两个返回值会自动展开到目标区。模型是否含捕获在编译期可知，因此不会引入返回形态歧义。
- **Errors-Constraints**: 运行时参数必须是脚本类序列；匹配失败返回 `false`，不会触发引擎错误，除非嵌套指令明确规定失败语义。即使设置了捕获标志，模型块仍不得超过 15 位长度上限。

```go
// 目标脚本：
CODE {
     TOP FN_PUBHASH
     DATA{<hashResult>}
     EQUAL PASS
     FN_CHECKSIG PASS
}

// 含捕获：严格指令必须一致；DATA 载荷被捕获。
MODEL{
     TOP FN_PUBHASH      // 严格全量匹配
     DATA[32]{?}         // 内容任意，但长度必须是 32 字节
     #[64]               // 捕获 DATA{} 的关联数据
     EQUAL PASS          // 严格全量匹配
     FN_CHECKSIG PASS    // 严格全量匹配
}                        // => ([<hashResult>], true)
@POPS[2] OUTPUT          // 导出两个返回值

// 不含捕获：仅返回 Bool。
MODEL{
     TOP FN_PUBHASH      // 严格全量匹配
     DATA[32]{?}         // 定长、内容任意
     EQUAL PASS          // 严格全量匹配
     FN_CHECKSIG PASS    // 严格全量匹配
}                        // => true
PASS                     // 模型匹配成功则通过
```

### `_` — Single-instruction wildcard（单指令通配符，opcode 117）

```go
117     _               // 匹配任意一条指令
```

- **Type**: 模式专用单指令通配符。
- **Operands**:
  - **Immediate**: 无。
  - **Associated data**: 无。
  - **Runtime args**: 无。
- **Stack effect**: 在 `MODEL{}` 内不影响数据栈。
- **Description**: 匹配当前位置上的任意目标指令，包含该指令的 opcode、立即数操作数和关联数据。目标位置必须存在指令。
- **Errors-Constraints**: 仅在 `MODEL{}` 内有效。若当前位置不存在目标指令，则模型失败。

```go
_        // 精确消费一条目标指令，不关心其内容
```

### `_(~)` — Fixed-length instruction-segment wildcard（定长指令段通配符，opcode 118）

```go
118     _(n)            // 匹配任意 n 条同层级指令
```

- **Type**: 模式专用指令段通配符。
- **Operands**:
  - **Immediate**: `n`（varint），要消费的目标指令数量。
  - **Associated data**: 无。
  - **Runtime args**: 无。
- **Stack effect**: 在 `MODEL{}` 内不影响数据栈。
- **Description**: 匹配当前层级上连续的 `n` 条目标指令。`_[1]` 等价于 `_`；`_[0]` 无实际意义。计数仅包含当前层级同级指令。块指令按一条计，其子代码在此语义下视为关联数据。
- **Errors-Constraints**: 仅在 `MODEL{}` 内有效。若当前层级剩余目标指令少于 `n` 条，则模型失败。

```go
_[3]     // 通配当前目标指令及后续两条
_[1]     // 与 _ 效果相同
```

### `?{}(~)` — Optional instruction sequence（可选指令序列，opcode 119）

```go
119     ?{}(~)          // 可选的精确指令序列
```

- **Type**: 模式专用可选序列。
- **Operands**:
  - **Immediate**: `sequenceLength`（varint），可选指令序列的编码长度。
  - **Associated data**: `{}` 包裹的编译后指令序列。
  - **Runtime args**: 无。
- **Stack effect**: 在 `MODEL{}` 内不影响数据栈。
- **Description**: 被包裹的指令序列要么完整出现在当前目标位置，要么不存在。这是包含 opcode、立即数操作数和关联数据在内的等值测试。若序列不存在，当前位置原始目标指令保留，供后续测试继续使用。
- **Errors-Constraints**: 仅在 `MODEL{}` 内有效。即使只有一条指令也必须使用花括号，以避免与其他指令形态歧义。匹配为 **non-backtracking**：若首条不匹配，整个可选序列直接跳过；若前缀匹配但后续某条不匹配，则整个 `MODEL{}` 失败。

```go
?{ @TOP PRINT }  // 目标位置可包含该打印序列
?{ POPS[3] }     // 目标位置可包含一条 POPS[3]
?{ {1,3,5} }     // 可选的值集合指令
```

### `^?(1)` — Wildcard directive for the following instruction（后续指令通配指示器，opcode 120）

```go
120     ^?(1)           // 标记后续指令中可通配的部分
```

- **Type**: 模式专用修饰器，作用于后续一条指令。
- **Operands**:
  - **Immediate**: `mask`（1 字节），位掩码，描述后续指令中哪些部分为通配或可选。
  - **Associated data**: 指示器自身无关联数据。
  - **Runtime args**: 无。
- **Stack effect**: 在 `MODEL{}` 内不影响数据栈。
- **Description**: 指示匹配器在与目标比较时，后续指令哪些部分允许变化。前缀 `^` 用于解析器识别该指令。`?` 标记嵌入在后续指令源码形态中，以增强可读性。
- **Errors-Constraints**: 仅在 `MODEL{}` 内有效，且只作用于紧随其后的那条指令。语法规则禁止的混合写法在编译期报错。哈希匹配只适用于关联数据的严格相等测试，且仍需显式给出目标指令。

Bit mask:

| Bit | Meaning |
|-----|---------|
| `0x01` | 第一立即数操作数任意 |
| `0x02` | 第二立即数操作数任意 |
| `0x04` | 第三立即数操作数任意 |
| `0x08` | 第四立即数操作数任意 |
| `0x10` | 未定义 |
| `0x20` | 关联数据任意 |
| `0x40` | 指令本身可选 |
| `0x80` | 关联数据按哈希匹配 |

```go
^DATA[20]{?}         // 目标是 DATA，数据 20 字节，内容任意
^[0x20] DATA[20]     // 同上，显式掩码写法

^DATA[?]{?}          // 目标是 DATA，立即数与数据都任意
^[0x21] DATA         // 同上

^DATA[20]{?}?        // 目标需满足 DATA[20]{?}，但该指令可选
^[0x60] DATA[20]     // 同上
^[0x20] DATA[20]?    // 语义等同，但不支持混合写法

^DATA[?]{?}?         // 目标是 DATA，立即数和数据任意，且可选
^[0x61] DATA         // 同上
^DATA?               // 同上；可省略内部 immediate/data 的 ?，但后缀 ? 必须保留

^""[50]{?}           // 匹配任意 50 字节文本
^[0x20] ""[50]       // 同上
^{123}?              // 目标是整数值 123，且可选
^{?}                 // 目标是任意数值（Int|Float|BigInt）
^[0x40] {123}        // 同一可选值语义
^'A'?                // 目标是 Byte 或 Rune 'A'，可选

^$?                  // 匹配本地存储指令 $，可选
^$[?]                // 匹配本地加载指令 $(1)，槽位任意
^$[?]?               // 同上且可选；此处 immediate 位不能省略
^$[1]?               // 匹配 $(1) 的第二项，可选
```

Deep matching and single matching for block instructions:

```go
// 深度匹配：若前面没有通配指示器，块指令会进入子块继续匹配。
// 其立即数表示当前子块长度，而不是待比较值。
IF{...}              // 进入 IF 子块继续匹配

// 单条匹配：若有通配指示器，块指令按一条指令处理，子块视为关联数据。
^[0x20] IF[60]       // 匹配 IF，子块长度 60 字节，内容任意
^IF[60]{?}           // 同上，更易读
^IF[?]{?}            // 匹配任意 IF 指令；长度与内容任意

^()[30]{?}           // 目标表达式占 30 字节，内容任意
^()[?]{?}            // 目标是任意表达式且必须存在
^(?)                 // 同上，简化写法
^(?)?                // 目标是任意表达式，且可选
^()?                 // 同上；省略内部 ?，后缀 ? 必须保留
```

Hash-match examples:

```go
^[0x80] DATA[80]{0x4f262ae8...}
// 目标是 DATA，数据长度 80 字节，且关联数据哈希为 4f262ae8...

^[0x80] CODE{0x4f262ae8...}
// 目标 CODE 的关联数据哈希为 0x4f262ae8...；省略 immediate 表示长度任意

^[0x80] CODE[?]{0x4f262ae8...}
// 不支持：哈希匹配形式必须省略 [?]，不能混用两种写法
```

没有“对任意指令做哈希匹配”的记法；目标指令必须显式指定。

### `#(1)` — Instruction-part capture（指令片段捕获，opcode 121）

```go
121     #(1)            // 捕获前一条已匹配指令的部分内容
```

- **Type**: 模式捕获。
- **Operands**:
  - **Immediate**: `mask`（1 字节），选择要捕获的前一条已匹配目标指令部分。
  - **Associated data**: 无。
  - **Runtime args**: 无。
- **Stack effect**: 无即时数据栈效果；捕获值进入内部队列，并由 `MODEL{}` 作为捕获切片返回。
- **Description**: 捕获由紧邻前一条模式指令所匹配到的目标指令中的选定部分。可用于任何已匹配指令，但最常见于 `^?` 通配指示器之后。若选择多个部分，会按顺序展开进内部捕获队列。捕获整条指令时，opcode、立即数操作数和关联数据视为一个整体。
- **Errors-Constraints**: 仅在 `MODEL{}` 内有效。必须紧跟其所捕获的目标模式指令。

Bit mask:

| Bit | Captured value |
|-----|----------------|
| `0x01` | 第一立即数操作数 |
| `0x02` | 第二立即数操作数 |
| `0x04` | 第三立即数操作数 |
| `0x08` | 第四立即数操作数 |
| `0x10` | 未定义 |
| `0x20` | 关联数据 |
| `0x40` | 仅 opcode |
| `0x80` | 整条指令（含 opcode、立即数操作数、关联数据） |

```go
^DATA[20]{?} #[32]       // 捕获 DATA 目标的关联数据
^GOTO[?,?,?] #[1|2|4]    // 捕获 GOTO 的三个立即数操作数，按序展开
^GOTO[?,?,?] #[7]        // 同上；7 = 1|2|4
DATA[3]{414243} #[32]    // 严格匹配后，捕获已知值 0x414243
_ #[128]                 // 匹配任意指令并捕获整条指令
```

### `!?(1)Type` — Type match or optional type match（类型匹配或可选类型匹配，opcode 122）

```go
122     !?(1)Type       // 匹配失败即失败，或可选类型测试
```

- **Type**: 模式类型测试。
- **Operands**:
  - **Immediate**: `typeId`（1 字节）。最高位标记该操作是可选（`?`）还是匹配失败即失败（`!`）。
  - **Associated data**: 无。
  - **Runtime args**: 无。
- **Stack effect**: 在 `MODEL{}` 内不影响数据栈。
- **Description**: 测试当前位置目标指令是否表示指定类型值。`!Type` 为匹配失败即失败：目标不匹配则 `MODEL{}` 失败。`?Type` 为可选：目标不匹配时不消费目标指令，模型继续执行。
- **Errors-Constraints**: 仅在 `MODEL{}` 内有效。实现需通过 immediate 高位区分 `!` 与 `?`。类型匹配检查的是目标数据类型，具体值可变化。

Supported type names:

| Type | Meaning |
|------|---------|
| `Nil` | nil 值 |
| `Bool` | 布尔值，覆盖两个 Bool 值指令 |
| `Byte` | 字节值 |
| `Rune` | rune 值 |
| `Int` | 整数值 |
| `BigInt` | 大整数（`BigInt` 或 `*BigInt`） |
| `Float` | 浮点值 |
| `String` | 字符串值，覆盖两类字符串指令 |
| `Bytes` | 字节序列，覆盖两类字节序列指令 |
| `RegExp` | 正则表达式（`RegExp` 或 `*RegExp`） |
| `Time` | 时间值（`DATE{}`） |
| `Script` | 脚本（`Script` 或 `*Script`） |
| `Dict` | 字典 |
| `Number` | 整数或浮点数 |
| `Expr` | 表达式 |
| `Model` | 作为整体处理的模型块 |
| `Values` | 值集合，例如由 `{...}(~)` 创建的 `{1, 3, 5}` |
| `Loopv` | 循环变量，例如 `$X(1)` |

```go
!Bool    // 匹配 Bool 类型（TRUE 或 FALSE）
?Bool    // 目标可为 Bool 值

!Bytes   // 匹配字节序列类指令
?Bytes   // 同上，但可选
!String  // 匹配字符串指令
!Script  // 匹配 Script 或 *Script

!Number  // 匹配任意整数或浮点值
!Model   // 匹配一个 model block；语义同 MODEL[?]{?}
!Expr    // 目标是 expression；语义同 (?)
```

### `>{}(~,~)` — Integer-like value range match（整数类值区间匹配，opcode 123）

```go
123     >{}(~,~)        // 整数类下闭上开区间匹配
```

- **Type**: 模式区间测试。
- **Operands**:
  - **Immediate**: `lower`（varint），包含下界；`upper`（varint），不包含上界。
  - **Associated data**: 无。
  - **Runtime args**: 无。
- **Stack effect**: 在 `MODEL{}` 内不影响数据栈。
- **Description**: 若目标值落在半开区间 `lower <= value < upper`，则匹配成功。支持目标类型为 `Byte`、`Rune`、`Int` 与 `Time`，其中 `Time` 以 Unix 毫秒比较。
- **Errors-Constraints**: 仅在 `MODEL{}` 内有效。若目标类型不在支持范围或超出区间，则模型失败。不支持 `BigInt` 区间测试。

```go
>{0, 1024}     // 要求目标值满足 0 <= n < 1024
```

### `>{}(8,8,4)` — Float value range match（浮点值区间匹配，opcode 124）

```go
124     >{}(8,8,4)      // 浮点下闭上开区间匹配
```

- **Type**: 模式区间测试。
- **Operands**:
  - **Immediate**: `lower`（8 字节，float64）包含下界；`upper`（8 字节，float64）不包含上界；`epsilon`（4 字节）为等值容差。
  - **Associated data**: 无。
  - **Runtime args**: 无。
- **Stack effect**: 在 `MODEL{}` 内不影响数据栈。
- **Description**: 若 `Float` 目标值处于该半开区间，则匹配成功；等值比较允许误差不超过 `epsilon`。目标类型为 64 位双精度浮点。
- **Errors-Constraints**: 仅在 `MODEL{}` 内有效。若目标不是 `Float` 或超出容差范围，则模型失败。`epsilon` 以 4 字节浮点编码，其最小正值约为 `1.4013e-45`。

```go
>{0, 1.0, 0}         // 目标为 0 或任意小于 1.0 的 float
>{0.1, 10, 1e-10}    // 目标在 0.1 到 10 之间，下界比较带容差
```

### `RE{[INST]/.../gGS}(1,1,1)` — Regular-expression match/search（正则匹配/检索，opcode 125）

```go
125     RE{[INST]/.../gGS}(1,1,1)   // 对指令关联数据执行 regex 检索
```

- **Type**: 模式正则测试与结果生产器。
- **Operands**:
  - **Immediate**: `inst`（1 字节），可选指令名或 opcode，默认 `0xff` 表示任意指令；`mode`（1 字节）检索模式；`patternLen`（1 字节）正则源码长度，必须 `< 256`。
  - **Associated data**: `/.../` 之间的正则源码文本。
  - **Runtime args**: 无。
- **Stack effect**: 在 `MODEL{}` 内不影响数据栈；匹配结果保留供后续 `&(1)` 捕获。
- **Description**: 匹配目标指令，并将正则应用到该指令的关联数据上。关联数据可为字符串指令中的文本，或 `DATA`、`CODE`、`BLOCK`、`IF` 等块类指令中的字节。块指令子代码按字节序列处理。若省略 `[INST]`，编译后的第一立即数为 `0xff`，即任意带关联数据的指令。
- **Errors-Constraints**: 仅在 `MODEL{}` 内有效。若目标指令无关联数据，执行该模型指令即为错误，`MODEL{}` 失败。字符串关联数据与字节序列关联数据使用不同的正则处理函数。

Lookup modes:

| Mode | Meaning | Processing shape |
|------|---------|------------------|
| `0` | 首次匹配，不含捕获组；默认 | `.Find` / `.FindString` |
| `g` | 全部匹配，仅完整匹配项 | `.FindAll` / `.FindAllString` |
| `G` | 全部匹配，含子匹配 | `.FindAllSubmatch` / `.FindAllStringSubmatch` |
| `S` | 首次匹配，含子匹配 | `.FindSubmatch` / `.FindStringSubmatch` |

```go
RE{8/(?i)([a-z]+):(\d+)/S}
// 在 opcode 8 的目标字符串指令上，用 FindStringSubmatch 检索。
// 结果是 []String：先完整匹配，再两个捕获组。

RE{DATA/\d+/g}
// 在 DATA 关联数据上，用 FindAll 查找所有数字字节序列。
// 结果是 []Bytes，内容为完整匹配项。

RE{/(?i)([a-z]+)/g}
// 在任意带关联数据的指令上，查找所有连续字母。
// 字符串指令产生 []String；字节序列指令产生 []Bytes。
// 省略的 [INST] 会编译为 0xff。
```

该语法刻意区别于用于创建正则字面量值的值指令写法（`/.../`）。

### `&(1)` — Regular-expression result capture（正则结果捕获，opcode 126）

```go
126     &(1)            // 捕获一个 regex 结果成员
```

- **Type**: 模式捕获。
- **Operands**:
  - **Immediate**: `index`（1 字节），结果序号 `< 256`。
  - **Associated data**: 无。
  - **Runtime args**: 无。
- **Stack effect**: 无即时数据栈效果；捕获值进入内部队列，并由 `MODEL{}` 作为捕获切片返回。
- **Description**: 从最近一次 `RE{...}` 结果中捕获一个成员。应紧跟所属 `RE{...}` 指令，但在下一条 `RE{...}` 出现前可重复使用。若目标无匹配结果或请求索引越界，则捕获值为 `nil`。
- **Errors-Constraints**: 仅在 `MODEL{}` 内有效。最多可索引 256 个正则结果成员；超出范围的匹配结果无法被捕获。

```go
RE{/(?i)[a-z]+/S} &[0]       // 捕获首个匹配结果成员
RE{/Ver-(\d+)/S} &[1]        // 捕获第一子表达式，例如 major version
RE{/(?i)[a-z]+/g} &[3] &[1]  // 捕获全局匹配中的第 4 项和第 2 项
```

### `...` — Arbitrary instruction-segment wildcard（任意指令段通配符，opcode 127）

```go
127     ...             // 匹配任意连续指令段
```

- **Type**: 模式专用任意段通配符。
- **Operands**:
  - **Immediate**: 无。
  - **Associated data**: 无。
  - **Runtime args**: 无。
- **Stack effect**: 在 `MODEL{}` 内不影响数据栈。
- **Description**: 匹配任意连续的同层级指令序列，包含零长度序列。匹配按层独立：每个块层级单独处理，块指令在当前层按单实体计数。该通配符为非贪婪，会最小化完整匹配。
- **Errors-Constraints**: 仅在 `MODEL{}` 内有效。当模型需要匹配到目标脚本末尾时，必须使用尾部 `...`。该指令可能较耗资源；当已知段长度时，优先使用定长 `_(n)`。

```go
// 目标脚本：
ENV{Height} {<expireHeight>} GT
@TOP PRINT          // 1
IF{
    @POP NIL
}
TRUE
IF{
    "Hello"         // 2
    @TOP PRINT      // A
}
"Done!"
@TOP PRINT          // B
@POPS[] NOP         // End

// Model 1:
MODEL{
     ENV{Height} {<expireHeight>} GT
     ...                 // 1 => TRUE
     IF{
         ...             // 2 "Hello"
         @TOP PRINT      // A
     }
     "Done!"
     @TOP PRINT          // B
     @POPS NOP           // 必须抵达完整终点
}

// Model 2:
MODEL{
     ENV{Height} {<expireHeight>} GT
     ...
     IF{?}               // 因为非贪婪，匹配到第一个 IF
     ...                 // 其余匹配，必需
}

// 最后的 ... 由完整性规则要求：匹配必须到达目标脚本末尾。

// Model 3:
MODEL{
     ENV{Height} {<expireHeight>} GT
     ...
     IF{?}               // 匹配到第二个 IF，
     "Done!"             // 因为这里与 IF 构成组合约束
     ...                 // 到达末尾
}
```

## Design notes（设计说明）

- **Strict-by-default model semantics**: 在 `MODEL{}` 中，普通指令默认表示与目标指令进行严格全量匹配，包含 opcode、立即数操作数和关联数据。因此值指令也天然包含其类型逻辑。
- **Capture-aware return shape**: `MODEL{}(2)` 的返回形态在编译期确定。无捕获时返回 `Bool`；存在至少一个 `#` 或 `&` 捕获时返回 `(Slice, Bool)` 且自动展开。捕获状态标志存于 2 字节 immediate 的最高位，因此块长度缩减为 15 位。
- **Wildcard directive readability**: `^?(1)` 既可写成显式掩码（`^[0x20] DATA[20]`），也可写成内嵌语法糖（`^DATA[20]{?}`）。仅在语法定义允许的场景下二者等价；不支持的混合写法必须报错，不能静默归一化。
- **Deep matching vs. single matching**: 若无前置通配指示器，`IF{...}` 这类块指令会进入子块渐进匹配；若有前置 `^?` 指示器，块按单条指令处理，子块作为关联数据。
- **Non-backtracking optional sequences**: `?{}` 不会搜索替代对齐路径。首条缺失则整个可选序列跳过；若前缀匹配但剩余不匹配，则模型失败。
- **Hash matching is explicit**: 关联数据哈希匹配可用于长数据等值检查，但必须显式点名目标指令。不存在“任意指令”的哈希匹配形式。
- **Segment matching granularity**: `_(n)` 与 `...` 都工作在当前指令层级。块指令在同层仅算一个实体，其嵌套序列不会被拍平到父层。
- **Regular-expression separation**: `RE{...}` 是模式检索指令，不等同于值层正则字面量语法。文本与字节关联数据刻意使用不同匹配 API。

## Pattern Matching Summary（模式匹配摘要）

- **Positional matching**: 模型指令与目标指令按位置进行比较。
- **Instruction matching**: 指令完全一致则匹配。若该指令是值指令，其类型逻辑也作为匹配的一部分。
- **Immediate-operand matching**: 在 opcode 匹配后，立即数操作数参与比较，除非被通配。
- **Data matching**: 在 opcode 匹配后，关联数据参与比较，除非被通配或按哈希匹配。
- **Type matching**: 可匹配目标的数据类型，同时允许具体值任意。
- **Range restriction**: 整数类与浮点值可按数值区间匹配。
- **Regular-expression matching**: `RE{/../}` 可在文本或字节序列上测试并捕获。
- **Fragment matching**: 连续指令片段可通过 `_(1)`、`_(n)` 和 `...` 匹配或通配。
