# Comparison Instructions（比较指令）

## 来源构想

- `docs/conception/Instruction/10.比较指令.md`：定义 opcode `104-111`。
- `docs/proposal/Instruction/0.Base-Constraints.md`：定义运行时值和失败规则。

## 范围

比较指令包括：

- `EQUAL`
- `NEQUAL`
- `LT`
- `LTE`
- `GT`
- `GTE`
- `ISNAN`
- `WITHIN`

## 规范要求

- 必须提供可比较类型矩阵。
- 不同类型之间是否自动转换必须明确；默认应拒绝隐式跨类比较。
- Bytes 比较按字节序列。
- Bool 只支持相等/不等。
- String 比较若允许，必须按 Unicode code point、UTF-8 bytes 或其它明确规则。
- Time 比较使用确定性时间戳。
- Float NaN 与相等、大小比较必须明确；`ISNAN` 专门处理 NaN。
- `WITHIN` 边界建议为 `[min, max)`，但需确认。

## 风险

- 自动类型转换会产生非直观验证结果。
- Float 相等比较易误用。
- String 本地化排序不可进入公共验证。
- Bytes 与 String 混比容易造成编码混淆。

## 未决问题

- 不同数值类型比较是否允许提升。
- BigInt 与 Float 比较是否允许。
- String 排序规则。
- `WITHIN` 是否包含上边界。
- NaN 在 `EQUAL/NEQUAL` 中的结果。

## 对后续 Plan 的影响

- 比较测试必须覆盖每种类型组合、NaN、边界相等和非法混比。
