# Comparison Instructions（比较指令）

Status: 草案提案
Source conception: `conception/Instruction/10.比较指令.md`

## Prerequisites（前置条件）

关于栈模型、本地帧、opcode 布局、操作数来源以及运行时参数解析，参见 `0.Base-Constraints.md`。

下文使用概念回顾：

- 所有比较 opcode 都会消费两个运行时参数（对于 `WITHIN`，则是一个值加一个范围定义），并产出单个 `Bool` 结果。
- `Float` 在技术上定义了相等比较，但当需要处理精度误差时，建议使用 `COMFLO`（见 `14.Tool-Instructions.md`）。

## Overview（概述）

比较指令执行值级别的关系测试。结果始终是一个 `Bool`，压入数据栈（或路由到 capture 目标）。

支持对 `Bytes` 进行按字节比较（`<`、`>` 等），比较按从左到右、逐 octet 进行。`Bool` 仅支持 `EQUAL` / `NEQUAL`。

## Opcode range（Opcode 范围）

范围：`[104 – 111]`，共 **8** 个 opcode。

| Sub-group（子组）      | Range（范围） | Count（数量） |
| ---------------------- | ------------- | ------------- |
| Relational binary（双目关系） | `104 – 109`   | 6             |
| NaN test（NaN 测试）   | `110`         | 1             |
| Bounded range test（有界区间测试） | `111`         | 1             |

## Instruction Catalog（指令目录）

### `EQUAL` / `NEQUAL` / `LT` / `LTE` / `GT` / `GTE` (opcodes 104 – 109)（关系比较）

```go
104     EQUAL       // a == b
105     NEQUAL      // a != b
106     LT          // a < b
107     LTE         // a <= b
108     GT          // a > b
109     GTE         // a >= b
```

- **Type**: 关系比较。
- **Operands**: 两个可比较的运行时值。
- **Stack effect**: `[ … a b ] → [ … bool ]`。
- **Description**:
  - 标准关系语义。
  - 对于 `Bytes`，比较为 **lexicographic, byte-by-byte from the left**。
  - 对于 `Bool`，仅接受 `EQUAL` 和 `NEQUAL`；在 `Bool` 操作数上使用 `LT` / `GT` 等会构成错误。
  - 对于 `Float`，虽然定义了相等比较，但应谨慎使用；容差比较请参见 `COMFLO`。
- **Errors**:
  - `a` 与 `b` 类型不匹配。
  - 在 `Bool` 操作数上应用 `LT`/`LTE`/`GT`/`GTE`。

### `ISNAN` — Floating-point NaN test (opcode 110)（浮点 NaN 测试）

```go
110     ISNAN       // 该值是否为 NaN？
```

- **Operand**: 任意值。
- **Stack effect**: `[ … v ] → [ … bool ]`。
- **Description**: 当且仅当 `v` 是 `Float` 且 `math.IsNaN(v)` 为真时返回 `true`。该指令是必要的，因为浮点运算可能产出 `NaN`，从而污染后续比较结果。
- **Errors**: 无（非 Float 操作数仅返回 `false`）。

### `WITHIN` — Bounded range membership (opcode 111)（有界区间归属测试）

```go
111     WITHIN      // min <= x < max
```

- **Operands**:
  1. 待测试值（`x`）；
  2. 一个包含 2 个成员的 slice `{min, max}`，表示半开区间。
- **Stack effect**: `[ … x {min,max} ] → [ … bool ]`。
- **Description**:
  - 当且仅当 `min <= x < max` 时返回 `true`。
  - 支持类型：`Byte`、`Rune`、`Int`、`BigInt`、`Float`。
  - **Caveat**: 对 `Float` 而言，下界处的相等判断可能存在精度误差。

```go
ENV{Height} {0, 240} WITHIN
// 当且仅当当前区块高度属于第一天时为 True
// （链每天产出 240 个区块）。
```

- **Errors**:
  - 范围参数不是 2 成员 slice。
  - `min` 与 `max` 的类型与 `x` 或彼此不匹配。

## Design notes（设计说明）

- 提供 **`Bytes` lexicographic ordering**，使哈希摘要、账户哈希和其他二进制标识符可以无需转换就直接做区间检查或排序。
- **`Bool` is restricted to equality** —— 布尔值不存在对脚本引擎有实际意义的自然排序；该限制在 dispatcher 层强制执行。
- **`WITHIN` is half-open `[min, max)`** —— 该设计与典型的 "block range" / "index range" 用法兼容，其中上界为排他。
- **`BigInt` is supported in `WITHIN`**，但不在 symbolic expression block 中支持（见 `9.Arithmetic-Instructions.md`）。
