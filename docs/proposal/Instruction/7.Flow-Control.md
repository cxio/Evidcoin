# Proposal: Flow Control Instructions [58–66]（提案：流程控制指令 [58–66]）

> **Prerequisite:** 本规范建立在 [Base Constraints](0.Base-Constraints.md) 文档之上；该文档定义了全局采用的指令格式、运行时模型与记号约定。


## 1. Overview（概览）

流程控制指令占用 opcode 范围 **[58–66]**（共 9 条）。它们通过条件分支、多路选择、迭代与块作用域来控制脚本执行路径。所有流程控制指令都携带**子语句块**作为关联数据，即按条件执行的嵌入式指令序列。

### Design Principle（设计原则）

流程控制指令本身**不返回值**，仅改变执行流。其子块内产生的数据会写入共享 data stack（或通过 capture prefix 写入其他目标空间），但流程控制指令自身不产生返回。

每个子语句块都会创建独立的**local scope**（block domain），用于隔离本地存储值，避免与同级或父级块互相干扰。

### Important Constraint（重要约束）

脚本系统**不存在无限循环**。所有迭代都受 `EACH{}` 输入集合大小约束。循环体支持 `BREAK`（提前退出）与 `CONTINUE`（跳到下一次迭代），但不支持 `RETURN`（`RETURN` 仅保留给函数式块 `MAP{}`/`FILTER{}`）。


## 2. Instruction Definitions（指令定义）

### 2.1 Conditional — IF（条件分支 IF）

```
58      IF{}(~)     Conditional execution
```

- **Auxiliary params:** Varint —— 子块字节长度。
- **Arguments:** 一个 boolean 值。
- **Return value:** 无。
- 参数为 `true` 时执行子块；为 `false` 时跳过。
- 条件通过参数表达，需在 IF 前准备好。
- **IF state:** 在同一语法层级中，后出现的 `IF` 会**覆盖**先前 `IF` 状态；`ELSE` 基于该状态决定是否执行。
- **Example:**
  ```
  IF {
       @POP PRINT    // 弹出并打印栈顶项
  }
  ```

### 2.2 Conditional — ELSE（条件分支 ELSE）

```
59      ELSE{}(~)   Alternative execution
```

- **Auxiliary params:** Varint —— 子块字节长度。
- **Arguments:** 无。
- **Return value:** 无。
- 在当前语法层级读取 IF-state：若最近一次 `IF` 结果为 `false`，则执行 ELSE 块。
- `ELSE` 匹配同层级内最近的前置 `IF`。
- `IF` 与 `ELSE` 不要求紧邻，中间可插入其他指令。
- **State machine rules:**
  - IF 与 ELSE 是**同级关系**—— 同层不允许彼此嵌套，但父子层嵌套是正常的。
  - 无论执行 IF 块还是 ELSE 块，之后 IF-state 都会标记为**completed**。
  - 连续多个 ELSE：同一 IF 后仅第一个生效，后续同层 ELSE 均为**inactive**（非法用法）。
  - 同层全部 IF/ELSE 对共享一个状态变量。
- **Valid patterns:**
  ```
  IF{} ELSE{} IF{} ELSE{}           // 合法：两组独立 IF/ELSE
  IF{ IF{} ELSE{} } ELSE{}          // 合法：外层 IF 内部再嵌套 IF/ELSE
  ```
- **Invalid pattern:**
  ```
  IF{} IF{} ELSE{} ELSE{}           // 非法：第二个 ELSE 无可匹配 IF
  ```
- **Example with interleaved instructions:**
  ```
  IF {
       ...
  }
  @POP PRINT        // 不影响 IF/ELSE 配对关系
  ELSE {
       ...
  }
  ```

### 2.3 Switch Block（SWITCH 块）

```
60      SWITCH{}(~) Multi-way branch block
```

- **Auxiliary params:** Varint —— 块字节长度。
- **Arguments:**
  1. **target value** —— 用于与各 CASE 分支值比较，类型需可比较。
  2. **branch value list** —— 分支值序列，与 CASE 按位置一一对应。
- **Return value:** 无。
- 创建 switch 选择上下文。target value 与所有分支比较值一次性提供，使 CASE 块保持简洁（CASE 本身不包含比较逻辑）。
- **Design note:** 将比较值集中提供（而非分散到每个 CASE）可让 CASE 代码更干净。
- **Tip:** 若 target value 是 `true`，则 CASE 分支值集为 boolean 列表，可模拟 `if/elseif/else` 链逻辑。

### 2.4 Case Branch（CASE 分支）

```
61      CASE{}(~)   Conditional branch within SWITCH
```

- **Auxiliary params:** Varint —— 子块字节长度。
- **Arguments:** 无（比较由 SWITCH 预先计算）。
- **Return value:** 无。
- 在 SWITCH 中，CASE 按位置对应比较：CASE[0] 对应 branch value[0]，CASE[1] 对应 branch value[1]，依此类推。
- **First-match semantics:** 仅首个匹配 CASE 会执行（目标值和分支值相等）；执行后 SWITCH 直接结束（无 fall-through）。
- CASE 内也可使用 `BREAK` 提前退出。
- **Example:**
  ```
  {10}                // SWITCH 的 target value
  {3, 10, 20}         // 分支值：[0]=3, [1]=10, [2]=20
  SWITCH {
       CASE{          // [0]: 3 == 10 → false，跳过
            ...
       }
       CASE{...}      // [1]: 10 == 10 → true，执行后退出 SWITCH
       CASE{...}      // 不会到达
       DEFAULT{        // 不会到达（已有 CASE 匹配）
            ...
       }
  }
  ```
- **Note:** CASE 块之间可以插入其他指令，但实际较少见。

### 2.5 Default Branch（DEFAULT 分支）

```
62      DEFAULT{}(~)    Default branch within SWITCH
```

- **Auxiliary params:** Varint —— 子块字节长度。
- **Arguments:** 无。
- **Return value:** 无。
- 当所有 CASE 比较结果都为 false 时执行。该分支可选；若省略且无 CASE 命中，SWITCH 直接退出。
- 应当作为 SWITCH 内最后一个分支。

### 2.6 Iteration — EACH（迭代 EACH）

```
63      EACH{}(~)   Bounded iteration (loop)
```

- **Auxiliary params:** Varint —— 子块字节长度。
- **Arguments:** 一个集合（slice 或 dict），作为迭代目标。
- **Return value:** 无。
- 逐个遍历集合成员，每个成员执行一次子块。
- **Loop variables** 可通过 `$X` capture instructions 访问（见 [Capture Instructions](2.Capture-Instructions.md)）：
  - `$Value`（index 0）—— 当前元素值。
  - `$Index` / `$Key`（index 1）—— 当前索引（slice）或键（dict）。
  - `$Slice` / `$Dict`（index 2）—— 集合本身。
  - `$Size`（index 3）—— 集合大小。
- 与 `MAP{}`/`FILTER{}`（tool instructions）不同，EACH **不会**创建私有 data stack，而是在共享 data stack 上操作。
- 若需提前退出循环，使用 `BREAK`（不是 `RETURN`）。
- **Examples:**
  ```
  {1, 3, 5}
  EACH{
       ($Value * 10)     // 表达式：当前值乘以 10，并压入结果
  }
  @SHIFT[3] PRINT        // → [10, 30, 50]
  ```
  ```
  {10, 20, 30}
  EACH {
       $Value PUSH        // 将当前值压入 data stack
  }
  // 等价于：{10, 20, 30} SPREAD
  ```

### 2.7 Continue（继续下一轮）

```
64      CONTINUE    Skip to next iteration
```

- **Auxiliary params:** 无
- **Arguments:** 可选 —— 一个 boolean 值。
- **Return value:** 无。
- 结束当前 `EACH{}` 迭代并进入下一轮。
- 若提供参数，仅当该值为 `true` 时执行 `CONTINUE`；无参数则无条件执行。

### 2.8 Break（中断）

```
65      BREAK       Exit loop or switch
```

- **Auxiliary params:** 无
- **Arguments:** 可选 —— 一个 boolean 值。
- **Return value:** 无。
- 完全退出当前外围 `EACH{}` 循环或 `SWITCH{}` 块。
- 若提供参数，仅当该值为 `true` 时执行 `BREAK`；无参数则无条件执行。
- **Example:**
  ```
  {1, 3, 30, 5}
  EACH{
       $Value {10} GT     // 比较当前值是否 > 10
       @POP BREAK          // 若为 true，退出循环
       @$Index PRINT       // 打印当前索引
  }
  @"Done!" PRINT

  // 输出：
  // 0        (value=1, 不大于 10)
  // 1        (value=3, 不大于 10)
  // Done!    (value=30, 大于 10 → break)
  ```

### 2.9 Block Scope（块作用域）

```
66      BLOCK{}(~)  Block scope
```

- **Auxiliary params:** Varint —— 子块字节长度。
- **Arguments:** 无。
- **Return value:** 无。
- 创建显式子块并提供独立 local scope，主要用途：
  1. **Local variable isolation** —— 在大型脚本中更好地组织局部作用域。
  2. **Code region demarcation** —— 定义可通过 `SOURCE` 抽取的连续指令区间。
- 块内容在共享 data stack 上按常规方式执行（属于当前脚本，非独立）。
- **Example:**
  ```
  BLOCK{
       ......           // 常规执行
       SOURCE           // 将当前块指令序列抽取到栈上
  }
  @POP PRINT            // 打印抽取出的指令序列
  ```
- **Note:** 若既不需要局部隔离也不需要 `SOURCE` 抽取，使用 `BLOCK` 与不使用在行为上等价。


## 3. Scoping and State Rules（作用域与状态规则）

### 3.1 Block-Level Local Scope（块级局部作用域）

每个流程控制子块（`IF{}`、`ELSE{}`、`SWITCH{}`、`CASE{}`、`DEFAULT{}`、`EACH{}`、`BLOCK{}`）都会创建自己的 local scope：

- 容量：每块 128 个条目。
- 仅追加（通过 `$` prefix）。
- 同级与父级块均不可互访。
- 块退出时销毁。

### 3.2 IF/ELSE State Machine（IF/ELSE 状态机）

在每个语法层级，使用一个 IF-state 变量跟踪最近一次 IF 结果：

```
状态迁移：
  IF(true)  → state = EXECUTED     → ELSE 跳过
  IF(false) → state = NOT_EXECUTED → ELSE 执行 → state = EXECUTED
  IF(any)   → 覆盖同层之前的 IF state
  ELSE      → 消费状态，无论结果都置为 EXECUTED
```

### 3.3 SWITCH First-Match Semantics（SWITCH 首匹配语义）

SWITCH 会预先完成全部比较。CASE 按位置顺序评估；首个命中即执行并结束 SWITCH，不存在 fall-through。

### 3.4 EACH Bounded Iteration（EACH 有界迭代）

- 迭代次数由集合大小决定，不可能出现无限循环。
- data stack 为共享栈（不是循环体私有栈）。
- `BREAK` 退出循环，`CONTINUE` 进入下一轮。
- `RETURN` 在 `EACH{}` 中非法（仅保留给 `MAP{}`/`FILTER{}`）。
