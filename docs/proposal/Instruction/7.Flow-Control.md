# Flow Control（流程控制）

Status: Draft proposal（草案提案）
Source conception: `conception/Instruction/7.流程控制.md`

## Prerequisites（前置条件）

有关栈模型、参数区、本地帧与 opcode 布局，请参见 `0.Base-Constraints.md`。简要如下：

- **data stack** 保存中间值；**argument zone** 为下一条指令暂存输入；**local frame** 是按 block 划分且不继承的作用域；**global frame** 作用于整个脚本。
- 每条指令均为 `opcode + immediate + associated data`；其中仅 opcode 是必需项。
- 对本指令族而言，**associated data is itself a sub-instruction sequence**（子 block），其字节长度由 immediate 操作数给出。

## Overview（概览）

该指令族用于控制脚本执行路径：条件分支（`IF` / `ELSE`）、多路分发（`SWITCH` / `CASE` / `DEFAULT`）、有界迭代（`EACH` 配合 `CONTINUE` / `BREAK`），以及显式词法作用域（`BLOCK`）。这些指令都不会产生栈值——它们只塑造控制流。系统刻意 **no unbounded loop construct**；所有迭代都必须通过 `EACH` 遍历有限集合完成。

## Opcode range（操作码范围）

`[58 – 66]`，共 9 条指令。

## Instruction Catalog（指令目录）

### 58 — `IF{}(~)` — conditional execution（条件执行）

| Field | Value |
|---|---|
| Type | Branch |
| Immediate | length (in bytes) of the child block |
| Associated data | child instruction sequence |
| Runtime args | a boolean |
| Stack effect | consumes one boolean |

**Description.** 仅当 boolean 参数为 `true` 时执行子 block。条件值需在 `IF` 执行前先计算并放入 argument zone。在同一 sibling 层级内，后出现的 `IF` 会覆盖先前 `IF` 的状态（见 Design notes）。

```go
IF {
    @POP PRINT   // 弹出并打印栈顶值
}
```

---

### 59 — `ELSE{}(~)` — fall-through branch（补充分支）

| Field | Value |
|---|---|
| Type | Branch |
| Immediate | length (in bytes) of the child block |
| Associated data | child instruction sequence |
| Runtime args | none |
| Stack effect | none |

**Description.** 仅当同级最近一次 `IF` 的求值结果为 `false` 时执行其子 block。`ELSE` 与对应 `IF` 不要求词法上紧邻——中间可以插入其他指令。

```go
IF {
    ...
}
@POP PRINT          // 不影响 IF/ELSE 的配对关系
ELSE {
    ...
}
```

```go
IF {...}
IF {...}            // 覆盖第一个 IF 的状态
ELSE {              // 与第二个 IF 配对
    ...
}
```

**Design.** 同层级的 `IF` / `ELSE` 按配对处理，**not nestable as siblings**；但父子层级嵌套是正常允许的：

- `IF{} ELSE{} IF{} ELSE{}` —— 合法（两组独立配对）。
- `IF{} IF{} ELSE{} ELSE{}` —— 第二个 `ELSE` 非法（没有可消费的 IF 活跃状态）。
- `IF{ IF{} ELSE{} } ELSE{}` —— 合法（作用域不同）。

任一分支执行后，其 IF 状态会被标记为 "consumed"。同层级全部 `IF` / `ELSE` 共享一个 IF 状态槽；孤立且拖尾的 `ELSE` 视为 no-op。

---

### 60 — `SWITCH{}(~)` — multi-way dispatch region（多路分发区域）

| Field | Value |
|---|---|
| Type | Branch |
| Immediate | length (in bytes) of the child block |
| Associated data | child instruction sequence (containing `CASE` / `DEFAULT` blocks) |
| Runtime args | `arg1` subject value (any comparable type); `arg2` slice of CASE candidate values (length must equal the number of CASE blocks) |
| Stack effect | consumes both args |

**Description.** 定义一个 switch 区域。subject 会按对应 CASE 位置与每个 candidate 执行相等比较。若 subject 为 `true`，可将其建模为 `if / elseif / else` 链。

**Design.** 在入口一次性预供 subject 与完整 candidate 列表，可让每个 `CASE` 体在比较点保持简洁、无副作用。

---

### 61 — `CASE{}(~)` — switch branch（switch 分支）

| Field | Value |
|---|---|
| Type | Branch (within `SWITCH`) |
| Immediate | length (in bytes) of the child block |
| Associated data | child instruction sequence |
| Runtime args | none |
| Stack effect | none |

**Description.** subject 与 candidate 的比较在 `SWITCH` 入口完成——到 `CASE` 位置时，仅按位置消费预计算的 boolean：

- true → 执行子 block，随后 **exit the SWITCH**。
- false → 跳过子 block。

只有 **first matching** `CASE` 会执行。`CASE` 内部的 `BREAK` 会显式退出 `SWITCH`。

```go
{10}                 // SWITCH 的 subject
{3, 10, 20}          // CASE 的 candidate 列表
SWITCH {
    CASE{            // [0] 与 subject 比较——此处为 false
        ...
    }
    CASE{...}        // [1] 与 subject 比较——此处为 true，会执行
    CASE{...}        // 跳过（SWITCH 已结束）
    DEFAULT{
        ...          // 仅当所有 CASE 都为 false 才执行
    }
}
```

`CASE` block 之间允许出现其他指令，但较少见。

---

### 62 — `DEFAULT{}(~)` — default branch（默认分支）

| Field | Value |
|---|---|
| Type | Branch (within `SWITCH`) |
| Immediate | length (in bytes) of the child block |
| Associated data | child instruction sequence |
| Runtime args | none |
| Stack effect | none |

**Description.** 可选，且必须是 `SWITCH` 内最后一个 block。仅当所有 `CASE` 求值为 false 时执行。

---

### 63 — `EACH{}(~)` — bounded iteration（有界迭代）

| Field | Value |
|---|---|
| Type | Loop |
| Immediate | length (in bytes) of the child block |
| Associated data | child instruction sequence |
| Runtime args | a target collection (slice or dict) |
| Stack effect | consumes the collection |

**Description.** 遍历目标集合，对每个成员执行一次子 block。与 `MAP` / `FILTER`（collection 指令族）不同，`EACH` **not** 分配私有 data stack——其主体与外层共享栈和 frame。

循环体可见以下循环变量（capture 风格；见 capture 指令）：

- `$Value` —— 当前成员值
- `$Index`（slice）/ `$Key`（dict）—— 当前位置或键
- `$Slice` / `$Dict` —— 源集合
- `$Size` —— 总长度

循环终止请使用 `BREAK`；**do not** 使用 `RETURN`（`RETURN` 仅保留给 `MAP` / `FILTER`）。

```go
{1, 3, 5}
EACH{
    ($Value * 10)        // 表达式结果会压入栈
}
@SHIFT[3] PRINT          // [10, 30, 50]
```

```go
{10, 20, 30}
EACH {
    $Value PUSH          // 压入当前值
}
// 等价于：
// {10, 20, 30} SPREAD   // {10} {20} {30}
```

---

### 64 — `CONTINUE` — skip to the next iteration（跳到下一次迭代）

| Field | Value |
|---|---|
| Type | Loop control |
| Runtime args | optional boolean |
| Stack effect | consumes the boolean if present |

**Description.** 在 `EACH` 内结束当前迭代并进入下一轮。带 boolean 参数时仅在 true 时执行；不带参数时无条件执行。

---

### 65 — `BREAK` — exit a loop or switch（退出循环或 switch）

| Field | Value |
|---|---|
| Type | Loop / branch control |
| Runtime args | optional boolean |
| Stack effect | consumes the boolean if present |

**Description.** 退出外层 `EACH` 循环或 `SWITCH` 区域。带 boolean 参数时为条件退出；不带参数时为无条件退出。

```go
{1, 3, 30, 5}
EACH{
    $Value {10} GT       // 压入当前值与 10 后比较
    @POP BREAK           // 若为 true，则退出循环
    @$Index PRINT        // 否则打印当前索引
}
@"Done!" PRINT

// 输出：
// 0        // $Value = 1
// 1        // $Value = 3
// Done!
```

---

### 66 — `BLOCK{}(~)` — explicit lexical block（显式词法 block）

| Field | Value |
|---|---|
| Type | Scope |
| Immediate | length (in bytes) of the child block |
| Associated data | child instruction sequence |
| Runtime args | none |
| Stack effect | none |

**Description.** 创建一个子 block——主要用于引入新的 local frame，或为工具链划定一段指令边界（例如通过 `SOURCE` 捕获该 block 自身字节）。

```go
BLOCK{
    ......           // 正常执行
    SOURCE           // 将该 block 的指令字节（含 SOURCE）压入栈
}
@POP PRINT           // 打印捕获到的 block
```

若既不需要隔离作用域也不需要提取源码，则省略 `BLOCK` 在功能上等价。

## Design notes（设计说明）

- **No infinite loops.** VM 有意不提供 `WHILE`/`LOOP`。所有迭代都受已有集合大小（`EACH`）约束，或受 `GOTO` / `EMBED` 递归调用深度的外部限制。
- **Sibling-level `IF` shares state.** 这可避免解析器侧歧义，并与 `CHECK` 已采用的 "last-one-wins" 语义一致。
- **`SWITCH` pre-evaluates candidates.** 在 SWITCH 入口一次性计算 candidate 列表，可获得紧凑、易审计的 `CASE` 主体，并消除逐 CASE 表达式求值开销。
- **`BREAK` vs. `RETURN`.** `BREAK` 用于退出循环或 switch；`RETURN` 用于退出函数式子 block（`MAP{}`、`FILTER{}`）。混用会触发编译期错误。
- **`BLOCK` is opt-in.** 它用于作用域隔离与源码内省场景（例如通过 `SOURCE` 拼接子脚本）；省略时没有隐式副作用。

## Appendix — On loops（附录：关于循环）

该脚本系统不存在无限循环；`EACH` 是最接近的等价机制。作为循环结构，`EACH` 遵循 `BREAK`，**not** `RETURN`。
