# Flow Control（流程控制）

## 来源构想

- `docs/conception/Instruction/7.流程控制.md`：定义 opcode `58-66`。
- `docs/proposal/Instruction/0.Base-Constraints.md`：定义局部域、执行状态和成本边界。

## 范围

流程控制指令包括：

- `IF`
- `ELSE`
- `SWITCH`
- `CASE`
- `DEFAULT`
- `EACH`
- `CONTINUE`
- `BREAK`
- `BLOCK`

## 规范要求

- 子块关联数据必须包含字节长度，长度编码依赖 canonical varint 未决项。
- `IF/ELSE` 的配对和同层状态必须确定，不能依赖格式化缩进。
- `SWITCH` 使用标的值，`CASE` 包含匹配值列表，首个匹配分支执行。
- `DEFAULT` 位置和唯一性必须固定。
- `EACH` 只能迭代确定性集合，迭代顺序必须固定。
- 循环变量只在循环块内有效。
- `BREAK/CONTINUE` 在嵌套 SWITCH/EACH 中的目标必须明确。
- `BLOCK` 建立新的局部域，不继承父块局部值。

## 风险

- `IF/ELSE` 若不要求紧邻，解析和审计容易误解。
- `EACH` 迭代大型集合会造成成本爆炸。
- Dict 迭代顺序必须是插入顺序而不是 map 顺序。
- `BREAK` 同时适用于 SWITCH 和 EACH 时会有目标歧义。

## 未决问题

- 最大块嵌套深度。
- `ELSE` 是否必须紧跟同层 `IF`。
- `DEFAULT` 是否允许不在最后。
- `BREAK/CONTINUE` 可选布尔实参的精确语义。
- EACH 对 Dict 返回 `$Key/$Value` 还是结构项。

## 对后续 Plan 的影响

- 流程控制测试必须覆盖嵌套块、局部域隔离、CASE 匹配顺序、BREAK/CONTINUE 目标和成本限制。
