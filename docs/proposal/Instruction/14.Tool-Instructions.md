# Tool Instructions（工具指令）

## 来源构想

- `docs/conception/Instruction/14.工具指令.md`：定义 opcode `138-163`。
- `docs/proposal/Instruction/0.Base-Constraints.md`：定义成本、公共/私有边界和执行状态。

## 范围

工具指令包括：

- `EVAL`
- `COPY`
- `DCOPY`
- `KEYVAL`
- `MATCH`
- `SUBSTR`
- `REPLACE`
- `RANDOM`
- `SLRAND`
- `CMPFLO`
- `RANGE`
- `MAP`
- `FILTER`
- `SHELL`
- opcode `148-151`、`156-163` 保留。

## 规范要求

- `EVAL` 在独立环境执行 Script 值，返回内部数据栈残留切片。
- `COPY` 浅复制，`DCOPY` 深复制；对象边界必须固定。
- `MATCH` 使用 RE2 语义。
- `RANDOM/SLRAND` 必须是确定性伪随机，固定算法候选为 ChaCha8，seed 长度 32 bytes。
- `CMPFLO` 用于浮点容差比较，比较模式必须固定。
- `RANGE` 最大成员候选值 255。
- `MAP/FILTER` 在私有脚本域迭代，使用 `RETURN` 返回当前迭代结果。
- `SHELL` 只能在私有路径使用，公共验证必须忽略或拒绝。

## 风险

- `EVAL` 可执行动态脚本，需严格成本和深度限制。
- `DCOPY` 对 Module/Object 的边界不清。
- `RANDOM` 若使用系统随机源会破坏共识。
- `MAP/FILTER` 可造成大量执行。
- `SHELL` 绝不能进入公共共识路径。

## 未决问题

- `EVAL` 是否共享 GOTO/EMBED 深度限制。
- ChaCha8 是否最终固定为确定性随机算法。
- `CMPFLO` 容差类型和默认值。
- `RANGE` 是否包含终点。
- `SHELL` 在公共路径是静态拒绝还是运行时失败。

## 对后续 Plan 的影响

- 工具指令测试必须覆盖确定性随机、EVAL 沙箱、MAP/FILTER RETURN 和公共路径 SHELL 拒绝。
