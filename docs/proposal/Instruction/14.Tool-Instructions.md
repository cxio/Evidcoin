# Proposal: Tool Instructions [138–163]（工具指令）

> **Prerequisite:** 本规范建立在 [Base Constraints](0.Base-Constraints.md) 文档之上，该文档定义了全局使用的指令格式、运行时模型与记号约定。


## 1. Overview（概览）

工具指令占用操作码区间 **[138–163]**（共 26 条指令，其中 149–150 未使用，156–163 保留给未来量子安全密码支持）。它们提供常见功能操作，包括脚本求值、集合处理、模式匹配、字符串处理、随机数生成、数值序列、高阶迭代，以及嵌入代码的扩展入口。

### Design Principles（设计原则）

工具指令代表更高层的复合操作：把多个基础步骤合并为单条优化指令。部分指令（`MAP{}`、`FILTER{}`）会创建 **independent script domains**，拥有私有数据栈与参数空间，从而支持函数式编程模式且不污染调用者执行状态。

### Reserved Area（保留区）

操作码 **[156–163]**（8 个槽位）保留给未来量子安全密码原语。随着量子计算对现有密码算法的威胁上升，这些槽位可用于后量子签名校验、密钥交换与哈希函数升级，而无需改变指令集整体结构。


## 2. Instruction Definitions（指令定义）

### 2.1 Script Evaluation（脚本求值）

```
138     EVAL        Execute script
```

- **Arguments:** 一个 Script 值（强类型——必须由 `CODE` 或 `SCRIPT` 指令创建，不能是任意字节）。
- **Return value:** 一个 slice，包含被求值脚本执行后在数据栈中残留的值。
- **Behavior:** 将参数视为脚本指令序列，并在 **independent execution domain**（环境）中执行；该环境与调用者的数据栈、参数空间、全局变量完全隔离。
- 执行结束后，被求值脚本数据栈中剩余的值会被收集为一个 slice 返回给调用者。
- `EVAL` 与调用者仅通过这些返回的栈残留交互——在调用者栈上表现为普通条目。
- **Design note:** 严格的 Script 类型要求有利于防止任意代码注入。
- **Example:**
  ```
  CODE{ {1} {2} ADD }
  EVAL                    // → [3]（包含结果的 slice）
  ```

### 2.2 Shallow Copy（浅拷贝）

```
139     COPY        Shallow copy of a slice
```

- **Arguments:** 一个 slice。
- **Return value:** 一个新 slice，成员为浅拷贝。
- 原始类型成员按值复制；slice 内的子 slice 与 dict **not** 递归复制——新 slice 仍引用同一批内部集合。
- **Example:**
  ```
  {1, 2, {3, 4}}
  COPY                    // → [1, 2, [3, 4]]（内部 [3, 4] 仍是共享引用）
  ```

### 2.3 Deep Copy（深拷贝）

```
140     DCOPY       Deep copy of a slice
```

- **Arguments:** 一个 slice。
- **Return value:** 一个新 slice，成员为递归深拷贝。
- 子 slice 会递归复制直到遇到非 slice 类型。类型为 `any` 的成员若内部包含 slice 或 dict，也会被检查并复制。
- **Example:**
  ```
  {1, 2, {3, {4, 5}}}
  DCOPY                   // → [1, 2, [3, [4, 5]]]（所有嵌套结构彼此独立）
  ```

### 2.4 Dict Key-Value Extraction（Dict 键值提取）

```
141     KEYVAL(1)   Dict key-value extraction
```

- **Auxiliary params:** 1 byte——提取类型：
  - `0`——同时提取 keys 与 values，分别作为两个 slice。
  - `1`——仅提取 keys。
  - `2`——仅提取 values。
- **Arguments:** 一个 Dict。
- **Return value:** 依据提取类型返回 1 个或 2 个 slice。
- 当提取类型为 `0` 时，会返回两个位置一一对应的 slice（keys[i] 对应 values[i]）。两个 slice 会自动展开到接收域。
  - **注意**：不是 slice 的成员被展开，而是两个 slice 作为两个值被展开。
- **Examples:**
  ```
  KEYVAL[0]     // → keys_slice, values_slice（两个 slice，位置 1:1 对应）
                // 注：数据栈中会依次压入 keys_slice 和 values_slice
  KEYVAL[1]     // → 仅 keys_slice
  KEYVAL[2]     // → 仅 values_slice
  ```

### 2.5 Regex Match（Regex 匹配）

```
142     MATCH(1)    Regex match
```

- **Auxiliary params:** 1 byte——匹配模式：
  - `_`（默认）——测试是否匹配，返回 Bool。
  - `g`——FindAll | FindAllString：返回全部不重叠匹配。
  - `G`——FindAllSubmatch | FindAllStringSubmatch：返回全部匹配及其子组捕获。
  - `f`——Find | FindString：返回首个单一匹配。
  - `S`——FindSubmatch | FindStringSubmatch：返回首个匹配及其子组捕获。
- **Arguments:**
  1. 目标字符串或字节序列。
  2. 已编译正则（`*RegExp` 类型）。
- **Return value:** 依据匹配模式，返回 Bool、单个值或 slice。
- **Design note:** 会根据目标值的类型（String | Bytes）自动选择合适的处理方式：字符串用含 String 的系列处理。
- **Examples:**
  ```
  "hello world" /\w+/ MATCH                 // → true（默认测试模式）
  "abc 123 def 456" /\d+/ MATCH[g]          // → ["123", "456"]
  "2025-03-05" /\d+-(\d+)-(\d+)/ MATCH[S]   // → ["2025-03-05", "03", "05"]
  ```

### 2.6 Substring Extraction（子串提取）

```
143     SUBSTR(~)   Substring extraction
```

- **Auxiliary params:** Varint——rune 数量（要提取的字符数）。
- **Arguments:**
  1. 一个 UTF-8 编码字符串。
  2. 起始字符位置（0-based）。
- **Return value:** 提取得到的子串。
- 按 **rune boundaries** 而非字节偏移进行处理，确保多字节 UTF-8 字符可正确截取。
- 若字符串长度不足以完成提取，返回一个空字符串（而不是长度不足的子串）。
- **Examples:**
  ```
  "hello world" {6} SUBSTR[5]   // → "world"
  "hello" {0} SUBSTR[6]         // → ""（长度不足，返回空串）
  ```

### 2.7 String Replacement（字符串替换）

```
144     REPLACE(1)  String replacement
```

- **Auxiliary params:** 1 byte——替换次数。`0` 表示替换全部出现位置。
- **Arguments:**
  1. 目标字符串。
  2. 匹配模式——普通子串或已编译正则（`*RegExp`）。
  3. 替换字符串。
- **Return value:** 应用替换后的新字符串。
- **Examples:**
  ```
  "aabbcc" "bb" "XX" REPLACE[1]     // → "aaXXcc"
  "aabbccbb" "bb" "XX" REPLACE[0]   // → "aaXXccXX"（替换全部）
  "abc123def" /\d+/ "NUM" REPLACE   // → "abcNUMdef"
  ```

### 2.8 Secure Random Number（安全随机数）

```
145     RANDOM      Create secure random number
```

- **Arguments:** 可选——上界（Int 或 BigInt，必须 > 0）。默认上界为 `int64` 最大值。
- **Return value:** 区间 `[0, upper_bound)` 内的密码学安全随机数。
- 使用 `crypto/rand` 生成。
- **WARNING:** 同一脚本在不同节点会得到 **different values**。该指令只能用于 **non-consensus logic**（私有域、`END` 之后区段或 `EVAL` 上下文）。若用于共识关键校验会导致网络分歧。
- **Example:**
  ```
  {100} RANDOM          // → [0, 100) 内安全随机整数
  RANDOM                // → [0, int64_max) 内安全随机整数
  ```

### 2.9 Shuffle Slice（打乱 slice）

```
147     SLRAND      Shuffle slice member order
```

- **Arguments:** 一个 slice。
- **Return value:** 一个新 slice，成员顺序被随机重排。
- 种子来源是密码学安全的，但洗牌算法的随机质量 **not guaranteed** 达到密码学标准。
- 仍受相同共识警告约束——仅用于 non-consensus 逻辑。
- **Example:**
  ```
  {1, 2, 3, 4, 5}
  SLRAND                // → 例如 [3, 1, 5, 2, 4]（随机排列）
  ```

### 2.10 Float Comparison with Epsilon（带 Epsilon 的浮点比较）

```
148     CMPFLO(1)   Float comparison with epsilon
```

- **Auxiliary params:** 1 byte——比较类型：
  - `0`——相等（在 epsilon 容差内）。
  - `-1`——小于或等于（考虑 epsilon 容差）。
  - `1`——大于或等于（考虑 epsilon 容差）。
- **Arguments:** 三个值：
  1. 第一个数值。
  2. 第二个数值。
  3. Epsilon（容差阈值）。
- **Return value:** Bool。
- 该指令用于浮点相等性判断；由于精度限制，普通比较指令（`EQ`、`LT` 等）在该场景下不够可靠。
- **Example:**
  ```
  {0.1} {0.3} ADD {0.4} {0.0001} CMPFLO[0]   // → true（0.1 + 0.3 与 0.4 的差值在 0.0001 内）
  {0.1} {0.2} ADD {0.4} {0.0001} CMPFLO[0]   // → false（0.1 + 0.2 与 0.4 的差值超过 0.0001）
  ```

### 2.11–2.14 Reserved（保留）

```
148     (Reserved, unused)
149     (Reserved, unused)
150     (Reserved, unused)
151     (Reserved, unused)
```

### 2.15 Numeric Sequence（数值序列）

```
152     RANGE(1)    Create numeric sequence
```

- **Auxiliary params:** 1 byte——序列长度（最大 255）。值为 `0` 时创建空 slice。
- **Arguments:**
  1. 起始值（Int 或 Float）。
  2. 步长值（与起始值同类型）。
- **Return value:** 指定长度的数值 slice。
- 结果类型跟随起始值类型，支持 Int 和 Float。
- **Examples:**
  ```
  {10} {1} RANGE[5]     // → [10, 11, 12, 13, 14]
  {0.0} {0.5} RANGE[4]  // → [0.0, 0.5, 1.0, 1.5]
  {100} {-10} RANGE[3]  // → [100, 90, 80]
  {0} {1} RANGE[0]      // → []（空 slice）
  ```
- **Design note:** 最大长度为 255（1-byte 辅助参数限制）。如需更大集合，可生成多个 range 并用 `MERGE` 合并。

### 2.16 Map Iteration（Map 迭代）

```
153     MAP{}(~)    Map iteration
```

- **Auxiliary params:** Varint——子块字节长度。
- **Arguments:** 第一个参数是目标集合（slice 或 dict）。其余参数会成为子块私有数据栈的初始成员。
- **Return value:** 一个 slice，收集全部迭代结果。
- **Behavior:** 遍历目标集合每个成员，并对每个成员执行一次子块。每次迭代通过 `RETURN` 返回的值会被收集到结果 slice。
- **Execution domain:** 子块位于 **independent script domain**，拥有独立数据栈与参数空间。但同一个 `MAP{}` 内各次迭代 **share** 同一份数据栈与参数空间——状态可跨迭代保留。
- **Loop variables**（通过 `$X` 捕获指令访问）：
  - `$Value`（index 0）——当前元素值。
  - `$Index` / `$Key`（index 1）——当前索引（slice）或键（dict）。
  - `$Slice` / `$Dict`（index 2）——集合自身。
  - `$Size`（index 3）——集合大小。
- 某次迭代若返回 `nil`，该结果会被 **ignored**（不会加入结果 slice），从而内建过滤能力。
- **Dict iteration order:** 当目标为 dict 时，迭代顺序遵循 **insertion order**，与有序 map 设计一致（见 [Collection Operations](4.Collection-Operations.md)）。
- **Example:**
  ```
  {1, 2, 3, 4, 5}
  MAP{
       $Value {10} MUL
       RETURN              // 返回当前值 × 10
  }
  // → [10, 20, 30, 40, 50]
  ```

### 2.17 Filter Iteration（Filter 迭代）

```
154     FILTER{}(~) Member filtering
```

- **Auxiliary params:** Varint——子块字节长度。
- **Arguments:** 第一个参数是目标集合（slice 或 dict）。其余参数会成为子块私有数据栈的初始成员。
- **Return value:** 仅包含子块返回 `true` 成员的集合。
- **Behavior:** 执行语义与 `MAP{}` 相同，但子块预期返回布尔值。迭代返回 `true` 的成员被保留，其余剔除。
- 当目标为 **dict** 时，`FILTER` 返回一个新 dict，只保留满足条件的键值对。
- 返回类型与目标集合一致（slice 输入→slice 输出，dict 输入→dict 输出）。
- **Example:**
  ```
  {1, 2, 3, 4, 5, 6}
  FILTER{
       $Value {2} MOD {0} EQ
       RETURN              // 偶数返回 true
  }
  // → [2, 4, 6]
  ```

### 2.18 Embedded Shell Capability（Shell 能力）

```
155     SHELL{}(~)  Embedded shell capability
```

- **Auxiliary params:** Varint——内容字节长度。
- **Associated data:** 子块内容按原始字节处理（shell script）。首行作为解释器指令（shebang，例如 `#!/bin/bash`）。
- **Arguments:** 无。
- **Return value:** 无。
- **Restrictions:** 公共验证节点会忽略该指令，仅由需要该功能的私有域节点使用。

### 2.19–2.26 Reserved for Quantum Safety（量子安全保留区）

```
156–163 (Reserved) — Future quantum-safe cryptographic support
```

为后量子密码原语预留 8 个操作码槽位，可能包括：

- 抗量子签名校验算法。
- 后量子密钥封装机制。
- 量子安全哈希函数。

当量子计算威胁达到需要从现有密码算法迁移时，再定义这些具体指令。


## 3. Instruction Summary（指令总览）

| Opcode | Mnemonic | Category | Description |
|--------|----------|----------|-------------|
| 138 | EVAL | Execution | 在隔离域中执行脚本 |
| 139 | COPY | Collection | 对 slice 做浅拷贝 |
| 140 | DCOPY | Collection | 对 slice 做深拷贝 |
| 141 | KEYVAL | Collection | Dict 键值提取 |
| 142 | MATCH | Pattern | 多模式 Regex 匹配 |
| 143 | SUBSTR | String | 按 rune 位置截取子串 |
| 144 | REPLACE | String | 字符串替换（字面量或 regex） |
| 145 | RANDOM | Random | 随机数（ChaCha8） |
| 146 | SLRAND | Random | 随机打乱 slice 成员顺序 |
| 147 | CMPFLO | Comparison | 带 epsilon 容差的浮点比较 |
| 148–151 | — | — | 未使用 |
| 152 | RANGE | Sequence | 创建数值序列 |
| 153 | MAP | Iteration | 对集合执行函数式 map |
| 154 | FILTER | Iteration | 对集合执行函数式 filter |
| 155 | SHELL | Capability | 执行 shell 脚本（私有） |
| 156–163 | — | Reserved | 未来量子安全密码支持 |

### Design Notes（设计说明）

- **Consensus safety:** 两条指令（`RANDOM`、`SLRAND`）由实参提供随机数种子，使得在不同节点也能拥有确定性结果。
- **MAP/FILTER domain model:** 两条指令都会为子块创建独立脚本域，但同一子块内部各次迭代共享状态。该设计既支持累加器模式（第 N 次迭代依赖 N-1 次结果），又保持子块与调用者隔离。
- **Capability instructions**（`SHELL`）用于能力传递，主要用于有需求的私有节点执行，私有节点可通过花费者地址（公钥）识别是否为自己的需求。
