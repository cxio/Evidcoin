# Tool Instructions（工具指令）

Status: 草案提案
Source conception: `conception/Instruction/14.工具指令.md`

## Prerequisites（前置条件）

请先参阅 `0.Base-Constraints.md`，了解栈模型、本地 frame、opcode 布局、
立即数操作数记法 `(n)` 以及关联数据记法 `{}`。

相关概念回顾：

- **Data stack** 是 opcode 输入/输出的默认通道；
  **argument zone** 用于预先放置运行时参数。
- 本节少数 opcode 会开启隔离的 **child execution domain**（私有
  data stack、私有 argument zone、私有 global frame）。
- **Public validator nodes** 只观测与共识相关的效果；
  某些工具 opcode 在 public 节点上被明确规定为 no-op。

## Overview（概览）

本块聚合了一组通用且高频使用、但不属于更专门类别的工具能力：
slice 复制、dictionary 键值提取、正则匹配/替换、确定性随机数、
浮点比较、区间生成、两个集合迭代器（`MAP` / `FILTER`）、
隔离子脚本求值器（`EVAL`），以及带外 shell 逃逸（`SHELL`）。
此外，尾部保留了一段小型预留区（`156..163`，8 个槽位），
用于未来的 post-quantum 原语。

### 138 — `EVAL`（隔离求值）

- **Type**: tool, isolated execution
- **Operands**: 无（immediate）；1 个 runtime argument
- **Stack effect**: `Script -> Slice`
- **Description**: 将 runtime argument 视为已编译指令序列，
  并在 **independent execution domain** 中执行（拥有独立的
  data stack、argument zone 与 global frame）。该子域与调用方隔离；
  唯一返回通道是子域执行后残留的 data stack，
  其会被封装为单个 slice 并压回调用方 data stack。
  内层脚本的 pass 状态仅在内层生效，不会传播到外层 pass 状态。
- **Errors**: 参数不是 `Script` 类型（合法生产者仅 `CODE` 与 `SCRIPT`）；
  内层脚本触发硬失败；资源预算超限。

### 139 — `COPY`（浅拷贝）

- **Type**: tool
- **Operands**: 1 个 runtime argument
- **Stack effect**: `Slice -> Slice`
- **Description**: 分配一个新的 slice，并将输入 slice 的成员浅拷贝进去。
  引用类型成员会共享。
- **Errors**: 参数不是 slice。

### 140 — `DCOPY`（深拷贝）

- **Type**: tool
- **Operands**: 1 个 runtime argument
- **Stack effect**: `Slice -> Slice`
- **Description**: 与 `COPY` 类似，但会递归下钻嵌套 slice；
  对 `any` 类型成员会检查其是否为 slice，若是则继续递归。
  遇到非 slice 叶子节点即停止递归。
- **Errors**: 参数不是 slice；递归预算超限。

### 141 — `KEYVAL(1)`（字典投影）

- **Type**: tool, dictionary projection
- **Immediate**: 1 byte selector
  - `0` — 同时返回 keys 和 values（两个 slice）
  - `1` — 仅 keys（一个 slice）
  - `2` — 仅 values（一个 slice）
- **Operands**: 1 个 runtime argument（dictionary）
- **Stack effect**: `Dict -> Slice` 或 `Dict -> Slice, Slice`
- **Description**: 将 dictionary 投影为一个或两个 slice。
  当 selector 为 `0` 时，两个 slice 会自动在接收上下文中展开
  （即在 argument zone 中分别出现），且 keys-slice 与 values-slice
  按位置一一对齐。
- **Errors**: 参数不是 dictionary；selector 未知。

### 142 — `MATCH(1){}`（正则匹配）

- **Type**: tool, regular expression
- **Immediate**: 1 byte mode flag（ASCII）
  - `_`（零值）— 仅做匹配测试，返回 `Bool`
  - `g` — 查找所有完整匹配，返回 slice
  - `G` — 查找所有包含捕获组的匹配，返回二维 slice
  - `f` — 查找首个完整匹配，返回单值
  - `S` — 查找首个包含捕获组的匹配，返回 slice
- **Operands**: 2 个 runtime arguments — `target`, `regexp`
- **Stack effect**: `(String|Bytes), RegExp -> Bool|String|Bytes|Slice`
- **Description**: 根据 `target` 类型，分派到底层 regex API 的
  `String` 或 `Bytes` 变体。mode flag 用于选择匹配测试、首个匹配、
  全部匹配及其带子组版本。当 regex 不含捕获组时，优先使用 `g`
  以获得扁平 slice；`G` 始终返回二维 slice。
- **Errors**: mode flag 非法；regex 对象非法；
  `target` 与 regex 变体类型不匹配。

### 143 — `SUBSTR(~)`（子串截取）

- **Type**: tool, string slicing
- **Immediate**: varint，需截取的 rune 数量
- **Operands**: 2 个 runtime arguments — `s`（UTF-8 string）、`start`
  （rune 索引，0-based）
- **Stack effect**: `String, Int -> String`
- **Description**: 返回从 rune 索引 `start` 开始、长度为指定 rune 数量的子串。
  计数基于 UTF-8 rune，而非 bytes。若请求区间超出字符串长度，
  opcode 返回空字符串，并按错误条件处理。
  对栈脚本场景而言，使用 immediate（而非运行时长度）是出于
  安全性/静态分析的刻意设计。
- **Errors**: target 不是合法 UTF-8；start 或 length 越界。

### 144 — `REPLACE(1)`（字符串替换）

- **Type**: tool, string replacement
- **Immediate**: 1 byte 替换次数（`0` 表示全部替换）
- **Operands**: 3 个 runtime arguments — `target`, `pattern`,
  `replacement`
- **Stack effect**: `String, (String|RegExp), String -> String`
- **Description**: 执行字面量或 regex 驱动的替换。
  当 `pattern` 为 regex 对象时，会支持 `replacement` 中的
  捕获组反向引用。
- **Errors**: pattern 非法；类型不匹配。

### 145 — `RANDOM`（确定性伪随机）

- **Type**: tool, deterministic PRNG
- **Operands**: 2 个 runtime arguments — `seed`（32-byte 序列）、
  `bound`（`Int` 或 `Float`，排他上界）
- **Stack effect**: `Bytes, (Int|Float) -> (Int|Float)`
- **Description**: 生成一个非负伪随机数，且严格小于 `bound`，
  返回值数值类型与 `bound` 保持一致。PRNG 采用
  `math/rand/v2` 的 ChaCha8 算法，并由给定字节序列播种，
  因而输出是确定性的且可用于共识。
- **Errors**: seed 长度不是 32 bytes；bound 非正；
  数值类型不受支持。

### 146 — `SLRAND`（确定性洗牌）

- **Type**: tool, deterministic shuffle
- **Operands**: 2 个 runtime arguments — `seed`（32 bytes）、`slice`
- **Stack effect**: `Bytes, Slice -> Slice`
- **Description**: 返回一个新的 slice，其成员是输入成员在 ChaCha8
  （以 `seed` 播种）驱动下得到的确定性排列。输入 slice 不会改变。
  配合 `TOP` / `RANDOM`，调用方可链式洗牌并派生后续 seed，
  源构想中已有示例。
- **Errors**: seed 长度不匹配；参数不是 slice。

### 147 — `CMPFLO(1)`（容差浮点比较）

- **Type**: tool, floating-point comparison with tolerance
- **Immediate**: 1 byte 比较类型
  - `0` — 相等（`==`）
  - `-1` — 小于等于（`<=`）
  - `1` — 大于等于（`>=`）
- **Operands**: 3 个 runtime arguments — `a`, `b`, `epsilon`
- **Stack effect**: `Float, Float, Float -> Bool`
- **Description**: 若 `|a - b| <= epsilon` 与请求关系一致，则返回 `true`。
  该指令用于弥补通用比较块中缺乏稳健浮点相等比较的问题。
  若需要不考虑相等的严格有序比较，仍应使用比较块 opcode。
- **Errors**: 参数不是 `Float`；比较类型未知。

### 148–151（未使用）

未使用。预留给 active tool range 内未来工具 opcode。

### 152 — `RANGE(1)`（序列生成）

- **Type**: tool, sequence generation
- **Immediate**: 1 byte 序列长度（成员个数）
- **Operands**: 2 个 runtime arguments — `start`, `step`（同为 `Int` 或
  同为 `Float`）
- **Stack effect**: `(Int|Float), (Int|Float) -> Slice`
- **Description**: 生成等差序列。元素类型跟随 `start`。
  immediate 为 `0` 时返回空 slice；此时 `start` 仅用于传递类型信息。
  1-byte immediate 对单次调用结果规模施加上限，以兼顾性能与安全；
  更大集合需通过多次 `RANGE` 并 `MERGE` 组合。
- **Errors**: `start` 与 `step` 类型不匹配；immediate 超过 255。

### 153 — `MAP{}(~)`（Map 迭代器）

- **Type**: tool, iterator
- **Immediate**: varint — body 字节长度
- **Operands**: 可变（`>= 1`）runtime arguments — 第一个为
  目标集合，其余用于初始化迭代器的 **private data
  stack**
- **Stack effect**: `Slice|Dict, ... -> Slice`
- **Description**: 对目标集合每个成员，在独立脚本域中执行 body 块。
  该脚本域拥有 **private data stack, private argument zone, and private global
  frame**。其中 data stack、argument zone 与 global frame 在*同一次调用的各迭代之间共享*，
  因而可用于累加器。每次迭代的 `RETURN` 值会进入结果 slice；
  `nil` 返回会被丢弃，因此该 opcode 天然具备一定过滤能力。
  body 内可用循环局部变量：
  - `$Value` — 当前成员
  - `$Index` / `$Key` — 整数索引或字符串键
  - `$Slice` / `$Dict` — 目标集合
  - `$Size` — 迭代总次数
  当目标是 dictionary 时，迭代顺序为插入顺序。
- **Errors**: 目标不是集合；body 触发硬失败；迭代预算超限。

### 154 — `FILTER{}(~)`（Filter 迭代器）

- **Type**: tool, iterator
- **Immediate**: varint — body 字节长度
- **Operands**: 可变（`>= 1`）runtime arguments — 第一个为
  目标集合，其余用于初始化迭代器 private data stack
- **Stack effect**: `Slice|Dict, ... -> Slice|Dict`
- **Description**: 域语义与 `MAP` 相同。body 的 `RETURN` 值按布尔解释；
  truthy 项会保留到输出中，且输出保持源集合的*类型*
  （slice in → slice out，dict in → dict out）。
  原集合不会被修改。
- **Errors**: 目标不是集合；body 触发硬失败。

### 155 — `SHELL{}(~)`（Side-channel 逃逸）

- **Type**: tool, side-channel escape（private）
- **Immediate**: varint — payload 长度
- **Operands**: 1 个 runtime argument — payload bytes
- **Stack effect**: `Bytes ->`（不压入返回值）
- **Description**: 在隔离的 host 环境执行 payload；解释器由 payload 首行
  的 shebang 选择。**Public validator nodes silently ignore this opcode**
  （这*不是*错误）。该指令仅由识别支出公钥且需要该副作用的 private 节点消费。
  由于无返回值，此 opcode 无法影响共识状态，这也是 public 侧 no-op 的安全基础。
- **Errors**: 不会向 public 共识路径传播错误；在 private 节点上，
  host 级执行失败通过 host 通道上报，而非通过脚本上抛。

### 156–163 — Reserved（预留）

工具块尾部连续预留 8 个槽位。目标用途：post-quantum 原语，
或其他可在语义稳定后从 function 集晋升到 base block 的基础安全构件。
分配这些槽位中的任意一个都需要 hard upgrade。

## Design Notes（设计说明）

- **Isolation contract for `EVAL` / `MAP` / `FILTER`.** 三者都会打开子执行域。
  `EVAL` 的子域完全隔离，仅以单个 slice 形式暴露残留 data-stack 内容。
  `MAP` / `FILTER` 使用*迭代私有*的 data stack/argument zone/global frame；
  它们在一次调用内部的各迭代间共享，但对调用方不可见——
  这使循环可安全使用累加器且不会泄漏状态。
- **Determinism.** `RANDOM` 与 `SLRAND` 均使用有种子的 ChaCha8，
  从而保证所有共识参与者在相同 seed 下得到相同结果。
  seed 必须是 32 bytes；不符合即报错。
- **`SHELL` and consensus.** 由于 public 节点忽略 `SHELL`，
  它必须*严格*保持无返回值；任何引入返回路径的未来修订都会破坏 no-op 不变量，
  在本块语义下被禁止。
- **Reservation discipline.** `156..163` 保持连续且块对齐，
  以便未来 post-quantum 发布更易宣告并进行版本门控。
