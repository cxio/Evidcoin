# Base Constraints（基础约束）

Status: Draft proposal（草案提案）
Source conception: `conception/Instruction/0.基本约束.md`

## Overview（概述）

Evidcoin 脚本引擎是一个基于栈、近图灵完备的字节码虚拟机，用于校验
UTXO/UTCO 输出的花费是否合法。脚本字节码一旦作为交易输出的一部分被发布，
在执行期间即为 **immutable and read-only**。因此，基础指令集中不包含任何可
以修改脚本源码本身的 opcode，且未来所有扩展都 MUST 保持这一不变量。

本文档定义了所有指令族（文件 `1.*` 到 `18.*`）共享的基础概念：

- 运行时内存布局（data stack、argument zone、local frame、global frame）；
- 单条指令的二进制结构（opcode、immediate operand、associated data），以及
  immediate operand 与 runtime argument 的区别；
- 脚本级终止行为（END / FAIL）与对应的 PASS / CHECK pass-status 语义；
- 本系列每个指令目录表统一采用的格式约定（使用 `go` 高亮的文档块）。

后续所有指令文档（`1.Value-Instructions.md` 至
`18.Extension-Instructions.md`）都依赖本文件建立的定义，并以简短的
`Prerequisites` 小节回指本文件。

## Runtime Memory Model（运行时内存模型）

脚本执行上下文由四个相互独立的区域组成。它们在逻辑上彼此隔离：指令只能通过
显式规则与各区域交互，绝不通过隐式别名共享。

| Region | Purpose | Lifetime | Capacity |
|--------|---------|----------|----------|
| Data stack | 主要工作区；保存由指令产生并消费的中间值。 | 整个脚本 | 引擎全局上限（`MaxStackHeight = 256`） |
| Argument zone | 临时缓冲区，保存将被 **next** 指令消费的 runtime argument；可为空。 | 单条指令 | 受上游 capture 指令限制 |
| Local frame | 当前语法块私有的仅追加暂存区。父子块之间不继承；同一块内兄弟指令共享。 | 单个块 | 128 slots |
| Global frame | 脚本级共享区域，索引范围 `[0, 255]`，任意块可访问。 | 整个脚本 | 256 slots |

### Data Stack（数据栈）

data stack 是一个 LIFO 的类型值容器（Bool、Byte、Rune、Int、BigInt、
Float、String、Bytes、RegExp、Time、Dict、Script、…）。多数指令要么自动
将结果压入 data stack，要么在未指定显式参数来源时从栈顶消费操作数。

### Argument Zone（参数区）

argument zone 是一个短生命周期的暂存区，用于向 **next** 指令喂入 runtime
argument。它由 **capture instructions** 填充（见
`2.Capture-Instructions.md`，主要是 `@`），并在后继指令读取后立即清空。
对于可接收 runtime argument 的指令，当 argument zone 与 data stack 都可提
供操作数时，优先使用 argument zone。若消费指令不需要输入，argument zone
可以合法为空。

### Local Frame（局部帧）

local frame 绑定到当前执行的语法块。它是仅追加栈：值通过 `$` capture 写入
（见 `2.Capture-Instructions.md`），之后可用 `$[n]` 通过正/负索引
`[-128, 127]` 引用。local frame 不在嵌套块间共享；只有同一块内的兄弟指令可
见同一 frame。单个 local frame 容量固定为 128 entries。

### Global Frame（全局帧）

global frame 是脚本范围内的索引槽表，可按 `[0, 255]` 访问。它用于保存必须
在整个脚本执行期间保持可见的值，也是 environment lookup（见
`13.Environment-Instructions.md`）以及 module/function 指令族（`16.*`、
`17.*`）导出变量的自然目标。

## Opcode Layout（Opcode 布局）

每条指令都由 1-byte 的 **opcode** 标识。opcode 空间按连续区间划分，每个区
间对应一个指令族。本提案系列定义的区间如下：

| Range | Family | Document |
|-------|--------|----------|
| `0x00 – 0x12` | Value instructions | `./1.Value-Instructions.md` |
| `0x13 – 0x17` | Capture instructions | `./2.Capture-Instructions.md` |
| `0x18 – 0x22` | Stack operations | `./3.Stack-Operations.md` |
| `0x23 – …`    | Collection, interaction, result, flow control, … | `./4.*` and beyond |

完整 opcode 表汇总于 script engine 规范（`../05-Script-Engine.md`）。

## Anatomy of an Instruction（指令构成）

每条 on-chain 指令编码为三部分拼接：

```
opcode (1 byte) | immediate operand(s) | associated data
```

其中仅 opcode 为必选；其余两部分是否出现及其宽度取决于具体指令。

### Opcode（操作码）

单字节，唯一标识一条指令。某些情况下仅凭 opcode 即可完整表达该指令语义
（如 `NIL`、`TRUE`、`FALSE`）。

### Immediate Operand（附参）

与 opcode 绑定的编译期硬编码参数。immediate operand 描述的是
**properties of the instruction itself**（例如 associated data 块长度、
local frame 索引，或同类变体选择器）。一条指令 MAY 携带多个 immediate
operand；它们以固定位置顺序紧跟在 opcode 之后。

### Associated Data（关联数据）

可选负载，内联于 immediate operand 之后。associated data 常用于嵌入较大字面
量（byte string、code block、regular expression 等）。当指令携带
associated data 时，文档块会明确标注，且源码级语法使用花括号 `{}` 包裹该
payload。

### Runtime Argument（实参）

runtime argument **not** 属于指令编码本体。它们是运行期产生的值——若前置
capture 指令已填充 argument zone，则从 argument zone 获取；否则从 data
stack 弹出。若某指令不需要 runtime argument，其文档条目将省略该小节。

## Script-Level Behaviors（脚本级行为）

脚本终止方式严格二选一：

- **END** —— 正常退出。校验结果由当前 pass status 决定：
  - `PASS` → 校验成功；
  - `CHECK`（或未到达 PASS）→ 校验失败。
- **FAIL** —— 显式或隐式失败。执行过程中的任意错误（stack underflow、
  arithmetic overflow、type mismatch、capacity violation 等）都会立即使
  已设定的 `PASS` 失效，并中止脚本，返回失败校验结果。

负责设置 `PASS`、触发 `FAIL`、或基于 pass-status 分支的具体指令，定义于
`./6.Result-Instructions.md`。

## Documentation Convention（文档约定）

本系列每份 instruction-catalog 文档采用统一结构：先是叙述性头部，再是一个
`go` fenced code block 列出该族指令。`go` 高亮仅用于展示效果，内容
**not** 是 Go 代码，而是为阅读优化的格式化伪文本。

每个目录条目包含以下列与子字段：

```go
opcode  mnemonic    short description
        // Type: <指令产生的 runtime type>
        // Immediate: <name（字节宽度；~ = varint）>, ...
        // Data: <associated payload 的说明>
        // Args: <runtime argument 及其顺序>
        // Stack effect: ( before -- after )
        // Errors: <触发 FAIL 的条件>
        // Description、设计意图、示例...
```

本系列统一字段约定如下：

- **opcode** 使用十进制书写；每个 opcode 严格占用 1 byte。
- **Immediate operand** 的宽度在源码级语法中写在 mnemonic 后的括号内
  （例如 `VALUE(1){}` 表示 1-byte immediate）；`~` 表示 variable-length
  integer（varint）。
- **Associated data** 在 mnemonic 中用尾随花括号 `{}` 表示。
- **Stack effect** 在无歧义时采用标准 Forth 记法 `( a b -- c )`；否则用
  文本描述。
- **Errors** 列出所有导致脚本以 FAIL 中止的条件；若缺省该字段，表示该指令
  本身不会失败。

### Worked Example（示例解析）

字面量 `DATA{414243}`（承载字节 `"ABC"`）会编译为以下 on-chain 字节序列：

```
0x0A 0x03 0x41 0x42 0x43
 ^    ^    ^^^^^^^^^^^^
 |    |    associated data（"ABC"）
 |    immediate operand：payload length = 3
 opcode for DATA（10）
```

同理，源码级形式 `VALUE(1){}` 表示 immediate operand 位于可选花括号之前，
且该花括号仅用于给索引提供符号别名——它们 NOT 是 associated data。可对比
`DATA{}(~)`：其中花括号明确表示 associated data，且前置 varint 长度。
