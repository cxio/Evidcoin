# Proposal: Base Constraints for the Script Instruction Set（提案：脚本指令集的基础约束）

本文定义 Evidcoin 脚本系统中所有指令共同遵循的基础约束与结构约定。各类指令规范默认都隐含采用本文所述规则。


## 1. Script Immutability（脚本不可变性）

脚本在执行期间是 **read-only**。求值过程中，指令流在任何时刻都不得被修改。因此基础指令集中没有可以改变脚本本身的指令。

未来如果定义新的指令，需要遵循此原则：当指令在语义上需要“修改”其操作数（例如基于内联数据构造值）时，引擎必须在副本上操作，以保证原始脚本内容不变。


## 2. Core Runtime Concepts（核心运行时概念）

### 2.1 Data Stack（数据栈）

用于存放指令产生和消费的运行时值，采用 LIFO 结构。

- **Maximum height:** 256 entries.
- **Maximum item size:** 1,024 bytes (1 KB).
- 所有值类指令默认都会将结果压入 data stack，除非被捕获前缀（`@`、`$`）拦截。

### 2.2 Argument Space（实参空间）

在指令消费参数前用于暂存操作数的 FIFO 队列。

- 当指令需要参数时，引擎先检查 argument space；仅当其为空时，才回退到 data stack。
- 参数消费是 **atomically** 的：要么全部从 argument space 取齐，要么一个也不取（整体回退到 stack）。
- 该机制将操作数顺序与栈位置解耦，这是相较传统纯栈脚本语言的关键创新。

### 2.3 Local Scope（本地作用域）

块私有、仅追加的存储区。

- **Maximum capacity:** 128 entries per scope.
- 每个语法块（`IF{}`、`EACH{}`、`BLOCK{}` 等）都会创建独立的 local scope。
- 作用域之间 **No inheritance**：同级块与嵌套块都不能互相访问 local scope。
- 值通过 `$`（local store）捕获前缀写入，通过 `$[n]`（local retrieve）读取。

### 2.4 Global Scope（全局作用域）

脚本域内共享的变量空间。

- **256 slots**，索引范围 `[0-255]`。
- 通过 `VAR` / `SETVAR` 指令访问。
- 在同一次脚本执行中，所有块共享该空间，包含 `EMBED` 目标脚本。
- 跨 `GOTO` 边界 **Not shared**（每个 GOTO 都会形成独立的脚本域）。

### 2.5 Opcode（操作码）

每条指令对应的单字节唯一标识。

- 指令集可用范围：`[0-253]`（共 254 个可用 opcode）。
- `254` 和 `255` 预留给系统使用，不属于指令集。

### 2.6 Auxiliary Parameters（辅助参数）

紧随 opcode 字节、在编译期确定的常量参数。

- 表示指令自身的静态属性（如块长度、算法选择器、索引等）。
- 可为定长（1、2 或 64 字节）或变长整数（varint），以具体指令定义为准。
- 若一条指令有多个辅助参数，按声明顺序连续出现。

### 2.7 Associated Data（关联数据）

位于辅助参数之后、由指令携带的负载数据。

- 用于承载较大数据块：字符串内容、字节序列、嵌入代码块等。
- 长度通常由前置的辅助参数给出。
- 携带关联数据的指令会在 mnemonic 中用 `{}` 标识（如 `DATA{}`、`CODE{}`）。

### 2.8 Arguments（运行时参数）

指令执行时消费的运行时值。

- 来源优先级：argument space（优先）-> data stack（回退）。
- **不**属于脚本中该指令的字节表示。
- 每条指令都会定义参数需求：`0`（无）、`n`（固定个数）或 `-1`（可变/可选）。


## 3. Instruction Byte Layout（指令字节布局）

编译后的 bytecode 中，每条指令都遵循以下顺序布局：

```
[Opcode: 1 byte] [Auxiliary Params: 0-N bytes] [Associated Data: 0-N bytes]
```

- **opcode** 始终固定为 1 字节。
- **Auxiliary parameters** 紧随其后，其数量和长度由 opcode 对应规范决定。
- **Associated data** 位于辅助参数之后，其长度通常编码在前置辅助参数中。
- **Arguments** 仅存在于运行时，绝不会序列化进 bytecode。


## 4. Instruction Documentation Format（指令文档格式）

所有提案文档中的指令说明统一使用以下记号约定：

### 4.1 Header Line Format（头部行格式）

```
<opcode>    <mnemonic>[{}][(aux_params)]    <brief description>
```

- **Opcode**：十进制整数（`0`-`253`）。
- **Mnemonic**：全大写指令名（如 `DATA`、`GOTO`、`IF`）。
- 紧跟在 **Mnemonic** 之后的 **`{}`** 表示该指令携带 associated data。但类似 `STRING(1){}` 这样的指令，花括号在 `(aux_params)` 之后，因此不表达关联数据。
  - 此时 `{}` 通常表示 `(aux_params)` 可采用别名，如 `STRING{E}`，`E` 为 Auxiliary Params 某个值的别名。具体含义在指令定义中有说明。
- **`(aux_params)`**：括号中的辅助参数长度列表，用逗号分隔。`~` 表示 varint，`64` 表示固定 64 字节字段。多个参数示例：`(~,64,~)`。
- **Brief description**：该指令用途的简短摘要。

> **注意：**
> - 部分值指令直接使用花括号 `{}` 表达指令本身（`Mnemonic`），如 `{12345}`（一个整数），花括号为指令的语法形式。
> - 花括号 `{}` 的含义较为灵活，需结合上下文判断，其含义在指令的具体说明中有表达。

### 4.2 Detail Block（详情块）

在头部行下方，以清单的形式详细说明指令的各个方面：

- **Type**：压入栈中的结果类型（若有）。
- **Auxiliary parameter descriptions**：每个参数的用途与编码方式。
- **Associated data description**：内容格式与语义。
- **Argument requirements**：会消费哪些运行时操作数。
- **Examples**：给出具体用法。
- **Design notes**：给出设计依据或实现提示。

### 4.3 Syntax Conventions（语法记号约定）

| 记号 | 含义 |
|----------|---------|
| `INSTR{content}` | 带 associated data 的指令（花括号） |
| `INSTR[value]` | 使用索引/数值形式表达辅助参数的实际值（方括号），如 `DATE[1606227845000]` |
| `INSTR(n)` | 辅助参数定义：固定 `n` 字节 |
| `INSTR(~)` | 辅助参数：变长整数（varint） |
| `INSTR(~,64,~)` | 多个辅助参数按顺序出现 |


## 5. Instruction Set Segmentation（指令集分段）

253 个可用 opcode 被划分为连续的四个段：

| 分段 | Opcode 范围 | 数量 | 用途 |
|---------|-------------|-------|-------|
| Base Instructions | `[0-169]` | 170 | 核心语言：值、栈操作、流程控制、算术、比较、逻辑、模式匹配、环境、工具、系统调用 |
| Function Instructions | `[170-209]` | 40 | 内置函数：编码、哈希、签名校验，以及 1 个自扩展槽（`FN_X`） |
| Module Instructions | `[210-249]` | 40 | 分类模块：math、time、regex，以及 1 个自扩展槽（`MO_X`） |
| Extension Instructions | `[250-253]` | 4 | 通用扩展点：`EX_FN`、`EX_MO`、`EX_INST`、`EX_PRIV`，每个可通过 2 字节子 **Auxiliary parameter** 扩展到 64K 个指令 |

### Unlock Script Restriction（解锁脚本限制）

Unlock script（置于交易输入中）仅允许使用 `[0-50, 169]` 范围的 opcode（共 51+1 条）。这样可将其能力限制在值指令、捕获指令、栈操作、集合操作和交互指令，避免其直接执行流程控制、密码学操作或系统调用。


## 6. Resource Limits Summary（资源上限汇总）

| 资源 | 上限 |
|----------|-------|
| Data stack height | 256 entries |
| Stack item size | < 1,024 bytes (1 KB) |
| Lock script size | < 1,024 bytes (1 KB) |
| Unlock script size | < 4,096 bytes (4 KB) |
| Local scope capacity | 128 entries per block |
| Global variable slots | 256 |
| Maximum transaction size | 65,535 bytes |
| Unlock script opcode range | `[0-50, 169]` |
