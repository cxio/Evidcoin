# Base Constraints（基本约束）

## 来源构想

- `docs/conception/Instruction/0.基本约束.md`：定义数据栈、实参区、局部域、全局域、指令结构和说明格式。
- `docs/conception/6.脚本系统.md`：定义脚本 VM、opcode 分段、资源限制和公共/私有边界。
- `docs/proposal/09.Script-System.md`：定义脚本系统总 Proposal。

## 目标

本文件固定所有指令 Proposal 的共同元规则：指令结构、取参规则、返回值、错误/失败状态、源码与 bytecode 边界、公共验证和私有执行边界。

## 指令结构

每条指令由以下部分组成：

- `opcode`：1 byte，必需，唯一标识指令。
- `attached parameters`：附参，编译期固化，用于表达指令属性。
- `associated data`：关联数据，指令直接携带的数据块。
- `runtime arguments`：运行时实参，不属于指令 bytecode 本身。

链上脚本是已编译 bytecode。源码文本、注释、友好缩写和示例语法不得作为 Hash 或签名输入。

## 运行时空间

- 数据栈：LIFO，存放运行时值和返回值。
- 实参区：FIFO，临时存放指令实参。
- 局部域：块级私有，无父子继承。
- 全局域：脚本域共享，`GOTO` 不继承，`EMBED` 共享。
- INPUT/OUTPUT 缓存区：独立通道，不互通。

## 实参模型

指令实参数量必须属于以下模型：

```text
0   无实参
n   固定数量
-1  不定数量，只取实参区
-2  半确定性，仅 CALL 类指令
```

固定数量指令规则：

- 实参区为空时，从数据栈取 `n` 个值。
- 实参区非空时，数量必须恰好等于 `n`。
- 实参区数量不足或过多均失败，不能部分消费。

不定数量指令规则：

- 只读取实参区。
- 实参区为空时得到空实参。
- 不访问数据栈。

## 截取与自动展开

- 截取指令改变目标指令返回值的去向。
- 自动展开只适用于明确声明返回多值的指令。
- 返回 Slice 的指令默认不自动展开，除非使用显式展开指令或该指令专门声明。
- `~` 只影响目标指令的取参来源，不改变返回值去向。

## 执行状态

- 正常执行到脚本末尾时，以 pass 状态决定验证结果。
- 普通错误导致失败。
- `PASS false` 导致失败并结束。
- `CHECK false` 标记失败但不立即退出，最终以最后或当前 pass 状态为准；覆盖语义需由结果指令 Proposal 固定。
- `END` 是公共验证结束边界。
- `INPUT` 在公共验证中是隐式 `END`。

## 公共验证边界

公共验证路径不得依赖：

- 本机真实时间。
- 系统随机源。
- Shell。
- 私有扩展。
- 未验证外部输入。
- 服务节点返回的未验证数据。

私有监听逻辑应位于公共验证结束之后。公共节点可以忽略私有逻辑，但私有节点可继续执行并触发中间件业务。

## 指令元数据要求

后续 Plan 应为每条指令维护元数据：

- opcode。
- mnemonic。
- attached parameter schema。
- associated data schema。
- argument count model。
- return count。
- unlock-script availability。
- deterministic flag。
- public/private availability。
- cost class。
- error cases。

## 未决问题

- canonical unsigned varint 算法。
- signed integer 编码。
- 初始 pass 状态。
- `CHECK` 覆盖规则。
- 栈项大小对复合值的计算方式。
- 成本模型的具体数值。
- 公共路径中私有指令是静态拒绝还是运行时失败。

## 对后续 Plan 的影响

- 所有指令实现必须先通过统一 bytecode 解析器。
- 指令执行器必须有统一错误、失败、结束和 pass 状态模型。
- 每类指令测试必须覆盖取参来源、返回值去向、资源限制和公共/私有边界。
