# Script System（脚本系统）

## 来源构想

- `docs/conception/6.脚本系统.md`：定义栈脚本、实参区、局部域、缓存区、opcode 分段、执行、安全和成本模型。
- `docs/conception/Instruction/*.md`：定义 0-18 类指令的详细构想。
- `docs/conception/examples/*.md`：提供存在性证明、版权登记、链间传信、累积付款、网购支付、实时税务、选举子链等脚本用例。
- `docs/proposal/01.Types-And-Encoding.md`：提供基础规范化编码边界。
- `docs/proposal/02.Cryptography-And-Hashing.md`：提供 Hash、地址和签名接口边界。
- `docs/proposal/03.Identifiers-And-Constants.md`：提供脚本资源限制常量。

## 目标

本提案定义脚本系统的总边界。具体指令语义由 `docs/proposal/Instruction/` 下 0-18 类提案承接。

脚本系统目标：

- 提供确定性的栈式 VM。
- 支持标准支付验证、定制验证、外部脚本跳转、私有监听和中间件驱动。
- 通过硬资源限制、成本预算、公共/私有边界和扩展治理控制风险。
- 与交易、状态、签名和公共服务通过接口协作，不直接耦合高层实现。

## 非目标

- 不定义交易输入/输出结构。
- 不定义签名消息完整字段。
- 不实现外部脚本查询服务。
- 不允许公共验证依赖本地时间、随机源、Shell、私有扩展或未验证外部输入。
- 不把源码文本作为链上规范化编码。

## VM 模型

- 脚本是已编译 bytecode，即 `opcode +附参 +关联数据` 的值序列。
- opcode 为 1 byte。
- 基础可用 opcode 范围为 `0-253`；`254` 和 `255` 系统保留。
- 指令分段：`0-169` 基础指令、`170-224` 函数指令、`225-250` 模块指令、`251-253` 扩展指令。
- VM 从起始位置线性执行，脚本自身只读，不允许修改。
- 公共验证路径必须确定性。
- 源码文本只用于人工书写和反编译展示，不参与链上 Hash 或签名。

## 运行时空间

- 数据栈：LIFO，存放运行时值、返回值和中间结果，受 `MaxStackHeight` 限制。
- 实参区：FIFO，用于提前截取实参；固定参数指令优先从实参区取值，实参区为空时从数据栈取值。
- 局部域：块级私有，容量候选值 128，不继承父块，也不被子块访问。
- 全局域：脚本域共享，`VAR/SETVAR` 候选位置为 `0-255`。
- 循环域变量：`$Value`、`$Index`/`$Key`、`$Slice`/`$Dict`、`$Size`。
- INPUT 缓存区和 OUTPUT 缓存区相互独立，不互通。

## 实参规则

指令实参数量模型：

- `0`：无实参，不读取实参区。
- `n`：固定数量。实参区非空时数量必须恰好等于 `n`；实参区为空时从数据栈弹出 `n` 个。
- `-1`：不定数量，只读取实参区；实参区为空则取空，不访问数据栈。
- `-2`：半确定性，仅 `CALL` 类指令使用，运行时由附参决定数量。

`~` 实参直取前缀强制目标指令从数据栈取参，跳过实参区。与截取前缀组合时，截取目标决定返回值去向，`~` 只决定取参来源。

## 值模型

运行时值集合包括：

- `Nil`
- `Bool`
- `Byte`
- `Rune`
- `Int`
- `BigInt`
- `Float`
- `String`
- `Values` / `Slice`
- `Bytes`
- `RegExp`
- `Time`
- `Dict`
- `Script`
- `Module` / `Object`
- `Any`

链上 bytecode 编码和运行时值表示不同。脚本值编码、signed integer、BigInt、Float、Time、Script 等具体字节格式由指令提案固定；不能直接套用基础层 unsigned varint。

## 执行状态

脚本执行至少包含以下状态：

- `Running`
- `Passed`
- `Failed`
- `Exited`
- `Returned`
- `EndedForPublicValidation`

规则：

- 初始 pass 状态倾向为 `false`，实现前必须确认。
- `PASS true` 设置通过并继续执行；`PASS false` 失败并结束。
- `CHECK` 设置或覆盖 pass 状态，但不结束。
- 执行到脚本末尾时，以当前 pass 状态决定验证结果。
- 普通执行错误导致失败。
- 公共验证遇到无数据 `INPUT` 时视作隐式 `END`，不应引入本地非确定输入。
- `END` 对公共验证节点等价退出；私有监听节点可在 `END` 后继续执行私有逻辑。

## 脚本边界

解锁脚本：

- 仅允许 opcode `0-50`。
- `SYS_NULL` opcode `169` 是例外，可用于解锁脚本。
- 主要承载值、签名数据、公钥和简单准备逻辑。

锁定脚本：

- 可使用完整公共指令集。
- 必须受资源和成本限制。
- 公共验证逻辑应位于 `END` 或隐式 `INPUT` 之前。

外部脚本：

- `GOTO` 在独立脚本域执行目标输出脚本，传入实参，不返回数据栈残留。
- `EMBED` 在当前脚本域合并执行目标脚本，共享数据栈、实参区和全局域。
- `SCRIPT` 将外部输出脚本作为值入栈，不执行。
- `CODE` 将脚本文字值入栈，不执行。
- `EVAL` 执行 Script 值，在独立环境中返回内部数据栈残留切片。

## 资源与成本

硬上限引用基础常量：

```text
MaxStackHeight = 256
MaxStackItem = 1024
MaxLockScript = 1024
MaxUnlockScript = 4096
```

动态限制：

- `EMBED` 嵌入次数 `<= 4`。
- `EMBED` 嵌入深度 `== 0`，嵌入脚本不可再 `EMBED` 或 `GOTO`。
- `GOTO` 顶层跳转次数 `<= 2`。
- `GOTO` 跳转深度 `<= 3`。
- `EVAL`、`MODEL`、正则、`MAP/FILTER`、签名验证和外部脚本载入必须计入成本。

成本模型必须确定性，只依赖指令、附参、数据长度、实参类型/长度和固定算法。不得依赖硬件、墙钟时间或本地负载。

## 公共验证与私有监听

- 公共验证只执行确定性前缀。
- `END` 后可包含私有监听逻辑，例如输出消息、链间触发和中间件业务。
- `INPUT` 对公共验证节点是隐式 `END`。
- `SYS_TIME`、`SHELL`、`EXT_PRIV` 只能出现在公共验证结束后。
- `OUTPUT/BUFDUMP` 可用于外部监听，但不得改变共识状态。

## 未决问题

- 初始 pass 状态和无 `PASS/CHECK` 的脚本末尾结果。
- `INPUT` 无数据时是否保留既有 pass 状态。
- `CHECK` 是否允许从 true 覆盖为 false。
- `EXIT` 返回值是否可被交易验证使用。
- `SYS_TIME`、`SHELL`、`EXT_PRIV` 在 `END` 前出现时是静态拒绝还是运行时失败。
- `SCRIPT` 在解锁脚本中的政策：允许、限制或降低优先级。
- 完整成本公式、最大块嵌套深度和 `EVAL` 是否共享 `GOTO/EMBED` 深度限制。

## 对后续 Plan 的影响

- 脚本 Plan 必须先实现 bytecode 解析、栈、实参区、执行状态和资源限制，再实现指令。
- 每类指令必须引用 `Instruction/0.Base-Constraints.md` 的元规则。
- 公共验证和私有监听必须在执行器中显式区分，不能靠调用方约定。
- 签名、交易、状态和外部脚本访问必须通过接口注入。
